Applies to

Scala and Play Framework 2.1.0+

What to Check For

Verify that strong asymmetric encryption is used.

Why

Using strong encryption algorithms helps prevent unauthorized disclosure of sensitive information.

How to Check

To verify that strong asymmetric encryption is used:

  1. Identify sensitive data. Find out what data should be encrypted using asymmetric encryption.

  2. Identify code that handles sensitive data. Find code that handles the sensitive data.

  3. Verify that strong encryption is used. Examine code that handles sensitive data to make sure that it uses RSA with 2048-bit or longer keys whenever asymmetric encryption is needed.

How to Fix

To use strong asymmetric encryption:

  1. Use Scala cryptographic APIs. Use the built-in Scala cipher with 2048 bit keys. For example, use RSA in Scala.

  2. Write cryptographic utility functions that use RSA-2048. Write functions that generate keys, encrypt, and decrypt data.

  3. Use the utility functions. Use the cryptographic utility functions whenever data needs to be encrypted or decrypted using symmetric cryptography.
