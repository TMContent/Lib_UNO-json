Applies to

  • C++

What to Check For

Verify that regular expressions are used to constrain input as the first step of the input validation strategy.

Why

A good input validation strategy is to constrain input to known valid values, reject known bad values and to sanitize the input that passes validation. Rejecting bad values is not to be relied on, because there are too many possible bad values. However, it is possible to filter out at least some bad input. Constraining input can be done effectively using regular expressions.

How to Check

Use the following steps to verify that regular expressions are used to constrain input:

  1. A regular expression library is used. C++ doesn't have built-in regular expression functionality, so check if a regular expression library is used. One way to do this is to check project documentation and another is to see which libraries are linked to in the project.

  2. Identify the input and data validation subsystem. Examine the application structure with focus on input and data validation functions to verify that a dedicated class or library exists for input and data validation functions.

  3. Verify that input is constrained. Examine the validation functions to make sure they constrain known valid input first using regular expressions.

How to Fix

Perform the following actions to constrain input using regular expressions:

  1. Use a regular expression library. C++ does not have built-in regular expression functionality. The Boost.Regex library is recommended for regular expressions with C++. If your application uses managed code, .NET has a nice regular expressions API built-in as System.Text.RegularExpressions.

  2. Determine valid input patterns. Identify the types of valid input for your input application. For example, some input follows obvious patterns, such as social security or credit card numbers.
