Applies to

Scala and Play Framework 2.1.0+

What to Do

Require TLS for all private pages.

Why

TLS should be used to protect all pages that handle sensitive data, including all pages that use authentication or authorization features, to prevent authentication hijacking.

When

Always require TLS for all private pages.

How

The simplest approach for enabling TLS for all sensitive pages is to use TLS for the entire site. Enabling TLS for the entire site may require additional computing resources, so, for performance reasons, it may be acceptable to enable TLS only for sensitive pages. To enable TLS only for sensitive pages:

  1. Identify sensitive pages. Make a list of all pages that handle sensitive information, including all pages that use authentication or authorization features.

  2. Enable TLS for each folder containing sensitive pages. The exact instructions for doing so are different for different web servers. Use the web server administration interface or edit the server configuration files to enable TLS for all folders containing sensitive pages.

  3. Redirect non-TLS requests to TLS-protected pages. The exact instructions for doing so are different for different web servers. Use the web server administration interface or edit the server configuration files to redirect non-TLS requests to TLS-protected pages.
