Applies to

  • Microsoft® Windows Communication Foundation (WCF) 3.5
  • Microsoft Visual Studio® 2008

What to Do

If you need to support multiple transactions per session using Secure Conversation, consider using Message Security.

Why

This allows you to have improved performance.

When

When you want to exchange multiple messages with the WCF service.

How

If your WCF clients need to exchange multiple messages with the WCF service, you can use the Secure Conversation feature for improved performance. Using Secure Conversation means the token negotiation and authentication happens only once for all the requests in a session.

Secure Conversation is enabled with message security on all the standard bindings that support WS-Security (for example, WsHttpBinding, NetTcpBinding and netMsmqBinding).

Here is the configuration example showing the secure conversation turned on.

...
<wsHttpBinding>
  <binding name="NewBinding0">
    <security>
      <message establishSecurityContext="true" />
    </security>
  </binding>
</wsHttpBinding>
...
