Applies To

  • Web Applications

What to Do

Require authentication for all private resources.

Why

All pages that have private information should require authentication to prevent unauthorized information disclosure.

How

To require authentication for all private resources:

  1. Identify all sensitive pages. Make a list of all pages that handle sensitive or private information.
  2. Define roles. Define access roles for your application. These roles should include administrative users and various types of users specific to your application.
  3. Implement authentication. Implement and document authentication APIs to satisfy all application requirements.
  4. Require authentication for all sensitive pages. Add code to all sensitive pages that checks if the user is authenticated and authorized to view them.
