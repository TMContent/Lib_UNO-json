What to Check For

Verify that dynamic SQL in stored procedures is parameterized.

Why

Using dynamic SQL in stored procedures to build queries that include user input may result in SQL injection vulnerabilities, unless user input is explicitly parameterized inside the stored procedures themselves.

How to Check

To verify that dynamic SQL in stored procedures is parameterized:

  1. Identify all stored procedures that use dynamic SQL. Review the database to identify all stored procedures that use dynamic SQL.

  2. Review all stored procedures that use dynamic SQL. Examine each stored procedure that uses dynamic SQL to make sure that user input is separated from the SQL statement. Pay particular attention to the lines that include the EXEC or the EXECUTE statements. These lines should pass user input as parameters to the stored procedures.

How to Fix

To parameterize dynamic SQL in stored procedures:

  1. Identify all stored procedures that use dynamic SQL. Review the database to identify all stored procedures that use dynamic SQL.

  2. Review all stored procedures that use dynamic SQL. Examine each stored procedure that uses dynamic SQL to make sure that user input is separated from the SQL statement. Pay particular attention to the lines that include the EXEC or the EXECUTE statements. These lines should pass user input as parameters to the stored procedures.

  3. Rewrite affected stored procedures. Rewrite each stored procedure that does not explicitly parameterize untrusted data. The exact syntax for this is different for different platforms. Please, refer to your database server vendor's documentation for information about using parameterized stored procedures.
