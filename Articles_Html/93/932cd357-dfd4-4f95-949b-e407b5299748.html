Applies to

Scala and Play Framework 2.1.0+

What to Check For

Verify that domain and path values of session cookies are strict.

Why

Setting strict domain and path values on session cookies makes it harder for attackers to hijack user sessions.

How to Check

To verify that domain and path values of session cookies are strict:

  1. Define a strict cookie path. Use the application path as the cookie path.

  2. Define a strict cookie domain. If the application uses a subdomain, use it as the cookie domain value.

  3. Examine the path and domain values. Check the "application.conf" file to make sure that strict values for the session cookie domain and path properties are set, as follows:

    application.defaultCookieDomain=.example.com

    Where ".example.com" is the domain or subdomain used by the application.

How to Fix

To set strict domain and path values on session cookies:

  1. Define a strict cookie path. Use the application path as the cookie path.

  2. Define a strict cookie domain. If the application uses a subdomain, use it as the cookie domain value.

  3. Set the path and domain values. Set the cookie domain using “application.defaultCookieDomain=.example.com” in the “application.conf” file.
