Applies to

  • PCI DSS Requirement 10.2.2

What to Check For

Actions performed by a user with root or administrator privileges should be logged.

Why

All actions performed by an administrator or another privileged user should be logged to assist with investigations in the event of a compromise or some other situation.

How to Check

Interview the person who is responsible for reviewing the logs to verify that there is a solution in place that records all actions taken by users with Administrator privileges on Windows systems. Windows does not have built-in facilities for this, so the solution should be in the form of a third-party product. Examine the logs produced by the user-monitoring solution to verify that they are current and provide a sufficient level of detail.

How to Fix

Logging administrator actions on Windows is difficult, because the Windows interface is mostly graphical. There is no built-in functionality in Windows that allows recording Administrator activity to the extent required by PCI DSS. The functionality for satisfying Requirement 10.2.2 should be implemented by your centralized log server solution. Please refer to the vendor's documentation or technical support for more information about the capabilities of the product. Install and configure your Administrator logging solution of choice by following the vendor's directions.
