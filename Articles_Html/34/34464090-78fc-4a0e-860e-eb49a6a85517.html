What to Do

Ensure that applications cannot read each other's data or interfere with one another in any way when the deployment environment dictates that appications cannot assume implicit trust with each other. Place restrictions on the types of shared system resources that the applications can access. 

Why

Isolation strengthens resiliency because it makes each application separate and independent. In the event that an application is compromised, isolated apps are less prone to cascading failure (i.e., security vulnerabilities that affect one application will be isolated from the other applications.)

When

Service providers that need to host multiple applications or anyone responsible for running multiple applications in a shared environment should be isolating their software components. Whenever multiple applications reside on a shared hosting environment, this is appropriate. This is not appropriate when a single application resides on a server and does not share any resources with other applications or services.

How

To use code access security, follow these ordered steps:

  1. Identify the permissions your application needs. Identify the precise set of code access security permissions that your application requires. You can do this by manually reviewing your code, or by using the Permissions Calculator (permcalc.exe).

  2. Choose an appropriate trust level. Try to evaluate whether the permissions required for your application match those provided by any of the standard trust levels.

  3. Configure your ASP.NET application. You can configure your ASP.NET application to use a standard level either in your application's Web.config file or in the machine-level Web.config file. Machine level configuration affects all the ASP.NET Web Applications and Web services hosted on that server. Application-level configuration affects only a specific application.  

  4. Optionally create a custom trust level. You may require a custom trust level if none of the predefined trust levels provide the exact set of permissions that your application needs to run. For example, you may find that your application needs more permissions than Medium trust provides but fewer permissions than High trust provides.
