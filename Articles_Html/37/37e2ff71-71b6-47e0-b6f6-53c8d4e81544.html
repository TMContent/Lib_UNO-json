Applies to

  • PHP

What to Check For

Ensure that the account administration interface is separate from the main application interface.

Why

Account administration requires higher privileges than the main application interface. Separating the two interfaces decreases the risk of a normal user being able to escalate privileges to that of an administrator.

How to Check

Use the following steps to verify that the account administration interface is separate from the main application interface:

  1. Verify that your application has a separate administrative interface. Review your application's design and verify that your application's design specifies separation of administrative and main application interfaces. Verify that all administrative functionality is clearly identified and separated from the rest of your application's functionality. Exceptions are allowed only when your application allows users to modify their own accounts.

  2. Verify that account administration code is organized. Ensure all administrative functionality is structured as a single module and not scattered throughout your application.

  3. Verify that re-authentication is required for administrative functions. Verify that privileged users are forced to re-authenticate before accessing the administrative interface. Check if your application's administrative interface uses a separate authentication module from the rest of your application.

  4. Verify that appropriate access controls are used. Verify that only valid, privileged users can access your application's administrative interface.

  5. Verify that TLS is used. Check if your application's administrative interface is utilizing TLS.

How to Fix

Use the following steps when designing an administrative interface:

  1. Only privileged users can administer all accounts. Design your application such that the administrative module is separate from the main application interface. Since the separation requires using separate authentication mechanisms for the application and administration, there is a reduced risk of escalation-of-privilege attacks.

  2. Users may be allowed to administer their own accounts. It may be necessary to allow users to administer their own accounts. This involves the ability to change passwords, account details, etc. Such administration can be done through the main application interface. Because the user is modifying sensitive data, extra security precautions must be enforced:

    • Force re-authentication. Enforce re-authentication when modifying the user's password. This helps verify that the user is aware of the password change and is not a victim of a session hijacking.

    • Modify accounts in sections. Do not modify all account information at once. Split the account modification procedure in sections. For example, group the account information into credentials, personal data, contact information and additional information. Modifying the account information in groups reduces the risk of information leakage when the user is updating his/her account.

    • Use TLS when modifying accounts. The use of TLS provides encryption when transmitting sensitive data over the network. TLS also certifies the authenticity of the application against the user.
