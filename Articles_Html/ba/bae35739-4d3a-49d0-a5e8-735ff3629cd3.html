Applies To

  • Web Applications

What to Check For

Verify that access control permissions are checked whenever performing direct object references.

Why

Using direct object references may expose sensitive data to attackers. Direct object references often expose sensitive data, such as files, database entries, or personal information. Direct object reference attacks are very simple to execute and have been used in high profile, low-skill attacks. Checking access control permissions whenever referencing objects helps prevent direct object reference vulnerabilities.

How to Check

To verify that access control permissions are checked whenever referencing objects directly:

  1. Identify pages that use direct object references. Make a list of all pages in the application. Go through the list and identify all pages that use direct object references by examining each page.
  2. Identify code that references objects. For each page that uses direct object references, identify the code that references objects.
  3. Verify that access controls are checked. Examine code that performs direct object references to make sure that it performs appropriate access control checks.

How to Fix

To check access control permissions whenever performing direct object references:

  1. Identify pages that use direct object references. Make a list of all pages in the application. Go through the list and identify all pages that use indirect object references by examining each page.
  2. Identify code that references objects. For each page that uses indirect object references, identify the code that references objects.
  3. Check access controls. Add code that performs access control checks to code that performs object references.
