Applies to

  • PCI DSS Requirement 1.3.8.a
  • PCI DSS Requirement 1.3.8.b

What to Check For

Verify that the company network is using only RFC 1918 addresses, which are not routable from the Internet. Ensure that the internal IP addresses are hidden from the Internet by using Network Address Translation (NAT).

Why

Leaking IP addresses from the local network to the Internet may provide information to potential attackers that could help them carry out an attack.

How to Check

Check the router and firewall configurations to make sure that NAT is being used. Check that all computers on all internal networks, including the DMZ, are assigned only RFC 1918 IP addresses. Connect to an Internet web site that displays the IP address of the browser from the DMZ. Make sure the web site shows the external IP address of the Internet router or firewall to verify that NAT is working.

How to Fix

Assign RFC 1918 IP addresses to all network devices on all internal networks. Configure the Internet firewall or router to use NAT for Internet access to the DMZ. Configure port forwarding on the Internet firewall or router to handle incoming connections from the Internet. Computers in the cardholder data segment should not have Internet access.
