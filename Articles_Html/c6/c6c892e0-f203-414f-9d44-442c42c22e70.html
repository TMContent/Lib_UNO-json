Applies to

  • PCI DSS Requirement 4.1
  • PCI DSS Requirement 4.1.g

What to Check For

Check that HTTPS appears at the beginning of the URL in all pages that require cardholder data.

Why

HTTPS in the URL tells the user that the session is encrypted with TLS. If frames are used in a web application, then the URL may not accurately reflect whether TLS is used.

How to Check

Make a list of all parts of the web application that require cardholder data. Check each part to make sure HTTPS appears in the beginning of the URL when using those parts of the web application.

How to Fix

Use TLS to encrypt cardholder data sent to the web application. Install the TLS implementation that applies to the web server that you are using. Configure the web server so that TLS is used for accessing the web application. If TLS is not used for all parts of the web application, configure the server so that the parts of the web application that require cardholder data use TLS. If frames are used in the web application, place the page with the frame-set in a location that is accessible only through TLS-encrypted HTTP.
