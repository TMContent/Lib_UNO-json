Applies to

  • PHP

What to Check For

Verify that the "secure" flag is set on session cookies.

Why

Set the "secure" flag on session cookies to protect the contents of the session cookies from eavesdropping attacks. Setting the "secure" flag on session cookies ensures they are only transferred over connections that are secured by TLS.

How to Check

To verify that the "secure" flag is set on session cookies:

  1. Log in to the application. Open the application and authenticate. Navigate to pages that use sessions to make sure that a session is created.

  2. Verify that session cookies have the "secure" flag set. Use the web browser to view stored cookies. Make sure that the application session ID cookie has the "secure" flag set.

  3. Check the php.ini file. If testing with a browser is not appropriate, check the php.ini file to make sure the "session.cookie_secure" directive is set to "On".

How to Fix

To set the "secure" flag on session cookies:

  1. Verify that TLS is available. The "secure" flag forces cookies to be sent over TLS. TLS has to be available for this to work.

  2. Configure PHP to set the "secure" flag on session cookies. Edit the php.ini file and configure the "session.cookie_secure" setting, like this:

    session.cookie_secure = On
