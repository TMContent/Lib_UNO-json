Applies to

  • PCI DSS Requirement 4.1
  • PCI DSS Requirement 4.1.a

What to Check For

Strong cryptography should be used whenever cardholder data is being transmitted over open, public networks.

Why

Encrypt the cardholder data when transmitting to prevent it from being read if intercepted by an attacker.

How to Check

Identify if and how the cardholder data is transmitted over open, public networks, such as the Internet. Identify all the computers that are involved in the transmission. Make sure that a strong encryption solution is used on each involved computer to secure the transmission. Some acceptable network encryption solutions are IPsec and TLS.

How to Fix

Identify if and how the cardholder data is transmitted over open, public networks, such as the Internet. Identify all the computers that are involved in the transmission. Choose a strong encryption solution to secure the transmission. Some acceptable encryption solutions are IPsec and TLS:

  • IPsec is best used with VPN. IPSec is only an option when all hosts involved have static IPs.
  • TLS is best used for transmitting data between a web client and a web server.

Install and configure a strong encryption solution on each involved computer to secure the transmission.
