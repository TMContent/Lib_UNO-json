Applies to

  • Microsoft® Windows Communication Foundation (WCF) 3.5
  • Microsoft Visual Studio® 2008

What to Do

If you host your service in a Windows Service, expose a Metadata Exchange (mex) Binding.

How

If you are hosting your service as a Windows service and are exposing the service using the netTcpBinding, publish the service metadata by creating a mexTcpBinding endpoint so that your clients can discover and use the service. Clients will be able to generate a proxy file using the ServiceModel Metadata Utility Tool (Svcutil.exe).
