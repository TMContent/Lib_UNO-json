What to Do

Closely monitor the use of privileged accounts so that suspicious activity can be detected promptly.

Why

Privileged accounts by their nature are allowed to do potentially damaging activities. If compromised, they can cause the most damage. Close monitoring can catch unauthorized and/or malicious activities and help minimize the damage done.

When

If your application supports privileged accounts that enable potentially dangerous activities, ensure they can be easily monitored for malicious activity.

How

To monitor privileged accounts, use the following steps:

  1. Identify the privileged users. For the purposes of security monitoring, privileged accounts are those that can cause more damage than other users. This will vary by application and roles defined, however some examples are: Administrator, Power user, Team leader

  2. Ensure there are auditing mechanisms in place. Place logging mechanisms on functionality that is unique to privileged users. This will ensure there is a record of administrative activities by privileged users. Examples of such areas include: Project management, User management, Global application management.

Problem Example

MyApp is a multi-user financial application. The application records all basic security related operations for all users. Unfortunately, it fails to monitor all authorizations that accompany the transactions of large financial assets, an action for which a privileged account is needed. Therefore it is impossible to trace such financial fraud to the responsible user.

Solution Example

MyApp is a multi-user financial application. The application records all basic security related operations for all users and monitors closely the activities that require a privileged account. If there is a financial fraud involving the transaction of large financial assets, the application can identify the user that authorized the transaction.
