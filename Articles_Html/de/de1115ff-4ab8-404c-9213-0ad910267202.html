Applies To

  • ASP.NET 4.0

What to Do

Reduce application fingerprints by removing as many uniquely identifiable strings as possible from the public areas of your application.

Why

Uniquely identifiable strings, such as application name and version number, are telltale signs that the application is running on your system. These telltale signs, or fingerprints, are often searched for via automated scripts that look for a target to attack.

Removing uniquely identifiable strings from the public areas of your application makes it more difficult for attackersâ€™ automated scripts to find and attack your application.

(It is impractical to attempt to eliminate the possibility of fingerprinting altogether, because the effort required to do would almost certainly outweigh the benefits.)

How

To remove uniquely identifiable strings from the public areas of your application:

  1. Identify public areas of the application. Identify all portions of the application that send information of any kind to the user without requiring authentication. This includes the login page, the forgot password page, any kind of public content pages, help pages, contact pages, URL strings, error pages, and particularly banners that include the application name and version. Make a spreadsheet about all of these sources of information made publically available by the application.
  2. Consider locking down public areas. Go through the spreadsheet of sources of information disclosed to the public by the application. Consider whether it is possible to lock down any of these sources by requiring authentication before the information is disclosed.
  3. Find uniquely identifiable strings. In the code, examine each source of information disclosed by the application. Identify strings that can be used to uniquely identify your application. Add this information to the spreadsheet.
  4. Remove or replace uniquely identifiable strings. For each string that can be used to uniquely identify the application, consider whether this string can be removed completely or replaced by a more ambiguous string. Remove or replace every such string, if possible.
