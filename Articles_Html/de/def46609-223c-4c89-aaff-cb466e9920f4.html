Applies to

  • PCI DSS Requirement 11.2

What to Check For

Internal and external vulnerability scans should be performed at least quarterly. Internal vulnerability scans should be performed after any significant changes to the network.

Why

Automated vulnerability scans may detect security holes and misconfigurations with relatively little effort.

How to Check

Use the following steps to make sure vulnerability scans are performed regularly and in compliance with PCI DSS:

  1. Review policy. Examine the information security policy to verify that it includes the following requirements:

    • A requirement for internal vulnerability scans to be performed at least quarterly and after any significant changes to the network
    • A requirement for external vulnerability scans to be performed at least quarterly
    • A requirement for quarterly external vulnerability scan results to satisfy PCI Security Scanning Proceduresquarterly external vulnerability scans to be performed by an Approved Scanning Vendor (ASV)
    • A requirement for all found vulnerabilities to be fixed
    • A requirement for scans to be repeated until no more vulnerabilities are found

  2. Review documentation. Examine the results of previous vulnerability scans to verify that they have been performed regularly and that rescans have been performed until passing results were obtained.

How to Fix

Perform the following tasks to assure that vulnerability scans are performed regularly and in compliance with PCI DSS:

  1. Update policy. Add the following requirements to the information security policy:
    • A requirement to perform an internal vulnerability scan at least quarterly and after any significant changes to the network
    • A requirement to perform an external vulnerability scan at least quarterly
    • A requirement to have the quarterly external vulnerability scan results satisfy PCI Security Scanning Procedures
    • A requirement to have the quarterly external vulnerability scans performed by an Approved Scanning Vendor (ASV)
    • A requirement to to fix all discovered vulnerabilities
    • A requirement to to repeat the scan until no more vulnerabilities are found
  2. Perform scans. Perform vulnerability scans regularly, document them, and attach the results to the security documentation.
