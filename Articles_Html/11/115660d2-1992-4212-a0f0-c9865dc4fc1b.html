Applies to

  • PCI DSS Requirement 6.6

What to Do

Deploy a web application firewall in front of a public-facing web application.

Why

A web application firewall may prevent an attacker from successfully exploiting a vulnerability in a public-facing web application. It is not necessary to install a web application firewall to achieve PCI DSS compliance if all public-facing web applications are reviewed at least annually.

How

Use the following steps to deploy a web application firewall in compliance with PCI DSS:

  1. Choose a WAF. Decide whether to conduct regular reviews of the web application or to install a web application firewall. If you decide on a web application firewall, choose a web application firewall based on compatibility with your log server, availability of vendor support, and considerations listed on the OWASP Web Application Firewall page

  2. Deploy the WAF. Install the web application firewall in front of the public-facing web application. Please refer to the vendor's documentation for detailed instructions on how to install and configure your web application firewall (WAF) of choice. Often, the installation process simply means wiring the physical firewall in front of the web application and configuring some basic TCP/IP settings.

  3. Configure the WAF. The configuration process and available options vary by vendor. Please refer to your vendor's documentation or technical support for detailed instructions on setting up your web application firewall.

Additional Resources

  • For more information about web application firewalls and a list of web application firewall vendors, please see the OWASP "Web Application Firewall" page at http://www.owasp.org/index.php/Web_Application_Firewall.
