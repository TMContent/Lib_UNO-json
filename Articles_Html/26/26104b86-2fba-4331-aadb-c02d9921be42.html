Applies To

All web applications.

Description

In a forceful browsing attack, the attacker gains access to a restricted page within a web application by supplying a URL directly -- i.e., forcing the URL -- rather than by following links to access it from other pages in the application. The intended workflow is to get to the restricted page through another page which authorizes the user to access the target page. A forceful browsing attack also allows the attacker to gain access to resources to which no direct links exist.

Impact

  • Unauthorized access to web site functionality.

Vulnerabilities

  • Poor authorization control.
  • Missing authentication on pages.

Countermeasures

  • Every object needs to have an authorization control that authorizes access based on the identity of the authenticated principle requesting access.
  • Access to all sensitive pages should require a valid authentication ticket.
  • Pages which are intended to be part of a workflow should enforce that workflow.
