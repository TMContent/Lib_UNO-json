Applies to

Scala and Play Framework 2.1.0+

What to Check For

Verify that inactive sessions time-out.

Why

Make sessions expire after a period of inactivity to make it harder for an attacker to hijack a session.

How to Check

To verify that inactive sessions time-out:

  1. Identify the time-out period. Examine project documentation to identify the required time-out period. If the inactive time-out period is configurable, find out the currently defined time-out period from the administrative interface.

  2. Login to the application. Login to the application as a normal user.

  3. Verify that the session times out after a certain period. Wait for the duration of the defined time-out period. Verify that the session has timed out.

The reason for such a high-level approach is because there are multiple ways to make inactive sessions time-out and it would be difficult to look into all the locations seperately.

How to Fix

To make inactive sessions time-out:

  1. Identify login functionality. Find code that handles user logins.

  2. Add code that sets the inactive session time-out period. There is no technical timeout for the Session. It expires when the user closes the web browser. 

    If you need a functional timeout for a specific application, just store a timestamp into the user Session and use it however your application needs (e.g. for a maximum session duration, maxmimum inactivity duration, etc.). This link explains how to add session timeout functionality to the application.
