Applies To

  • ASP.NET 4.0

What to Check For

Verify that a new session ID is assigned on login.

Why

Assigning a new session ID on login makes it harder for an attacker to hijack a user's session.

How To Check

To verify that a new session ID is assigned on login:

  1. Identify login functionality. Find code that is responsible for performing user authentication, specifically logging in.

  2. Verify that session IDs are strong. A SessionIDManager object should be used to manage session IDs. Verify that the CreateSessionID method is used to generate cryptographically strong session IDs.

  3. Verify that strong session IDs are assigned to cookies. Find code that assigns the session ID to the session ID cookie by searching for ASP.NET_SessionId. ASP.NET_SessionId is the default name of the session ID cookie. Verify that the unique session ID is assigned to the session ID cookie with code such as the following:

    Response.Cookies.Add(new HttpCookie("ASP.NET_SessionId", StrongSessionID));

How To Fix

To assign a new session ID on login:

  1. Identify login functionality. Find code that is responsible for performing user authentication, specifically logging in.

  2. Generate a strong session ID. Use a SessionIDManager object to manage session IDs. Call the CreateSessionID method to generate a unique, strong session ID for the current session.

  3. Create the session ID cookie. Create a cookie called ASP.NET_SessionId and assign the cryptographically strong session ID to it. This can be done by using code similar to this:

    Response.Cookies.Add(new HttpCookie("ASP.NET_SessionId", StrongSessionID));
