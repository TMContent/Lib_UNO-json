Applies To

  • Web Applications

What to Check For

Verify that TLS is used.

Why

TLS should be used to protect all pages that handle sensitive data to protect from eavesdropping attacks.

How to Check

To verify that TLS is used:

  1. Browse areas on the web site that require authentication. Verify that the browser displays that TLS is being used.
  2. Examine the server configuration files. Review the web application server configuration files to make sure that TLS is properly configured and is being used.

How to Fix

The exact instructions for enabling TLS are different for different servers. Enabling TLS is a common task and is well documented for common web server platforms. Links at the bottom of this page lead to documentation that explains how to enable TLS on different platforms. There are some general guidelines that apply to different platforms:

  1. Get valid certificates. Purchase a valid TLS certificate. You may purchase a valid TLS key from VeriSign, Thawte, or your web hosting provider. Follow the vendor's instructions for purchasing the TLS certificate. The certificate is usually a .cer file.
  2. Identify sensitive pages. Make a list of all pages that handle sensitive information, including all pages that use authentication or authorization features.
  3. Enable TLS on the web server. The exact instructions for enabling TLS are different for different web servers. Please, see the links at the bottom of this page for more information about enabling TLS on some common web application platforms.
  4. Use TLS to protect the sensitive pages. The exact instructions for doing so are different for different web servers. Use the web server administration interface or edit the server configuration files to enable TLS for all folders containing sensitive pages.
  5. Redirect non-TLS requests to TLS-protected pages. The exact instructions for doing so are different for different web servers. Use the web server administration interface or edit the server configuration files to redirect non-TLS requests to TLS-protected pages.
