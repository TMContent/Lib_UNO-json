Applies to

  • PCI DSS Requirement 1.1.2.a
  • PCI DSS Requirement 1.1.2.b
  • PCI DSS Requirement 1.1.3

What to Do

Draw a network diagram to aid IT and Information Security tasks.

Why

The network diagram is useful for IT and Information Security tasks. The network diagram can help with identifying potential vulnerabilities in the network design, planning changes to the network, and performing routine maintenance tasks. The network diagram is required to pass the PCI DSS compliance certification process.

When

Draw a network diagram when preparing the network documentation. It may be helpful to draw a network diagram before deploying the network.

How

Perform the following actions to draw a network diagram and to keep it up-to-date:

  1. Choose diagramming software. To draw a network map, diagramming software is recommended. The choice of diagramming software may be left to the person responsible for the network diagram. One software capable of this task is MS Visio.

  2. Diagram the servers and workstations. Identify all network nodes that handle cardholder data. Draw the network nodes that handle cardholder data on the network map. You might want to add notes that explain the roles of specific network nodes in handling cardholder data. These notes should be located next to the symbolic representations of the respective network nodes.

  3. Diagram the routers and firewalls. Draw all the routers and firewalls that are connected to the network nodes that handle cardholder data. Draw the connections between the routers, the firewalls and the network nodes that handle cardholder data. Routers and firewalls are network nodes, but for the purposes of this article, we will use the term "network nodes" to refer to devices that have a purpose other than operations on the network layer.

  4. Diagram the wireless networks. Draw all wireless networks that may be routable to the cardholder data environment.

  5. Diagram the Internet connections. Identify all Internet connections. Draw all Internet connections on the network map. Draw the relationships of each Internet connection to the routers and firewalls.

  6. Diagram the flow of cardholder data. Identify the flow of cardholder data. Make a copy of the network diagram and add notes to this copy that explain the flow of the cardholder data. Usually these notes should be located next to the links between the network devices.

  7. Delegate responsibility. Designate a qualified person to prepare the network diagram and to keep it up-to-date.
