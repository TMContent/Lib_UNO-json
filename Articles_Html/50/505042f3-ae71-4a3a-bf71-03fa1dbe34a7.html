Applies To

  • ASP.NET 4.0

What to Check For

Make sure that ASP.NET session management is used.

Why

Use ASP.NET session management, because it is well implemented and fully functional. Using it prevents potential vulnerabilities introduced in a custom session management solution.

How To Check

To verify that ASP.NET session management is used:

  1. Verify that ASP.NET session state is enabled. Examine&nbsp;the&nbsp;Web.config file to make sure that the sessionState element exists and is properly configured. Search for the string "sessionState".

  2. Identify all pages that use session management. Make a list of all pages that use session management functionality.

  3. Verify that ASP.NET session state is used. Review code to make sure it uses ASP.NET session state functionality by searching for the "Session" keyword. The "Session" property of the page exposes ASP.NET session state functionality.

How To Fix

To use ASP.NET session management:

  1. Configure ASP.NET session state. Configure the sessionState element in the system.web session of the Web.config file. The exact settings depend on your application and a lot depends on the settings you choose here. Please see the related article in the "Additional Resources" section for more information about configuring the sessionState element.

  2. Generate a strong session ID. Use a SessionIDManager object to manage session IDs. Call the CreateSessionID method to generate a unique, strong session ID for the current session.

  3. Create the session ID cookie. Create a cookie called ASP.NET_SessionId and assign the cryptographically strong session ID to it. This can be done by using code similar to this:

    Response.Cookies.Add(new HttpCookie("ASP.NET_SessionId", StrongSessionID));

  4. Manage the user session. Each ASP.NET page can access session variables through the Session property of the page. For more information about using session state, please see the relevant article in the "Additional Resources" section.
