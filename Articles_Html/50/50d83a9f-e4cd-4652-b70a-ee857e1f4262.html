Applies To

  • ASP.NET 4.0

What to Check For

Verify that all private resources require authentication.

Why

All pages that have private information should require authentication to help prevent unauthorized access.

How To Check

To verify that all private resources require authentication:

  1. Identify all sensitive pages. Make a list of all pages that handle sensitive or private information.

  2. Review Web.config files. Check Web.config files to make sure that all sensitive pages have an applicable and properly configured <authorization> section.

How To Fix

To require authentication for all private resources:

  1. Identify all sensitive pages. Make a list of all pages that handle sensitive or private information.

  2. Define roles. Define access roles for your application. These roles should include administrative users and various types of users specific to your application.

  3. Assign role-based access. Edit the Web.config file and configure the following markup for each sensitive page:

    <location path="[Filename of the page]">
       <system.web>
          <authorization>
             <allow roles="[Authorized roles]">
             <deny users="*">
          </authorization>
       </system.web>
    </location>

    The [Filename of the page] refers to the file name of the page. The [Authorized roles] is a comma-separated list of roles authorized to view the resource. The line <deny users="*"> fulfills the "Deny All Access by Default" principle by rejecting all access besides that explicitly authorized via role-based authorization.
