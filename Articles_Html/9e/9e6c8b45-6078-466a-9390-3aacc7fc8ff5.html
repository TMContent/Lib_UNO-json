Applies to

Applications written using Scala and Play Framework 2.1.0 using Nginx as front-end in reverse proxy mode.

What to Check For

Ensure that your application has directory browsing disabled.

Why

Directory browsing allows an attacker to browse your application's directory structure in search of unprotected files and components.

How to Check

Use the following steps to ensure that your application has directory browsing disabled:

  1. Identify if directory browsing is enabled. Navigate to your application's directories. Identify if your application's server returns the directory contents and not an HTTP error message.

  2. Locate the default Nginx configuration. In Nginx, the default configuration file is located at /etc/nginx/sites-availabe/default. In the default "server" block, look for the presence of the "location" element that is mapped to the directory for which directory browsing is to be disabled. 

    server {
        listen   80;
        server_name  domain.com www.domain.com;
        access_log  /var/...........................;
        root   /path/to/root;
        location / {
                index  index.html index.htm;
        }
        location /somedir {
               autoindex on;
        }
}

  3. Ensure directory browsing is disabled in the default Nginx configuration. Verify that the HttpAutoindexModule module is initialized with "off"  to disable directory browsing.

     location /somedir {
               autoindex off;
        }

How to Fix

To disable directory browsing:

  1. Identify if directory browsing is enabled. Navigate to your application's directories. Identify whether your application's server correctly returns an HTTP error message. If it incorrectly returns the directory contents, continue with the following steps.

  2. Locate the default Nginx configuration. In Nginx, the default configuration file is located at /etc/nginx/sites-availabe/default. In the default "server" block, look for the presence of the "location" element that is mapped to the directory for which directory browsing is to disabled. Example:

    server {
        listen   80;
        server_name  domain.com www.domain.com;
        access_log  /var/...........................;
        root   /path/to/root;
        location / {
                index  index.html index.htm;
        }
        location /somedir {
               autoindex on;
        }
}

  3. Deny directory browsing in the default Nginx configuration. Initialize the HttpAutoindexModule with "off" to disable directory browsing. Example:

    server {
        listen   80;
        server_name  domain.com www.domain.com;
        access_log  /var/...........................;
        root   /path/to/root;
        location / {
                index  index.php index.html index.htm;
        }
        location /somedir {
               autoindex off;
        }
}

Problem Example

An application is deployed on a production server. Unfortunately, the developers did not disable directory browsing. This allows an attacker to search for unprotected configuration files.

Solution Example

An application is deployed on a production server. Because the developers have disabled directory browsing, it is difficult for an attacker to use HTTP browsing to search for unprotected configuration files.

Deployment Guidelines

  • Remove Test Accounts
  • Disable Directory Browsing
  • Do not store Plain text data

Deployment Checklists

  • Test Accounts Are Removed
  • Directory Browsing Is Disabled
  • Sensitive Data Inside Configuration Files Is Protected
