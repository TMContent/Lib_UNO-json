Applies to

Scala and Play Framework 2.1.0+

What to Do

Check access control permissions when referencing objects.

Why

Checking access control permissions when referencing objects helps prevent direct object reference vulnerabilities. Direct object references often expose sensitive data, such as files, database entries, or personal information. Direct object reference attacks are very simple to execute and have been used in high profile, low-skill attacks. 

When

Always check access control permissions when referencing objects.

How

To check access control permissions when referencing objects:

  1. Identify pages that use direct object references. Make a list of all pages in the application. Examine each page to identify the pages that use direct object references.

  2. Identify code that references objects. In each page that uses direct object references, identify the code that references objects.

  3. Check access controls. Add code that checks access controls to the code that performs object references.
