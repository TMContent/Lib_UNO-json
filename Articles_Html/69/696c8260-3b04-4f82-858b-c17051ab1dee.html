Applies to

  • PCI DSS Requirement 10.5.3
  • PCI DSS Requirement 10.5.4

What to Check For

Current logs should be backed up promptly to a centralized log server or a media that is difficult to alter.

Why

Logs should be backed up promptly to prevent tampering.

How to Check

Verify that logs are backed up promptly by performing the following tasks:

  1. Examine the centralized log server. Verify that a centralized log server is installed and all log-generating devices are sending logs to it. Most centralized log server solutions collect data from network devices in real time. Examine the logs on the log server to make sure that they are current.

  2. Examine devices that are not compatible with the centralized log server. Make a list of devices that are not compatible with the log server. Verify that there is some other mechanism implemented for quickly sending logs from these devices to a secure storage location by examining the logs at the back-up location to make sure they are current.

How to Fix

Use the following steps to assure that logs are backed up promptly:

  1. Deploy a centralized log server. Choose a centralized log server solution. When choosing a centralized log server solution, it may be helpful to verify that it is explicitly designed to satisfy PCI DSS Requirement 10.2.2 - logging all activity by the administrator and the other privileged users - because this functionality is difficult to implement otherwise. Install and configure a centralized log server by following the vendor's instructions. Configure all network devices that are compatible with the log server to send logs to the log server.

  2. Configure devices that are not compatible with the log server. Make a list of the devices that are not compatible with the log server. Use this list daily to manually check and backup the logs on the devices that are not compatible with the log server.Consider replacing the incompatible devices.
