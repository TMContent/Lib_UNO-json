Applies to

  • Microsoft&#174; Windows Communication Foundation (WCF) 3.5
  • Microsoft Visual Studio&#174; 2008

What to Do

If non-repudiation is important, consider setting the SuppressAuditFailure property to false.

Why

This setting will cause an exception to be thrown when there is any audit failure.

How

By default your WCF will ignore audit failures and allow the service to continue running. By setting SuppressAuditFailure to false an exception can be thrown and handled by your WCF service.&nbsp;&nbsp;

<configuration>
&nbsp; <system.serviceModel>
&nbsp;&nbsp;&nbsp; <behaviors>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <behavior>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <serviceSecurityAudit
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; auditLogLocation="Application"
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; suppressAuditFailure="false"
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; serviceAuthorizationAuditLevel="Failure"
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; messageAuthenticationAuditLevel=
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; "SuccessOrFailure" />
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </behavior>
&nbsp;&nbsp;&nbsp; </behaviors>
&nbsp; </system.serviceModel>
</configuration>
