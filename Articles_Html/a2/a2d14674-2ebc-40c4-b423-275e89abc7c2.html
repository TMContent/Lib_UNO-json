Applies to

  • PCI DSS Requirement 2.1.1.b

What to Check For

The default SNMP community strings should be changed from their vendor defaults on wireless access points.

Why

SNMP community strings act as passwords for SNMP-enabled devices, and the default settings can be found by potential attackers on the Internet. Changing SNMP community strings on wireless access points prevents a potential attacker from changing wireless settings or discovering sensitive network information.

How to Check

Inspect the wireless access points to&nbsp;check if&nbsp;they have SNMP enabled. If the wireless access points support SNMP, examine the configuration on each access point to make sure that the SNMP community string is changed from the vendor default. Use the Internet or the vendor's documentation to find out what the default SNMP community strings are and make sure that the ones in the current configuration are different.

How to Fix

Change the SNMP community string on each wireless access point to something other than the default. The SNMP community string is essentially a password, so the password policy should be followed when choosing a new SNMP community string. The exact steps required to change the SNMP community string vary by hardware. Please refer to the vendor's documentation or contact tech support for instructions on changing SNMP community strings on your wireless access points.
