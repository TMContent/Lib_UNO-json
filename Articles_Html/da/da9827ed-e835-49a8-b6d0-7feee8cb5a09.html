Applies to

  • PHP

What to Check For

Verify that strong session IDs are used by checking the PHP configuration.

Why

Making session IDs cryptographically stronger makes them more difficult to predict and to hijack user sessions.

How to Check

To verify that strong session IDs are used:

  1. Check the "session.entropy_file" setting. Check the php.ini file to make sure that session.entropy_file is set to /dev/urandom. This setting tells PHP to use the operating system random number generator for additional entropy on UNIX-style and Windows systems. While there is no /dev/urandom file on Windows systems, setting session.entropy_file to any non-null value tells PHP to use the Windows random number generator for additional entropy.

  2. Check the "session.entropy_length" setting. Check the php.ini file to make sure that session.entropy_length is set to 512 or more.

How to Fix

To configure PHP to use strong session IDs, configure the following settings in php.ini:

session.entropy_file = "/dev/urandom"
session.entropy_length = 512

Configuring the settings above tells PHP to use more entropy for generating session IDs, thus making session IDs cryptographically stronger.
