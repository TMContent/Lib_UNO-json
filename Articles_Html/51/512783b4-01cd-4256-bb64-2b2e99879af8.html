Applies To

  • PHP

What to Check For

Verify that encryption keys are protected.

Why

Protect the encryption keys to prevent your encrypted data from being compromised. If the encryption keys are stored together with the data, then an attacker may be able to get the keys together with the data and then use them to decrypt the data. Therefore, keeping the keys together with the encrypted data defeats the purpose of encrypting the data.

There is no practical encryption key management method available natively in PHP. The only practical way to implement key management in PHP in a manner that will satisfy regulatory requirements is to use a third-party solution that has been found acceptable by various regulatory agencies.

How to Check

To verify that encryption keys are protected:

  1. Identify key management requirements. Determine the key management requirements for your application. These might be defined by the application design and/or by compliance requirements.
  2. Identify the mechanism that is used to protect the encryption keys. Find code that is responsible for key management in your application.
  3. Verify that encryption keys are protected. Review code that is responsible for key management to make sure that it matches the requirements defined for the application.

How to Fix

To use a trusted third-party for key management in PHP:

  1. Identify a trusted third-party key management solution. There are very few options available. One option is linked to at the bottom of this page.
  2. Use the third-party key management solution. Work together with the vendor of the third-party key management solution to use it in your application.
