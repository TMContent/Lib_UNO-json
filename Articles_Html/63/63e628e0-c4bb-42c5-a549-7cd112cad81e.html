What to Do

Enforce strong passwords when the application connects to a backend database server.

Why

Enforcing strong passwords minimizes the risk of a dictionary attack against accounts on the database server.

When

If your application uses a backend database server.

How

When possible, avoid using a password at all, and authenticate to the database using Windows authentication.

If SQL Authentication must be used, use these guidelines to construct a password policy:

  1. Define password requirements. Pick the schema that all passwords must follow and check them for length and certain characteristics. Some commonly agreed upon characteristics for strong passwords are:

    1. Should be atleast 8 characters in length
    2. Should include at least one numeric and one special character
    3. Should be changed every 90 days

  2. Enforce password requirements. The password requirements should be enforced while creating and modifying accounts.

Problem Example

MyApp is a multi-user financial tool. Unfortunately, it does not enforce strong passwords for its database access. The administrator, tired of having to deal with a long password for the database server, sets it to a simple password and doesn't change it. The database server, although protected against outside attack, isn't properly segmented from the internal network, and an inside attacker discovers it and starts trying to crack the password. The Credentials Brute Force Attack succeeds, allowing the attacker access to the database server and the confidential data stored on it. Because the password was simply guessed, there was little other evidence of an attack, and it was not detected for some time.

Solution Example

MyApp is a multi-user financial tool. Because it enforces strong passwords for its database access, no account on the SQL Server can have a weak password. Although the administrator wants to set a shorter password, the system won't accept it. Even though the server isn't properly segmented from the internal network, the inside attacker who discovers it is unable to compromise the server, as she cannot guess the password.
