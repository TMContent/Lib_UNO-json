Applies to

  • PCI DSS Requirement 8.5

What to Do

Use Group Policy Objects to implement the following password policies in a Microsoft Active Directory environment:

  • Passwords Expire After 90 Days
  • Password Length
  • Password Complexity
  • Password History
  • Account Lockout Threshold
  • Account Lockout Duration
  • Session Time-out

Why

Microsoft Active Directory is one of the most cost-effective ways of implementing the PCI DSS password requirements.

How

The password policy aspects of PCI DSS are best implemented using Microsoft Active Directory. Microsoft Active Directory allows implementing policy using Group Policy Objects.&nbsp;Follow these steps to configure a PCI DSS compliant password policy using Group Policy Objects in an Active Directory environment:

  1. Launch the "Group Policy Management" console. The "Group Policy Management" console is included in Windows 2008, but has to be downloaded and installed on Windows 2003 and 2000. The link to the download is available at the bottom of this page. To launch the "Group Policy Management" console, execute the "gpmc.msc" command.

  2. Edit or create a Group Policy Object (GPO). To create and activate a new Group Policy Object (GPO), launch "The Group Policy Management" console, right click on the domain and select "Create and Link a GPO Here." To edit an existing GPO, navigate to the "Group Policy Objects" folder under the current domain, right click on the GPO and select "Edit" - this will open the "Group Policy Object Editor." To change the settings of a GPO using the "Group Policy Object Editor, navigate the folders to find the setting of interest, double click on it, set the desired option(s).

  3. Implement&nbsp;password strength&nbsp;policies using Group Policy Objects. The default GPO that includes these settings is called "Default Domain Policy." Go to "Windows Settings", "Security Settings", "Account Policies", "Password Policy".

    • Under “Maximum password age," set the "Maximum password age" to 90 days or fewer.&nbsp;The "Maximum password age" policy is set by default to 42 days on Windows 2003 and 2008.
    • Under "Password must meet complexity requirements," set the "Minimum password length" policy requirement to 7 characters or more. The "Minimum password length" policy is enabled by default on Windows 2003 and 2008 and is configured to require passwords to be at least 7 characters long. Set the "Password must meet complexity requirements" setting to "Enabled".&nbsp;The "Password must meet complexity requirements" policy is enabled by default on Windows 2003 and 2008 and is configured to require both letters and numbers in passwords.
    • Under “Enforce password history,” set the "Enforce password history" policy to remember at least the last four passwords. Password history is enabled by default on Windows 2003 and 2008 and is configured to remember the last 24 passwords.

  4. Implement account lockout policies. For&nbsp;Account Lockout Threshold and Account Lockout Duration go to "Windows Settings", "Security Settings", "Account Policies", “Account Lockout".

    • Under "Account lockout threshold," set the "Account lockout threshold" setting to no more than 6 failed attempts. The "Account lockout threshold" is not enabled by default on Windows 2003 and 2008.
    • Under “Account lockout duration," set the "Account lockout duration" setting to 30 minutes or more. The "Account lockout duration" is not enabled by default on Windows 2003 and 2008.

  5. Implement idle terminal locking policy. For Session Time-out create a new GPO or edit an existing one to configure these policies, which are located under "User Configuration", "Administrative Templates", "Control Panel", "Display". Set "Password protect the screen saver" policy to "Enabled". Set&nbsp;"Screen Saver timeout" to 15 minutes or less. The "Password protect the screen saver" and the "Screen Saver timeout" policies are not enabled by default on Windows 2003 and 2008.
