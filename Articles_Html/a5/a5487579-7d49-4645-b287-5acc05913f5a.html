What to Do

Define policies for the acceptable use of technology.

How

Use the following steps to define the acceptable use policy:

  1. Add a requirement for management approval. Add a point that requires explicit management authorization to use the technologies.

  2. Add a requirement for authentication. Add a point that requires users to be authenticated with a password or some other security mechanism to use the technologies.

  3. Add a requirement for an&nbsp;authorized devices&nbsp;list. Add a point that requires a list of all devices and the staff authorized to use them.

  4. Add a requirement for labeling. Add a point that requires devices to be labeled with owner, contact information and purpose.

  5. Add a requirement for acceptable use. Add a point that requires acceptable uses for the technology. This requirement seems redundant, but it is actually explicit in demanding that technology be used only in its intended purpose.

  6. Add a requirement for network locations. Add a point that requires acceptable network locations for the technology. Acceptable network locations are the networks secured in compliance with PCI DSS. For example, a computer from the cardholder data environment may not be connected to the Internet directly.

  7. Add a requirement for a list of approved products. Add a point that requires a list of company-approved products. Make a list of company-approved products and add it to the information security documentation.

  8. Add a requirement for remote-access time-out. Add a point that requires idle remote sessions to be disconnected automatically after no more than 15 minutes.

  9. Add a requirement for managing vendor remote-access. Add a point that requires remote access accounts for vendors to be enabled only when needed and disabled immediately thereafter.

  10. Add a prohibition of remote-access downloads of sensitive data. Add a point that forbids downloading cardholder data to local storage media over a remote access connection.
