Applies to

  • PCI DSS Requirement 6.1

What to Do

Use on-line resources to stay informed about the latest security vulnerabilities.

Why

The information security community is very active online and provides many valuable resources that track the latest discovered vulnerabilities. Potential attackers monitor these resources as well, so it is important to mitigate new threats before they are exploited.

When

The responsible staff should be monitoring online resources to stay informed about the latest threats continually.

How

Perform the following actions to keep track of new security vulnerabilities:

  1. Update policy. Add a requirement to the information security policy for the responsible staff members to use outside resources to stay informed about the latest discovered security vulnerabilities. This requirement may be expressed along the lines of "The responsible staff shall regularly review online security vulnerability tracking resources to check whether the software on the company network is vulnerable. "Including a list of the online information security resources in the policy may be helpful. Recommended resources are: BugTraq, PacketStorm, SecurityFocus, Exploit-DB."

  2. Monitor information security web sites. Instruct the staff to review online security vulnerability tracking resources periodically to check whether the software you are using in your environment is susceptible to known vulnerabilities. Recommended web sites are: PacketStorm, SecurityFocus, Exploit-DBThe links to these resources are at the bottom of this article.

  3. Subscribe to information security mailing lists. Instruct at least one staff member to sign up for the BugTraq mailing list and other similar mailing lists as appropriate. The link to the BugTraq mailing list is at the bottom of this article

Additional Resources

  • To sign up for the BugTraq mailing list, please visit "Security Focus" at http://www.securityfocus.com.
  • For a list of recently discovered vulnerabilities, please see "Security Focus Vulnerabilities" at http://www.securityfocus.com/vulnerabilities.
  • To check the latest public exploits, please see "Last 100 Exploit Files" at http://www.packetstormsecurity.org/exploits100.html.
  • For an archive of exploits and vulnerabilities, please see “Exploit Database” at http://www.exploit-db.com/.
