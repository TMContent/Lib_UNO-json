Applies to

  • PCI DSS Requirement 2.2
  • PCI DSS Requirement 2.2.a
  • PCI DSS Requirement 2.2.c

What to Do

Prepare system configuration standards to use when setting up new systems.

Why

Company-wide system configuration standards make it possible to have the same secure settings across the entire company network.

How

Perform the following actions to prepare system configuration standards based on industry-accepted guides:

  1. Get industry-accepted guides that apply to your systems. There are a lot of hardening guides available in the public domain. Some options are:

    • CIS: The CIS hardening guides are high quality and very explicit. The CIS hardening guides are called "benchmarks" and are available for most common system types and many applications. The CIS hardening guides make an excellent foundation for the company's system configuration standards.

    • NSA SNACs: The NSA SNACs are high quality and very thorough but are not available for the latest system types. The available NSA SNACs may be used almost verbatim as the system configuration standards if they match the system types used by the company.

    • DISA STIGs: DISA STIGs are similar to NSA SNACs and are government system configuration guides. They are high quality, have lots of pictures, and are available for recent systems. DISA STIGs pick up where NSA SNACs left off. There are relatively few DISA STIGs overall. DISA STIGs may be used as a foundation for the system configuration standards.

    • SANS: The SANS configuration guides vary in quality and are poorly organized, and not all common platforms are covered by the SANS configuration guides. For example, there appears to be no comprehensive SANS Windows XP hardening guide. Some of the SANS hardening guides are high quality and very thorough. It is worth checking if the system types that you are using on your network have hardening guides available from SANS. If so, they may be used almost verbatim as a company system configuration standards consistent with PCI DSS requirements.

    • NIST: The NIST configuration guides are generally presented in SCAP format, which is an XML based format used by network vulnerability scanners. As such, NIST configuration guides are not very useful for preparing formal system configuration standards unless a SCAP-capable scanner is readily available. Even then, turning SCAP guidelines into formal system configuration standards means using the scanner in a backwards way to convert the SCAP instructions into human-readable format. This is doable and the actual instructions are pretty good. The University of Texas translated the SCAP guidelines for several Operating Systems into human-legible format. Even so, the NIST guidelines are very basic, so additional research is required to flesh them out. The real way to use NIST SCAPs is as add-ons to a network vulnerability scanner.

  2. Prepare hardening guides based on industry-accepted standards. Cut out pieces from them that you don't need. It may be helpful to mention what the original hardening guides were to point them out during PCI DSS compliance assessment. Put all the remaining guidelines together to form your company's system configuration standards.

  3. Apply the hardening guides. Configure your systems to match the hardening guides. Use the configuration guides when setting up new systems.

Additional Resources

  • For CIS hardening guides, please see "The Center for Internet Security" at http://cisecurity.org/en-us/?route=downloads.browse.category.benchmarks.
  • For NSA SNACs, please see "Security Configuration Guides" at http://www.nsa.gov/ia/guidance/security_configuration_guides/index.shtml
  • For DISA STIGs, please see "Security Technical Implementation Guides" at http://iase.disa.mil/stigs/stig/index.html.
  • For SANS White Papers, please see "SANS: Information Security Reading Room" at http://www.sans.org/reading_room/.
  • For NIST configuration guides, please see "National Checklist Program Repository" at http://web.nvd.nist.gov/view/ncp/repository.
