Applies To

  • Any

What to Do

Install the latest vendor-supplied security updates for all software and code libraries on the server at least once a month.

Why

Installing vendor-supplied security patches fixes known security vulnerabilities.

When

The latest vendor-supplied security patches should be installed within a month of being released.

How

Perform the following actions to install the latest security updates:

  1. Identify all software installed on the server(s). Create a spreadsheet of all software installed on each server. Record version information for each piece of software. If each server is different in purpose and configuration, it might be easiest to have a separate spreadsheet for each server. If there are multiple similar servers, it might be more effective to have a unified configuration standard and to maintain information about multiple servers in one spreadsheet. The important thing is to track all software installed on each server and to make sure that it is regularly updated.
  2. Identify all shared code libraries on the server. Create a spreadsheet of all code libraries installed on each server, including ones that are used by your application. Record version information for each library. If these libraries contain vulnerabilities, the applications using them will also be vulnerable. So, it is important to update libraries when updating the server, and to track the libraries in a manner similar to tracking installed applications. This can be tedious, especially on UNIX-type systems, but it is necessary. Fortunately, on UNIX-type systems the process can usually be at least partially automated using scripts.
  3. Identify sources of updates, and whether updates are manual or automated. For all the software and the libraries listed in your spreadsheets, record the distribution channel for security updates, and indicate whether the update is manual or automated. The distribution source will frequently be a web site from which patches can be downloaded. Some systems have automatic update features that can be used. Automated update systems are acceptable so long as they are reliable and provide information that allows you to verify that systems have been updated. Most common enterprise Linux distributions contain acceptable automatic update systems. Windows systems also can perform automated updates, but usually only update some system components. Most production systems will contain some components that need to be updated manually.
  4. Schedule regular updates. In your information security policy, add a requirement that critical new patches be installed at least once a month. Specify a date or time period when updates should be installed.
  5. Install new security patches at least once a month. Usually applying vendor-supplied patches means going to the vendor's web site, downloading the latest patches, and installing them, which is often a matter of running a packaged executable on the target host. The procedures for applying security updates vary by vendor, as vendors package their updates in different ways, such as compressed archives or source code patches. Please refer to your vendor's documentation or technical support for detailed instructions on installing their security updates. Note that a given update may include manual and automated portions. When the manual portion of the system update is performed, the results of the automated update should be verified to make sure that all the latest available patches have been installed by the automated update system.
  6. Track update and version information. In the software and library spreadsheets, record information about the results of the updates and the latest installed versions.
  7. (Highly recommended) Keep track of recently disclosed vulnerabilities. Monitor web sites that list recent exploits, vulnerabilities, and security updates. Some community web sites can provide the bulk of that information for common software products, but for software with small user bases, that information will usually come from the vendors. This type of information is very time-sensitive, so it needs to be checked at least once every three days. This can be a tedious and time-consuming process for someone that's not used to doing it already. This procedure must be carried out by someone who understands what they're looking at. If it is done right, this type of information can be very effective at preventing compromises. Conversely, the absence of such a process can lead to devastating results.
