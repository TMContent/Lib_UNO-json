Description

The software does not implement sufficient measures to prevent multiple failed authentication attempts within in a short time frame, making it more susceptible to brute force attacks.

Discussion

An often-used phrase is "If at first you don't succeed, try, try again." Attackers may try to break into your account by writing programs that repeatedly guess different passwords. Without some kind of protection against brute force techniques, the attack will eventually succeed. You don't have to be advanced to be persistent.

Mitigation

Common protection mechanisms are:

  1. Disconnect the user after a small number of failed attempts.

  2. Implement a timeout.

  3. Lock the targeted account.

  4. Require a computational task on the user's part. This technique is commonly known as CAPTCHA.

  5. Use a secure framework. Use a vetted library or framework that does not allow this weakness to occur or provides constructs that make this weakness easier to avoid. Consider using libraries with authentication capabilities such as OpenSSL or the ESAPI Authenticator.

Additional Information

  • For more information about improper restriction of excessive authentication attempts vulnerabilities, please see http://cwe.mitre.org/top25/index.html#CWE-307
