Applies to

  • PCI DSS Requirement 6.3

What to Do

Develop code in accordance with secure coding guidelines. Have code reviewed by someone else other than its author. Fix discovered vulnerabilities before the release. Ask management to approve code review results.

Why

Changes to application code should be reviewed to ensure that they do not introduce vulnerabilities.

How

Add a requirement to the information security policy that code changes should reviewed by someone other than their author(s). Add steps to the software development process for someone other than the code's author review the code. It is a good idea to have a dedicated testing team in the development environment. Add a requirement to the information security policy for code to be developed according to secure coding principles. Integrate security engineering techniques with your software development process. Information about security engineering techniques may be found in TeamMentor in the "Security Engineering" folder. Add a requirement to the information security policy for corrections to be implemented before release. Add steps to the software development process to fix any errors found in the code during review to be fixed before the release. Add a requirement to the information security policy for code review results to be approved by management before release. Add this requirement to the software development process.
