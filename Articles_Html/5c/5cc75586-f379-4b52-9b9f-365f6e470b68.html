What to Do

Encrypt off-site backups. Encrypt and store keys separately.

Why

Off-site backups should be encrypted to protect them if they are physically stolen. Encryption keys should be encrypted and stored separately to make sure that if the data backups are stolen, the attackers don't have the keys to go with them.

How

To protect off-site backups:

  1. Identify data to backup. Make a list of assets that should be backed up off-site.

  2. Choose an encryption mechanism. Choose a software suite to use for encrypting off-site backups.

  3. Establish key management. Define key management procedures for managing encryption keys used to protect off-site backups.

  4. Choose an off-site storage location for the data. Choose a safe location for off-site backups. One example is a safe deposit box

  5. Choose an off-site storage location for the keys. Choose a safe location for storing keys for off-site backups, separately from the backups themselves. One example is a safe deposit box. Make sure to store the encryption keys at a separate location from the encrypted backups.

  6. Encrypt and backup data. Establish a schedule for backing up data and encrypting the backups. Make sure that key management procedures are followed - keys are changed and backed up as necessary. Take the encrypted backups to the off-site storage location.

  7. Encrypt and backup encryption keys. Establish a schedule for backing up and storing the encryption keys used to encrypt off-site backups. Take the encryption keys to an off-site location separate from the backups themselves.
