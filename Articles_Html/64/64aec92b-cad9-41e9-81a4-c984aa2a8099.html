Applies to

  • PCI DSS Requirement 11.3

What to Check For

Penetration tests should be documented.

Why

Penetration tests should be documented to prove that they took place and to show what vulnerabilities may have already been found and fixed.

How to Check

Use the following steps to verify that penetration tests are performed and documented in compliance with PCI DSS:

  1. Verify the qualifications of the penetration testing team. Identify the penetration testing party and verify that their credentials qualify them for the task.

  2. Verify that a penetration test has been performed within a year. Check the results of the last penetration test in the security documentation to make sure that at least one has been performed in the past year.

  3. Confirm that penetration tests included network and application layer tests. Examine the results of the previous penetration test to make sure it includes network layer attacks, which include both operating system and network protocol attacks. Examine the results of the previous penetration test to make sure it includes application layer attacks. The application layer attacks may include attacks on client applications. For web applications, the penetration tests should include the attacks in the OWASP Top 10.

  4. Confirm that all discovered vulnerabilities have been fixed. Review documentation for information about discovered vulnerabilities and steps taken to eliminate them.

How to Fix

Perform the following actions to assure that penetration tests are performed and documented in compliance with PCI DSS:

  1. Choose a penetration testing team. Qualified professionals should perform penetration testing. It is acceptable to have the company staff perform the test if internal talent is readily available, but it is recommended to leave it to companies who specialize in security services. Security Innovation offers penetration-testing services.

  2. Perform penetration tests. Schedule yearly penetration tests. Perform network layer attacks, which include operating system and network protocol attacks. Perform application layer attacks. For web applications, have them tested for all the vulnerability types in the OWASP Top 10.

  3. Document fixed vulnerabilities. If vulnerabilities are found, fix them and perform another penetration test. Document all fixed vulnerabilities.

  4. Document the repeated tests. Continue testing until no more vulnerabilities are found. Document the results of each test.
