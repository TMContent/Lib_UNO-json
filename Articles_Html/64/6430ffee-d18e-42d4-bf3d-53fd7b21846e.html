What to Do

Avoid logging sensitive information.

Why

Log files are usually less protected and available to more users than sensitive data filaes. Including sensitive information in logs may lead to unauthorized information disclosure.

When

Use this guideline if your application handles sensitive data.

How

Perform the following actions to prevent logging of sensitive information:

  1. Identify sensitive data. Make a list of types of sensitive information handled by your application. Some examples are:

    • System information
    • Filenames and paths
    • Passwords
    • Encryption keys
    • Cardholder data
    • Personal information

  2. Identify logging functionality. Examine the application architecture to identify logging facilities, such as the logging API and where audit trails are actually stored. Make a list of APIs used to log events, document how to use them, and where they store logs.

  3. Review logging code. Identify code that generates log entries. Review this code to verify that it does not include sensitive information in log entries. If necessary, rewrite this code in a way that does not log sensitive data.

  4. Prevent logging of sensitive data. The most effective measure for avoiding logging of sensitive information is prevention, which means making sure that sensitive information does not get logged whenever writing code that generates log entries.

  5. Review logging of errors. Error logs often store sensitive data when they shouldn't. Pay particular attention to code that generates error logs, both when writing this code and when reviewing it, to make sure that sensitive information is not included in error messages or log entries.

Logging Guidelines

  • Centralize Logging
  • Protect Log Files
  • Log Important Event Details
  • Do Not Log Sensitive Information
  • Log Important Security Operations
  • Log Unusual Activity
  • Log Important Business Operations
  • Monitor the Use of Privileged Accounts
  • Log Authentication Events

Logging Checklists

  • Logging Is Centralized
  • Log Files Are Protected
  • Important Event Details Are Logged
  • Sensitive Information Is Not Logged
  • Important Security Operations Are Logged
  • Unusual Activity Is Logged
  • Important Business Operations Are Logged
  • The Use of Privileged Accounts Is Monitored
  • Authentication Events Are Logged
