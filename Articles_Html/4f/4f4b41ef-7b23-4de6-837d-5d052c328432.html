Applies to

  • PCI DSS Requirement 12.5

What to Do

Define daily information security operational procedures.

Why

Daily procedures should be clearly defined so that people can carry them out consistently and in a timely fashion.

How

Define daily operational security procedures and include the documentation in the information security documentation. The exact definitions and procedures will be specific to your company, and not all PCI DSS requirements dictate daily procedures. Use the following steps to define the daily operational procedures:

  1. Define procedures for updating the anti-virus. The anti-virus should be updated on a daily basis. Most commercial anti-viruses can update themselves automatically from the Internet, but the cardholder data environment should not have direct Internet access, so updating the anti-virus in it may be a manual task that needs to be performed daily. Describe the procedures for updating the anti-virus.

  2. Define procedures for updating software. Software should be updated regularly. PCI DSS specifies that it is acceptable to update software within a month of a critical patch being released, but it is recommended to apply vendor security patches as soon as possible. Since computers in the cardholder data environment should not have direct Internet access, updating them may be a manual task that needs to be performed regularly. Describe the procedures for updating software.

  3. Define procedures to identify new vulnerabilities. New vulnerabilities are discovered daily, which makes it necessary to track new vulnerabilities daily. Describe the procedures for checking Internet resources for information about new vulnerabilities.

  4. Define procedures to manage physical access. All physical entrances and exits should be monitored using video surveillance cameras. Someone should review the footage daily, ideally in real-time. Describe the procedures for monitoring and reviewing the video surveillance footage.

  5. Define procedures to secure logs. Logs should be backed up to a secure location daily. Ideally, logs should be sent to the centralized log server in real time. Describe the procedures for backing up logs, if needed.

  6. Define procedures to review Logs. Logs should be reviewed daily. Someone should be monitoring the log server for signs of unauthorized activity at any time. Describe the procedures for monitoring and reviewing logs, including any not sent to the centralized log server. Ideally, logs should be sent to the centralized log server in real time, including logs for:

    • A wireless IDS or IPS used to monitor the airwaves for unauthorized wireless devices. Ideally, the IDS/IPS should send alerts to the centralized log server.
    • File-integrity monitoring system. Ideally, the file-integrity monitoring system should send alerts to the centralized log server.
    • If a web application firewall is deployed, it should send alerts of suspected attacks to the centralized log server.

  7. Define the incident response plan. The incident response team should provide 24/7 monitoring of logs and response capabilities. The tasks for managing this should be defined as a part of the incident response plan and may also be included in the daily operational procedures.

  8. Define the key management procedures. Key management procedures involve generation and distribution of secure encryption keys. Describe the procedures for generating and distributing encryption keys.

  9. Define user management procedures. The following user management tasks may need to be performed any day:

    • Users should be assigned unique user IDs. Describe the procedures for creating new user IDs.
    • Describe the procedures for assigning new passwords and resetting passwords when requested.
    • Describe the procedures for additions, deletions and modifications of user accounts.
  10. Define procedures to handle visitors. The following procedures for handling visitors may have to be performed any day when there are visitors:
    • Visitors should be escorted when inside the facility. Describe the procedures for escorting visitors.
    • Visitors may need to be issued special badges that distinguish them from employees and authorize their access. Describe the procedures for issuing visitor badges.
    • Visitors should be logged when entering and exiting the facility. Describe the procedures for logging visitors.

  11. Define procedures for managing media containing sensitive data. The following storage media management tasks may need to be performed any day:

    • Media containing sensitive information should be labeled as "confidential," logged, and management should authorize the transfer. Describe the procedures for labeling, logging, and authorizing the transfer of media containing sensitive information.
    • Media containing sensitive information should be sent by secure courier, tracked when outside the facility ,and management should authorize the transfer. Describe the procedures for acquiring management authorization for transferring media containing sensitive data.
    • Media containing cardholder data should be destroyed once it is no longer needed for business or legal reasons. This includes hardcopy materials and digital media.

  12. Define procedures to perform background checks. Background checks should be performed on new employees. Describe the procedures for performing background checks on new employees.

  13. Define procedures to educate staff on security policies. New employees should be trained on the company security policies and practices upon hire, and all employees should undergo a review of these policies and practices at least annually.

  14. Define procedures to manage service providers. The PCI DSS compliance status of contracted service providers should be monitored.

More information about defining these procedures may be found in respective TeamMentor PCI DSS articles.
