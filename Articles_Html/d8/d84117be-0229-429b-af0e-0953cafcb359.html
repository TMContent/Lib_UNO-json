Applies to

  • PCI DSS Requirement 3.2.3

What to Check For

The cardholder PIN should never be stored.

Why

The cardholder PIN is used for authentication at ATMs and that makes it of high value to an attacker. In the rare event it is required for a transaction, it should not be stored after the transaction has been processed.

How to Check

Check the following to make sure that the PIN is not included:

  • Incoming transactions
  • All logs (transaction, history, debugging, error)
  • History files
  • Trace files
  • Database schemas
  • Database contents
  • Any other place where cardholder data is stored

How to Fix

Identify the storage mechanism that stores the PIN, for example a script or a database query. Change the storage mechanism so that it does not record the PIN.
