Applies to

  • PCI DSS Requirement 8.2.2

What to Check For

The user's identity should be verified before performing a requested password reset.

Why

The user's identity should be verified before performing a requested password reset to ensure that an attacker is not pretending to be the user to gain access.

How to Check

Review the password policy to make sure it requires that the user's identity is verified when a password reset is requested. Interview the responsible staff to make sure they verify the user's identity before resetting a password.

How to Fix

Add a requirement to the password policy to verify the user's identity when performing a requested password reset. Design and implement procedures for verifying the user's identity. For example, a userâ€™s identity may be verified by meeting in person.
