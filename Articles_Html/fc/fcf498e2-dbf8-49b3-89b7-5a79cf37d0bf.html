Applies to

  • PCI DSS Requirement 1.2.1.a
  • PCI DSS Requirement 1.2.1.b

What to Check For

Inbound and outbound network traffic should be limited to that which is necessary for business operations. Traffic that is not explicitly allowed should be denied by default.

Why

Denying unauthorized transmissions makes it more difficult for a potential attacker to successfully conduct an attack and extract sensitive information.

How to Check

Check the firewall configuration to make sure that there are rules to allow only connections necessary for business purposes. Check the firewall configuration to make sure that incoming and outgoing traffic is denied by default.

How to Fix

Identify network services necessary for business purposes. Implement a firewall configuration that reflects the ports and protocols used by the required network services. Add an explicit "deny all" statement to the firewall configuration or use an implicit "deny after allow" setting.
