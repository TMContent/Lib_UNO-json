What to Check For

Verify that session IDs are not disclosed.

Why

Do not disclose session IDs to make it harder for an attacker to compromise a user's session.

How To Check

To verify that session IDs are not disclosed:

  1. Find all pages that use session IDs. Make a list of all pages that use ASP.NET session management functions.

  2. Verify that a HTTPS binding exists. Start the "Internet Information Services (IIS) Manager". Right-click on the site and select "Edit bindings...". Verify that a https binding exists.

  3. Verify that TLS is enabled. Start the "Internet Information Services (IIS) Manager". Navigate the folders with pages that use ASP.NET session management. Double-click on the SSL icon in the right pane and make sure that is enabled.

  4. Verify that session cookies are marked as secure. Find code that handles session cookies. Make sure that form authentication and session ID cookies are marked as secure. Code similar to that below can be used to mark cookies as secure:

    if (Response.Cookies.Count > 0)
    {
    &nbsp;&nbsp;&nbsp; foreach (string cookie in Response.Cookies.AllKeys)
    &nbsp;&nbsp;&nbsp; {
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if (cookie == FormsAuthentication.FormsCookieName)
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; {
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Response.Cookies[cookie].Secure = true;
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; else if (cookie == “ASP.NET_SessionId”){
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Response.Cookies[cookie].Secure = true;
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }
    &nbsp;&nbsp;&nbsp; }
    }

How To Fix

To prevent disclosure of session IDs:

  1. The session IDs should be sent in non-persistent cookies. This is the setting used by ASP.NET by default. Review the application design to make sure that this default has not been changed.

  2. Find all pages that use session IDs. Make a list of all pages that use ASP.NET session management functions.

  3. Enable a HTTPS binding for the site. Start the "Internet Information Services (IIS) Manager". Right-click on the site and select "Edit bindings...". Add a binding of the "https" type.

  4. Enable TLS on all pages that use session IDs. Start the "Internet Information Services (IIS) Manager". Navigate the folders with pages that use ASP.NET session management. Double-click on the SSL icon in the right pane and enable TLS for the folder.

  5. Mark session cookies as secure. Marking session cookies as secure assures that they are only sent over HTTPS connections.

    if (Response.Cookies.Count > 0)
    {
    &nbsp;&nbsp;&nbsp; foreach (string cookie in Response.Cookies.AllKeys)
    &nbsp;&nbsp;&nbsp; {

    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; {

    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; }

    }
