What to Check For

Verify that all user access to cardholder data is logged.

Why

Logging user access to cardholder data helps investigate events in which cardholder data might have been accessed through compromised user accounts.

How to Check

Perform the following actions to make sure that user access to cardholder data is logged:

  1. Identify functionality that accesses cardholder data. Make a spreadsheet about all the functionality implemented by the application that involves cardholder data.
  2. Identify code that accesses cardholder data. For each feature that uses cardholder data, add information to the spreadsheet about the code that implements that feature. Record the names of the source code files and the names of the functions that handle cardholder data.
  3. Verify that access to cardholder data is logged. Examine the code that implements each feature that accesses cardholder data to make sure that it generates log entries that indicate that cardholder data has been accessed. Examine application logs to make sure that cardholder access events have been logged.

How to Fix

To log user access to cardholder data:

  1. Identify functionality that accesses cardholder data. Make a spreadsheet about all the functionality implemented by the application that involves cardholder data.
  2. Identify code that accesses cardholder data. For each feature that uses cardholder data, add information to the spreadsheet about the code that implements that feature. Record the names of the source code files and the names of the functions that handle cardholder data.
  3. Add code to log access to cardholder data. For each feature that accesses cardholder data, modify the code that implements that feature to generate log entries indicating that cardholder data has been accessed.
