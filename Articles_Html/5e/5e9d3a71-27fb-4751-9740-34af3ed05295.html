Applies to

  • PCI DSS Requirement 11.2

What to Check For

Vulnerability scans should be documented.

Why

Vulnerability scans should be documented to prove that they took place and to show what vulnerabilities may have already been found and fixed.

How to Check

Perform the following steps to verify that vulnerability scans are documented in compliance with PCI DSS:

  1. Verify that vulnerability scans have been documented. Examine the security documentation to verify that it includes the results of the previous vulnerability scans.

  2. Verify that scans have been performed regularly. Inspect documentation to verify that scans&nbsp;have been&nbsp;performed after changes to the network. It is acceptable to perform an internal scan after network changes. An internal scan is typically less expensive than an external scan. The scan results include rescans until no more serious vulnerabilities are found.

  3. Verify that a qualified scanning vendor has performed quarterly scans. An Approved Scanning Vendor (ASV) should perform the mandatory quarterly scans. Review documentation to verify that an ASV has performed the quarterly scans.

  4. Verify that vulnerabilities have been fixed.&nbsp;If vulnerabilities are found during scanning, they should be fixed and the scan should be repeated. The final results of the scan should not include any serious vulnerabilities. Review documentation to make sure that all found vulnerabilities have been fixed.

How to Fix

Use the following steps to assure that vulnerability scans are performed and documented in compliance with PCI DSS:

  1. Conduct vulnerability scans. Schedule quarterly internal and external vulnerability scans. Perform&nbsp;the vulnerability scans. If&nbsp;a scan is being performed as a result of network changes, it is acceptable to have qualified internal staff perform the scan.Required quarterly scans should be performed by a PCI DSS qualified ASV. It may be prudent to conduct a preliminary internal test with the available resources to eliminate the most obvious vulnerabilities, if any, before the ASV comes in.

  2. Fix detected vulnerabilities. If vulnerabilities are found, fix them and perform the scan again. Repeat scanning and fixing vulnerabilities until the scan returns acceptable results.

  3. Update documentation. Record the results. Attach them to the security documentation.
