What to Do

Protect back-end connections. For front-end to back-end connections: Three solutions are possible, although none are complete.

  1. Change passwords periodically. Use generated passwords or keys that are changed automatically and must be entered at given time intervals by a system administrator. These passwords will be held in memory and only be valid for the time intervals.

  2. Limit access to the back-end. The passwords or keys should be limited at the back end to only performing actions valid for the front end, as opposed to having full access.

  3. Time-stamp messages. The messages sent should be tagged and checksummed with time sensitive values so as to prevent replay-style attacks.
