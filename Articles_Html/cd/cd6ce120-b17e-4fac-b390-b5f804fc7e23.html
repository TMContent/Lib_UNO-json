What to Do

Require TLS for all private pages.

Why

TLS should be used to protect all pages that handle sensitive data, including all pages that use authentication or authorization features, to prevent unauthorized information disclosure.

When

Use TLS to secure all private pages.

How

To enable TLS on all sensitive pages:

  1. Identify sensitive pages. Make a list of all pages that handle sensitive information, including all pages that use authentication or authorization features.

  2. Enable a HTTPS binding for the site. Start the "Internet Information Services (IIS) Manager". Right-click on the site and select "Edit bindings...". Add a binding of the "https" type.

  3. Enable TLS for each folder containing sensitive pages. Start the "Internet Information Services (IIS) Manager". Navigate the folders with sensitive pages. Double-click on the SSL icon in the right pane and enable TLS for the folder.
