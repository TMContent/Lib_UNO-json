What to Do

Configure the Windows access control system to comply with the PCI DSS Requirement 7.2.

Why

The access control lists implemented in Windows may be used to satisfy the PCI DSS Requirement 7.2 if properly configured.

How

Perform the following tasks to configure the Windows access control system in compliance with PCI DSS:

  1. Set the deny-all default.  Right click on each drive or shared folder and select "Properties". Select the "Security" tab. Windows assigns some privileges by default. Remove the default Windows privileges where needed. Remove all the access privileges that are not required for each user's job functions. An empty Windows access control lists default to deny-all.

  2. Define data access requirements. Prepare access authorization documents for each employee based on job-related needs for data access. Have management authorize all data access authorization documents.

  3. Grant privileges. Review the access authorization documents for each user to determine the assigned privileges. Implement the assigned privileges using Windows access control lists.
