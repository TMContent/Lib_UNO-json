Applies to

  • Microsoft® Windows Communication Foundation (WCF) 3.5
  • Microsoft Visual Studio® 2008

What to Do

Consider using basicHttpBinding in your service, when you have ASMX client base. 

Why

BasicHttpbinding is a flexible way to provide support for the existing legacy clients, because it does not require upgrading them to WCF. Additionally basicHttpbinding provides a wide range of authentication schemes. Transport security supports basic, digest, windows and certificate authentication; message security supports username and certificate authentication

When

If you have ASMX client base, consider using basicHttpBinding in your service, because it  can be consumed by both WCF and legacy clients.

How

To use basicHttpBinding, create an endpoint, with binding configuration to use basicHttBinding. Security mode is turned off by default. To enable security and authentication you will need to create a binding configuration and configure the endpoint to use it.
