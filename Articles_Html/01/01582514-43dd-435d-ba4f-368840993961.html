Applies to

  • Microsoft&#174; Windows Communication Foundation (WCF) 3.5
  • Microsoft Visual Studio&#174; 2008

What to Do

If you store role information in SQL Server, configure your application to use the SqlRoleProvider for authorization.

Why

The role provider allows you to load the roles for users without writing and maintaining custom code.

When

Consider using this guideline when&nbsp;you store role information in SQL Server.

How

The following steps show how to use SQL Server Role to provide role based authorization.

  1. Enable the Role provider as shown below and configure the connection string pointing to the role store in SQL server 
    1. &#8230;
      <configuration>
      &#8230;
      <connectionStrings>
      &nbsp;&nbsp;&nbsp; <add name="MyLocalSQLServer"
      &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; connectionString="Initial Catalog=aspnetdb;data source=Sqlserver;Integrated Security=SSPI;"
    2. <system.web>
      <roleManager enabled="true" defaultProvider="MySqlRoleProvider" >
            <providers>
              <add name="MySqlRoleProvider"
                   connectionStringName="MyLocalSQLServer"
                   applicationName="MyAppName"
                   type="System.Web.Security.SqlRoleProvider" />
            </providers>
          </roleManager>
      &#8230;
      <system.web>
  2. Configure the Service Behavior. Set the principalPermissionMode attribute as UseAspNetRoles and the roleProviderName attribute to MySqlRoleProvider.
&#8230;
<system.serviceModel>
    <behaviors>
      <serviceBehaviors>
        <behavior name="BehaviorConfiguration">
          <serviceAuthorization principalPermissionMode="UseAspNetRoles"
            roleProviderName="MySqlRoleProvider" />          
         </behavior>
      </serviceBehaviors>
    </behaviors>
<services>
    <service behaviorConfiguration=" BehaviorConfiguration " name="MyService">
      <endpoint binding="wsHttpBinding" bindingConfiguration=""
        name="httpsendpoint" contract="IMyService2" />
     </service>
  </services>
  </system.serviceModel>
&#8230;
