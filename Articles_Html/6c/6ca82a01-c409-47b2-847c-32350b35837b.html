Applies to

  • PCI DSS Requirement 7.2.2

What to Check For

Access control systems should enforce the access privileges, which should be assigned based on job function.

Why

Access control privileges should reflect job requirements in order to allow the employees to perform their duties but prevent them from waging insider attacks.

How to Check

Review the access authorization documents to make sure access privileges are explicitly defined. Verify that the access privileges are granted based on job functions. For a sample of users, verify that the defined access privileges match the implementation.

To check the access control settings on Windows, right click on the drive or folder that you are auditing and select "Properties" then go to the “Security” tab. Verify that only the needed access privileges are granted. To check access control settings on UNIX-type systems, issue the "ls -Al" command to view a listing of files in the current folder, or the "ls -ld ." command to view the access control settings of the current folder.

How to Fix

Identify the job functions of each employee. Identify the data access required for each employee. Define the data access privileges for each employee based on job responsibilities. Have the management review and authorize the data access privilege specifications. Once approved, implement the data access control systems. Attach the access authorization documents to the network documentation.
