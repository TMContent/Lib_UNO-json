Applies to

  • PCI DSS Requirement 6.6

What to Do

Review public-facing web applications regularly.

Why

Reviewing public-facing web applications may help discover and fix vulnerabilities before they are discovered and exploited by attackers.

When

Public-facing web applications should be reviewed at least annually and after any changes if a web application firewall is not deployed.

How

Perform the following tasks to review public-facing web applications in compliance with PCI DSS:

  1. Choose a team to conduct the web application security review. Have an organization or department that specializes in application security perform the review. If you have an in-house department that specializes in application security, then it is natural to assign this duty to them. Most likely, though, you will need to look for an outside company to do the review. Security Innovation specializes in application security and can assist your company with reviewing your web application(s) for PCI DSS compliance.

  2. Fix discovered vulnerabilities. Correct all discovered vulnerabilities. The organization that does the review should provide guidance on fixing any discovered vulnerabilities. Fix all the discovered vulnerabilities and document the changes in accordance with&nbsp;the change-control procedures.

  3. Re-evaluate the web application after any corrections. If any vulnerabilities are found and fixed, re-evaluate the application by having the selected organization or department that specializes in application security review it again.
