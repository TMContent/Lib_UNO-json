What to Do

Use password protected .pfx files to protect your private key.

Why

Using .pfx instead of plain .snk files for signing gives extra security from intruders.

When

If you do not use delay-signing, this approach is more appropriate for small to medium sized projects. If you previously passed around or had access to .snk files that included a private key, you should consider using .pfx files instead.

How

Visual Studio has support for .pfx files, which makes this very convenient. When using the .pfx files for sigining your aseembly, the strong naming utility (SN.EXE) will not accept .pfx files which contains multiple certificates. Also the .pfx files needs to have a password, the SN.exe will not work with a .pfx file with null password. 

If you want to create a self signed .pfx key, the easiest way is to use Visual Studio: 

  • With the project node selected in Solution Explorer, from the Project menu, click Properties (or right-click the project node in Solution Explorer, and click Properties).

  • In the Project Designer, click the Signing tab.

  • Select the Sign the assembly check box.

  • Specify a new key file. In the Choose a strong name key file drop-down list, select <New...>. Note that new key files are always created in the .pfx format.

  • In the Create Strong Name Key dialog box, enter a name and password for the new key file, and then click OK.
