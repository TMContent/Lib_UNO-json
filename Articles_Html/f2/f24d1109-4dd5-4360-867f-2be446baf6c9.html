Applies To

  • Web Applications

What to Do

Mask all password fields - use input controls that hide user input for password fields by covering user input with a "mask", typically a character, such as "*". When a user enters a password "password", the masked field may look like "".

Why

Masking password fields helps protect user passwords from someone that may be looking at their screen.

How

To mask all password fields:

  1. Identify all password-type fields in the application. Review all the pages in the application to find all password-type fields. Password-type fields include secret answers, CVV, PIN, and other data, which is equivalent to passwords.
  2. Use password fields that mask user input. Masking password fields is usually done by using a "password" type HTML input field, like so: <form>
    ...
    <input type="password" name="pass" AUTOCOMPLETE="off"/>
    ...
    </form>
    Please, note that the autocomplete property is set to "off" - this prevents the browser from remembering the password in its autocomplete list. Preventing the browser from storing passwords in the autocomplete list protects the passwords from being compromised if the user's operating system is compromised.
