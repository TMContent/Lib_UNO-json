What to Check For

Verify that session IDs are changed during authentication.

Why

Change session IDs during authentication to make sure that the session state corresponds to the authenticated user.

How to Check

To verify that session IDs are changed during authentication:

  1. Identify authentication functionality. Find the code that is responsible for authentication.

  2. Verify that session IDs are changed during authentication. Review code responsible for logging in to make sure that code exists that clears the existing session state and creates a new session ID. For example:

    HttpSession session = request.getSession();
    session.invalidate();
    session = request.getSession(true);

How to Fix

To change session IDs during login:

  1. Identify&nbsp;authentication functionality. Find the code that is responsible for authentication.

  2. Change session IDs during authentication. Add code that invalidates the current session ID and creates a new one. For example:

    HttpSession session = request.getSession();
    session.invalidate();
    session = request.getSession(true);
