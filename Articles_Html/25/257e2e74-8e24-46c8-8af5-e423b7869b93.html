Applies to

  • PCI DSS Requirement 3.2.2

What to Check For

The card verification code (CVV2, CVC2, CID, CAV2) should not be stored under any circumstances.

Why

The card verification code should not be stored in order to limit the potential damage in the event of a successful attack. The card verification code is often used for authentication in online transactions but does not need to be stored after the transaction has been processed.

How to Check

Check the following to make sure they don't include the card verification code:

  • Incoming transaction data
  • All logs (transaction, history, debugging, error)
  • History files, trace files
  • Database schemas
  • Database contents
  • Any other place where the cardholder data is stored

How to Fix

Identify the storage mechanism used to store the card verification code, such as a program or a database query. Change the storage mechanism so that it does not record the card verification code.
