What to Do

Prevent Developers from modifying the trust of level of applications. 

Why

Developers may not be aware of other applications that are running on the hosted environment that may compromise security if isolation is not maintained. If a Web server administrator uses code access security to ensure application isolation and restrict access to system-level resources, it defeats the purpose if developers can change their permission levels at will. Developers are prone to taking deployment shortcuts that sometimes short-change security. Preventing modification of the trust level helps to ensure that proper controls are not bypassed.  

When

Application service providers or anyone responsible for running multiple Web applications on the same server should lock the trust level for all Web applications. This guideline is appropriate whenever multiple applications need to be isolated from each other in a shared hosting environment, or whenever the temptation to increase an applications privilege level for the sake of convenience is significant.

How

Enclose the element in the machine-level Web.config file in a <LOCATION> tag, and set the allowOverride attribute to false, as shown in the solution example.

To lock the trust level set the allowOverride element to false inthe machine-level Web.config file:

<location allowOverride="false">
  <system.web>
   <trust level="Medium" originUrl="" />
  </system.web>
 </location>

Solution Example

Enclose the <trust> element in the machine-level Web.config file in a <location> tag, and set the allowOverride attribute to false:

...
<location allowOverride="false">
   <system.web>
     ...
    <trust level="Medium" originUrl=""/>
    ...
   </system.web>
...
</location>           
...

Putting these tags into the machine-level config file prevents overriding the settings contained therein.
