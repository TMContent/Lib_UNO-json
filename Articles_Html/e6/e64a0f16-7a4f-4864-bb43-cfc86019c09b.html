Applies to

  • PCI DSS Requirement 7.2.1
  • PCI DSS Requirement 7.2.2
  • PCI DSS Requirement 7.2.3

What to Check For

An access control system should be implemented and it should cover all system components, enforce the data control policy and individual privileges, and have a default "deny all" setting.

Why

A properly configured access control system helps prevent unauthorized access.

How to Check

Use the following steps to check if the access control system:

  1. Check that ACLs are supported. Identify all computers and other system components that store sensitive information. Examine each system component to make sure it implements an access control system and that the access control systems are properly configured. Most modern Operating Systems and network devices implement access control systems.

  2. Review data access requirements. Review the access authorization documents to make sure access privileges are explicitly defined. Verify that the data access privileges are granted based on job functions.

  3. Check the data access permissions. Verify that all the access control system implementations have a "deny-all" default. For a sample of users, verify that the defined access privileges match the implementation.

How to Fix

Perform the following actions to implement an access control system:

  1. Deploy ACLs. Implement an access control system. Most modern Operating Systems and network devices implement access control systems.

  2. Set a "deny-all" default. Enable the "deny-all" default setting on all access control systems.

  3. Configure ACLs. Identify the access privileges for each user, using the data access authorization documents. Implement the correct data access privileges for each user.
