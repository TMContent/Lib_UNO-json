Applies to

  • PCI DSS Requirement 8.2.6

What to Check For

Passwords for new user accounts should be unique and changed after first use.

Why

Passwords for new user accounts should be unique and changed after first use to ensure that strong passwords are used and that as few people as possible know the passwords.

How to Check

Use the following steps to check that passwords for new users accounts are unique and have to be changed after first use:

  1. Review policy. Review the user authentication policy to make sure that it includes a requirement for the passwords for the new accounts to be unique and changed on first use.

  2. Conduct interviews. Interview the responsible personnel to verify that procedures are in place to generate unique passwords and that the passwords must be changed after first use.

  3. Examine the Active Directory configuration. In a Microsoft Active Directory environment, verify that when a new account is created, the box, "User must change password at next logon," is checked.

How to Fix

Perform the following actions to assure that passwords for new user accounts are unique and have to be changed after first use.

  1. Update policy. Formulate the user authentication policy based on the guideline article linked at the bottom of this page.

  2. Generate unique passwords. Use a random password generator to generate new unique passwords. The probability of the same strong password being generated twice is so low that it is negligible.

  3. Configure Microsoft Active Directory. When creating a new account, check the box that says, "User must change password at next logon." If there has been no dialog that has presented such option, this option may be found by following these directions: Click on "Start", "Programs", "Administrative Tools", "Active Directory Users and Computers", navigate to the user list, right click on the new user account, select "Properties", select the "Account" tab, the "User must change password at next logon" checkbox will be there.
