Applies To

  • Web Applications
  • Server-side

What to Do

Centralize authentication controls.

Why

Implementing a single set of centralized authentication controls helps implement them correctly, use them correctly in the application, and manage them later on.

How

To centralize authentication controls:

  1. Identify application requirements. Consider the following aspects of authentication controls:
    1. Authentication should be required for all private pages.
    2. All password fields should be masked.
    3. Accounts should be locked after multiple failed authentication attempts.
    4. Authentication should be implemented on the server-side.
    5. Authentication controls should be centralized.
    6. Authentication controls should fail securely.
    7. Strong authentication credentials (passwords) should be enforced.
    8. Account management functions should be protected.
    9. User credential (password) changing functions should be protected.
    10. Re-authentication should be required for sensitive operations.
    11. Authentication credentials (passwords) should expire.
    12. Authentication events should be logged.
    13. Authentication attempts should be throttled.
    14. Passwords should be hashed and salted.
    15. Authentication credentials (connection strings) for external resources should be protected.
    16. Authentication with external services.
  2. Define authentication control APIs. Make the exposed APIs simple to use for developers. Define authentication APIs to satisfy all application requirements. Include functionality to authenticate with external services.
  3. Implement and document authentication controls. Implement and document authentication APIs to satisfy all application requirements.
  4. Use authentication controls in the application. Developers should use the centralized authentication controls whenever authentication is necessary.
