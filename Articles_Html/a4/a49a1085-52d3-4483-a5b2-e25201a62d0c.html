Applies to

  • PCI DSS Requirement 4.1
  • PCI DSS Requirement 4.1.c

What to Do

Use TLS for web pages that handle cardholder data. Design the site in such a way that https:// is always visible in the URL of those pages.

Why

HTTPS in the URL tells the user that the session is encrypted with TLS. If frames are used in a web application, then the URL may not accurately reflect whether TLS is used.

How

Use TLS to encrypt cardholder data sent to the web application. Install the TLS implementation that applies to the web server that you are using. Configure the web server so that TLS is used for accessing the web application. If TLS is not used for all parts of the web application, configure the server so that the parts of the web application that require cardholder data use TLS. If frames are used in the web application, place the page with the frame-set in a location that is accessible only through HTTPS.
