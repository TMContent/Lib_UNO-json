Applies To

  • iOS applications that handle sensitive data.

What to Do

Do not log sensitive data.

Why

If sensitive data is logged, it may be recovered by an attacker if the device is lost or compromised.

How

To prevent logging of sensitive data:

  1. Identify sensitive data. Make a list of all sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Review logging code. Identify code that generates log entries by searching for NSLog() function calls. Review this code to verify that it does not include sensitive information in log entries. If necessary, rewrite this code in a way that does not log sensitive data. If NSLog() calls are necessary for debugging, they can be conditionally commented out using #define directives.
