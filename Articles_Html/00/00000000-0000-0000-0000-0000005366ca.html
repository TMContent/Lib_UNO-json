Applies To

  • iOS applications that access data stored in a SQL database.

What to Do

Use parameterized methods for database access.

Why

Use parameterized methods for database access to prevent SQL injection vulnerabilities.

How

To use parameterized methods for local database access:

  1. Identify the APIs used for SQL database access. The most common library for SQL database access in iOS applications is SQLite.
  2. Identify available parameterized SQL access APIs. Research the methods for executing queries provided by the API used in your application and choose ones that separate SQL commands and data. APIS that accept SQL queries as strings are dangerous and should be avoided. In SQLite, the way to use parameterized queries is to use prepared statements and pass data as parameters. The statements and data must first be converted to UTF-8 or UTF-16 character encodings. The sqlite3_prepare, sqlite3_prepare_v2, sqlite3_prepare16, and sqlite3_prepare16_v2 functions create prepared statements. After a statement is prepared, variables are bound to it using sqlite3_bind_* functions. After variables have been bound, the query is executed using the sqlite3_step function as many times as necessary to get the full resultant data set - each call to sqlite3_step produces one row. The query can be reset with the sqlite3_reset function - this resets the query result pointer to zero, so that sqlite3_step would start from the first result row again. After database access is complete, the query and its result should be destroyed using the sqlite3_finalize function.
  3. Identify code that executes SQL queries. Find all application code that executes SQL queries.
  4. Use parameterized APIs to execute SQL queries. Make sure that the application only executes parameterized SQL queries. Rewrite existing code if necessary or add code that uses parameterized queries to perform database access.

There is no practical way to access remote SQL databases using native iOS API calls, so the most common way to access remote SQL databases from iOS applications is through a Web Service wrapper for the database server. To use parameterized methods for remote database access:

  1. Set up a Web Service back-end server. Choose a platform for deploying the Web Services back-end and configure the server appropriately.
  2. Write the Web Service code. Implement the necessary Web Services on the platform of your choice.
  3. Use parameterized APIs to execute SQL queries. When implementing the Web Services, make sure to use parameterized APIs for database access to prevent SQL injection vulnerabilities.
  4. Connect the application to the back-end. Add code to the iOS application to interact with the back-end. It might be necessary to add some pages for configuring the connection to the back-end, such as setting the URL of the Web Services and the authentication credentials.
