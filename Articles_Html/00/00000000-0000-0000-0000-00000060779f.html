Applies To

  • iOS applications that send or receive sensitive data over the network.

What to Do

Use TLS to protect network communications.

Why

Properly using TLS to encrypt network communications prevents eavesdropping attacks and other types of man-in-the-middle attacks.

How

To use TLS to protect communications:

  1. Identify all communications performed by the application. Review the application design to identify all network communications performed by the application.
  2. Define communication security requirements. Define which network communications need to be encrypted. All transmissions that include sensitive data, including authentication credentials and session IDS, should be encrypted. In some cases, it might be practical to encrypt all network connections.
  3. Identify code that uses the network. Identify code that is responsible for each network communication that needs to be encrypted.
  4. Add code that uses TLS. Rewrite code that performs sensitive network communications to use TLS. The NSURLRequest and NSURLConnection classes support TLS with an acceptably secure default configuration starting with iOS 5 - this is the most practical way to use TLS on iOS and should be sufficient for most applications.
