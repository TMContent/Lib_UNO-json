Applies To

  • HTML5

What to Check For

Verify that session identifiers are not stored in local storage.

Why

HTML5 local storage is not designed to store sensitive information and is not suitable for that purpose. Data in local storage may be accessed by JavaScript, which means that it can be compromised via cross-site scripting (XSS) attacks. Cookies with the HTTPOnly flag set may not be accessed with JavaScript, so they are much harder to compromise via XSS.

How to Check

To verify that session identifiers are not stored in local storage:

  1. Find code that stores data in local storage. Search the application code for references to the localStorage object. Create a spreadsheet to list all code that stores or accesses session identifiers in local storage.
  2. Verify that session identifiers are not stored in local storage. Examine each reference to the localStorage object to identify whether it is used to store or access session identifiers.

How to Fix

To remove code that stores session identifiers in local storage:

  1. Find code that stores session identifiers in local storage. Search the application code for references to the localStorage object. Examine each reference to the localStorage object to identify whether it is used to store or access session identifiers. Create a spreadsheet to list all the code that stores or accesses session identifiers in local storage.
  2. Store session tokens in protected cookies. The recommended way to handle sessions in web applications is to store session tokens in cookies that have HTTPOnly and Secure flags set. If the application uses a mechanism for session management other than storing session identifiers in protected cookies, it should be rewritten. Most web application platforms are designed in a manner to facilitate using cookies to store session identifiers, so there should be plenty of documentation and the overall process should be pretty simple.
  3. Remove code that stores session identifiers in local storage. Once the application stores session identifiers in protected cookies, code that stores and accesses session identifiers in local storage can be safely removed. Use the spreadsheet you created to find this code and remove it. Test these changes thoroughly, especially if you are making them to a live application.

Local Storage Guidelines

  • Host Different Applications on Different Subdomains
  • Do Not Store Sensitive Information in Local Storage
  • Use Session Storage Instead of Local Storage If Possible
  • Do Not Store Session Identifiers in Local Storage
  • Do Not Store Authentication Information in Local Storage
