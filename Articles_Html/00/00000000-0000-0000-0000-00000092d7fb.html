Applies To

  • iOS applications that support file sharing.

What to Check For

Verify that file sharing is disabled in applications that support it.

Why

Disable file sharing to prevent unauthorized access. Some applications share all the user's files by default. If file sharing is necessary, it should only be enabled when it is being used.

How to Check

To verify that file sharing is disabled:

  1. Identify applications with file sharing capabilities. Enumerate all the installed applications. Check each application to see if it supports file sharing.
  2. Verify that file sharing is disabled. Examine application settings to make sure that file sharing is disabled.
  3. Verify that file sharing is disabled by default. When developing applications, check the default settings to make sure that file sharing is disabled by default.

How to Fix

To disable file sharing in applications that support it:

  1. Identify applications with file sharing capabilities. Enumerate all the installed applications. Check each application to see if it supports file sharing.
  2. Disable or secure file sharing. Disable file sharing for each application that enables such functionality when file sharing is not being used. If the application offers security features to protect file sharing, such as password protection, use these features to protect file sharing. Even if security features are available for file sharing, file sharing should be disabled when it is not being used.
  3. Disable file sharing by default. When developing applications, disable file sharing by default in your applications.

Authorization Guidelines

  • Disable File Sharing in Applications by Default
  • Use Role-based Access Control

Authorization Checklists

  • File Sharing Is Disabled in Applications by Default
  • Role-based Access Control Is Used
