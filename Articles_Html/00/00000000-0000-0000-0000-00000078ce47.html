What to Check For

Verify that error information is being logged. Make sure that error logs include sufficient information to identify the time, the source, and the nature of the errors.

How to Check

To make sure that error information is being logged.

  1. Identify error handlers. Enumerate all the code that handles application errors. There may be a considerable amount of this code, which is a good thing. It should be alarming if there is very little error handling code.
  2. Check error handlers. Review error handling code to make sure that it includes instructions that logs error information. Verify that relevant information about the errors is being recorded, such as time-stamps, resources being accessed and the active username.
  3. Check error logs. Examine the error logs generated by the application to make sure they include relevant information.

How to Fix

To log error information:

  1. Identify the logging subsystem. Logging is one of the essential security components of an application. Find the logging functions that are provided by the application. Review them to figure out how they work and where they store the error files.
  2. Identify error handling code. Enumerate all the code that handles application errors. There may be a considerable amount of this code, which is a good thing. It should be alarming if there is very little error handling code.
  3. Determine relevant error details. For each block of error handling code, determine the information that should be logged to help debug the error. Make sure to include time-stamps. It may be helpful to log the active username and the accessed resources as applicable to help identify attacks.
  4. Add logging code to error handling routines. Add code that records the relevant information to the error handlers
