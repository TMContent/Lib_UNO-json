What to Check For

Verify that user permission is required before using geopositioning.

Why

Ask the user for permission before using geopositioning functionality to avoid privacy issues.

How to Check

To verify that user permission is required to use geopositioning functions:

  1. Identify if the application uses geopositioning. Search the application code for the use of getCurrentPosition and watchPosition functions to identify whether the application uses geopositioning. Create a spreadsheet to list each piece of code that calls these functions.
  2. Verify that the user is asked for permission to use geopositioning. Log in to the application and access the functions that require geopositioning. Take note of whether the application asks you for permission before using geopositioning.

How to Fix

To ask the user for permission before calling geopositioning functions:

  1. Determine if the application uses geopositioning. Search the application code for use of the getCurrentPosition and watchPosition functions to determine whether the application uses geopositioning. Create a spreadsheet to list each piece of code that calls these functions.
  2. Write a message asking the user for permission to use geopositioning. If the application uses geopositioning functions, write a polite message explaining to the user the benefits of using them, and asking for permission to use these functions.
  3. Ask the user for permission to use geopositioning. For each piece of code that uses getCurrentPosition and/or watchPosition functions, add code that displays the message from the previous step. Add a conditional statement that only executes code that uses getCurrentPosition and/or watchPosition functions if the user has granted permission to do so.
