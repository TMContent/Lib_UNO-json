What to Check For

Verify that sensitive data is not cached.

Why

Avoid caching sensitive data to prevent this data from falling into the wrong hands if the device is stolen or compromised.

How to Check

To verify that sensitive data is not cached:

  1. Identify sensitive data. Make a list of types of sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Identify code that creates temporary files used by the application. Review the application source code to find functions that are responsible for creating the temporary files to find ones that write sensitive data.
  3. Verify that sensitive data is not cached. Examine the code that creates temporary files used by the application to make sure that sensitive data is not stored in them.

How to Fix

To prevent caching of sensitive data:

  1. Identify sensitive data. Make a list of types of sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Identify code that creates temporary files used by the application. Review the application source code to find functions that are responsible for creating the temporary files to find ones that write sensitive data.
  3. Remove code that writes sensitive data to temporary files. Edit the functions that write sensitive data to temporary files to prevent caching of sensitive data.

Privacy Guidelines

  • Do Not Cache Sensitive Data
  • Do Not Log Location Data
  • Do Not Log Sensitive Data
  • Delete Cached Data after Use

Privacy Checklists

  • Sensitive Data Is Not Cached
  • Location Data Is Not Logged
  • Sensitive Data Is Not Logged
  • Cached Data Is Deleted after Use
