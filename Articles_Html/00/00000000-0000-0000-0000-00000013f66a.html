What to Check For

Verify that TLS is configured to use strong cipher suites.

Why

Choose strong TLS suites to avoid known vulnerabilities in TLS. Weak crypto suites are subject to crypto attacks which will decipher and compromise confidential data.

How to Check

Use SSLAudit, SSLScan, SSLDigger, or SSL Server Test to check the TLS cipher suites supported by the server:

  1. Choose a TLS scanning tool. SSLAudit is recommended for Windows and UN*X systems. SSLScan is an option for UN*X systems. SSLDigger is an option for Windows systems. SSL Server Test is a Web Application and is recommended for most applications, however it is limited to scanning only Internet-accessible servers.
  2. Use the TLS scanning tool. Run the TLS scanning tool of your choice against your server to make sure that only strong cipher suites are chosen. The exact definition of strong cipher suites varies between different standards. Make sure that your server's configuration matches the applicable standards. The simplest way to make sure of this is to run the IISCrypto tool as described in the "How to Fix" section below.

How to Fix

Use the free IISCrypto tool to choose strong TLS cipher suites:

  1. Download IISCrypto. IISCrypto may be downloaded at https://www.nartac.com/Products/IISCrypto/Default.aspx
  2. Run IISCrypto. IISCrypto is easy to deploy and has a very simple interface. Download and run IISCrypto.
  3. Chooose your desired compliance standard(s). IISCrypto comes with presets based on common information security standards. It's simplest just to choose the ones that are the most applicable to your situation. In a couple of clicks, your IIS TLS cipher suites should be configured in compliance with the selected standards.
