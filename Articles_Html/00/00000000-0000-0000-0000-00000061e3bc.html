What to Check For

Verify that strong session identifiers are used.

How to Check

To make sure that strong session identifiers are used:

  1. Load the application. Open the application in a web browser. Log in to the application. Any other way to start a session is okay, but logging in is a pretty sure way to start a session for any reasonably designed application.
  2. Examine the session identifier. Use the browser plug-in or settings page that allows viewing the cookies. Examine the session cookie.
  3. Review application code. Review the application code that handles session state. Make sure that the code either generates strong session IDs or the platform is configured to generate strong session IDs automatically.

How to Fix

Most platform-provided session management implementations use sufficiently strong session IDs by default. Examine your platform's configuration settings to make sure the strength of the session IDs has not been degraded by inappropriate settings.
