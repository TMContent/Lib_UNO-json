Applies To

  • Java

What to Check For

Verify that the default HttpSession object is used to identify sessions.

How to Check

To make sure that the default HttpSession object is used to identify sessions:

  1. Load the application. Open the application in a web browser. Log in to the application. Any other way to start a session is okay, but logging in is a pretty sure way to start a session for any reasonably designed application.
  2. Examine the session identifier. Use the browser plug-in or settings page that allows viewing the cookies. Make sure the JSESSIONID cookie is present. This cookie indicates that HttpSession object is used. Absence of this cookie should be alarming.
  3. (Optional) Review application code. The presence of the JSESSIONID session cookie is a strong indicator that the default session ID generation algorithm is being used. To be absolutely sure, review the application code that handles session state. Make sure that when session gets established HttpServletRequest.getSession method is used to start a new session.

How to Fix

Java HttpSession interface uses strong session IDs by default. Please, make sure that your application uses the default HttpSession object. Rewrite the application session state handling code, if necessary, to use the default HttpServletRequest.getSession to generate the new session ID.
