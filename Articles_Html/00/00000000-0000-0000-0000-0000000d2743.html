Applies To

  • HTML5

What to Do

Use the sandbox attribute to apply restrictions to iframe elements.

Why

The sandbox attribute can be used to prevent scripts from being executed from iframe elements, and to apply other restrictions to iframe elements. This reduces the risk that the content in that frame will be used maliciously. Iframe elements that load content from untrusted sites can be abused if the attacker replaces the legitimate content with malicious content. This functionality is commonly abused by attackers to run JavaScript that loads a sequence of exploits to try to compromise the user's system via the browser and plug-ins.

How

To use the sandbox attribute to apply restrictions to iframe elements:

  1. Find all iframe elements. Search the application code for "iframe" to find all iframe elements. Create a spreadsheet of all iframe elements used by the application.
  2. Determine acceptable restrictions that can be applied. Review each iframe element and determine the maximum restrictions that can be applied to it without impacting application performance. To specify restrictions, explicitly grant permissions. The options are:
    1. "" - This makes no explicit permissions, i.e., all restrictions are applied. Iframe content is treated as not from the same origin, the iframe cannot load content from the containing document, form submission is not allowed in the iframe, and scripts are not allowed to execute in the iframe.
    2. allow-same-origin - Allows iframe content to be treated as coming from the same origin.
    3. allow-top-navigation - Allows iframe to load content from the page that contains the iframe.
    4. allow-forms - Allows form submission with the iframe.
    5. allow-scripts - Allows scripts to execute within the iframe.
  3. Add sandbox attributes to iframe elements. Add sandbox attributes to all iframe elements and apply the maximum applicable restrictions to each respective element.
