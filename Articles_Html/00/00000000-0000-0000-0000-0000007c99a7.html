What to Check For

Verify that cached data is deleted after use.

Why

If the device is lost or compromised, cached data may be recovered by attackers.

How to Check

To verify that cached data is deleted after use:

  1. Identify cached data. Make a list of all data cached by the application. Find code that is responsible for each peace of data that is cached.
  2. Verify that sensitive data is not cached. Examine code that caches data to verify that sensitive data is not cached.

How to Fix

To delete cached data after use:

  1. Identify cached data. Make a list of all data cached by the application. Find code that is responsible for each peace of data that is cached.
  2. Do not cache sensitive data. Examine code that caches data to verify that sensitive data is not cached.
  3. Add code that removes cached data. For all code that caches data, make sure that there is code that removes the data that has been cached.

Privacy Guidelines

  • Do Not Cache Sensitive Data
  • Do Not Log Location Data
  • Do Not Log Sensitive Data
  • Delete Cached Data after Use

Privacy Checklists

  • Sensitive Data Is Not Cached
  • Location Data Is Not Logged
  • Sensitive Data Is Not Logged
  • Cached Data Is Deleted after Use
