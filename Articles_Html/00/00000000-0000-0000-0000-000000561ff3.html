Applies To

  • Android applications that cache data.

What to Do

Do not cache sensitive data.

Why

Avoid caching sensitive data to prevent this data from falling into the wrong hands if the device is stolen or compromised.

How

To prevent caching of sensitive data:

  1. Identify sensitive data. Make a list of all sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Identify code that creates temporary files used by the application. Review the functions in the application source code that are responsible for creating the temporary files. Find any such functions that write sensitive data.
  3. Remove code that writes sensitive data to temporary files. Edit the functions that write sensitive data to temporary files to prevent caching of sensitive data.

Privacy Guidelines

  • Do Not Cache Sensitive Data
  • Do Not Log Location Data
  • Do Not Log Sensitive Data
  • Delete Cached Data after Use

Privacy Checklists

  • Sensitive Data Is Not Cached
  • Location Data Is Not Logged
  • Sensitive Data Is Not Logged
  • Cached Data Is Deleted after Use
