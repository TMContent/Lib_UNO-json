Applies To

  • HTML5

What to Do

Do not evaluate data passed in messages as code.

Why

You should prevent data passed in messages from being evaluated as code in order to prevent malicious scripts sent by an attacker from being executed within the browser. While it is possible to restrict the ability of attackers to send messages to the application by validating the origin of the message, this control could be circumvented in a Man-in-The-Middle (MitM) scenario. It is therefore best to simply not evaluate data received by the application as code. Generally speaking, eval is a dangerous function and should always be used with care.

How

To prevent evaluating data passed in messages as code:

  1. Find all references to eval. Search application code for all references to the eval method. Create a spreadsheet to list all code that uses eval.
  2. Find the source of the data being evaluated. For each piece of code that uses eval, determine where the data supplied as the argument to eval is coming from. The main concern for the purposes of this control is whether this data has been received as part of an event message. In the spreadsheet, record whether the data supplied to eval has been received as a part of a message.
  3. Remove code that passes message data to eval. For each piece of code that passes message data to eval, remove the line of code that calls eval. Then remove the code that is no longer needed because message data is no longer being passed to eval. If this breaks some functionality, consider alternative ways to rewrite this functionality without calling eval to execute data passed in messages.

Web Messaging Guidelines

  • Do Not Evaluate Messages as Code
  • Use element.textContent to Assign Values to DOM Elements
  • Explicitly State the Expected Origin with postMessage
  • Validate Data Received by Event Handlers
  • Validate the Origin Attribute Against the FQDN
  • Check The Origin Attribute of Messages

Web Messaging Checklist Items

  • Messages Are Not Evaluated as Code
  • Element.textContent Is Used to Assign Values to DOM Elements
  • Expected Origin Is Explicitly Stated with postMessage
  • Data Received by Event Handlers Is Validated
  • The Origin Attribute Is Validated Against the FQDN
  • The Origin Attribute of Messages Is Checked
