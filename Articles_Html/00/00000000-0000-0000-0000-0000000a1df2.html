Applies To

  • Android applications that transmit sensitive data over the network.
  • Android applications that authenticate to a remote server.

What to Do

Use TLS to secure network communications.

Why

Using TLS to secure network communications prevents eavesdropping attacks.

How

To use TLS to protect network connections:

  1. Identify all communications performed by the application. Review the application design to identify all network communications that are performed by the application.
  2. Define communication security requirements. Define which network communications need to be encrypted. All transmissions that include sensitive data should be encrypted, including authentication credentials and session tokens. In some cases, it might be practical to encrypt all network connections.
  3. Identify code that sends data over the network. Identify the code responsible for each network communication that needs to be encrypted.
  4. Add code that uses TLS. There are multiple methods for using TLS in Android to choose from, depending on application requirements. Using the SSLSocketFactory class is one of the most common solutions.
