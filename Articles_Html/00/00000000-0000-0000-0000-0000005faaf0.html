What to Check For

Verify that TLS is used to protect network connections.

Why

Proper use of TLS to encrypt network communications prevents eavesdropping attacks and other types of man-in-the-middle attacks.

How to Check

To verify that TLS is used to protect network connections:

  1. Identify all communications performed by the application. Review the application design to identify all network communications that are performed by the application.
  2. Review communication security requirements. Examine which network communications need to be encrypted. All transmissions that include sensitive data, including authentication credentials and session IDs, should be encrypted. In some cases, it might be practical to encrypt all network connections.
  3. Identify code that uses the network. Identify code that is responsible for each network communication that needs to be encrypted.
  4. Verify that TLS is used to protect network connections. Examine code that is responsible for network connections that need to be encrypted and make sure that it uses TLS.

How to Fix

To use TLS to protect network connections:

  1. Identify all communications performed by the application. Review the application design to identify all network communications that are performed by the application.
  2. Define communication security requirements. Define which network communications need to be encrypted. All transmissions that include sensitive data should be encrypted. In some cases, it might be practical to encrypt all network connections.
  3. Identify code that uses the network. Identify code that is responsible for each network communication that needs to be encrypted.
  4. Add code that uses TLS. Rewrite code that performs sensitive network communications to use TLS.
