Applies To

  • iOS applications that handle sensitive data.

What to Check For

Verify that sensitive data is not logged.

Why

If sensitive data is logged, it may be recovered by an attacker if the device is lost or compromised.

How to Check

To verify that sensitive data is not logged:

  1. Identify sensitive data. Make a list of all sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Review logging code. Identify code that generates log entries by searching for NSLog() function calls. Review this code to verify that it does not include sensitive information in log entries.

How to Fix

To prevent logging of sensitive data:

  1. Identify sensitive data. Make a list of all sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Review logging code. Identify code that generates log entries by searching for NSLog() function calls. Review this code to verify that it does not include sensitive information in log entries. If necessary, rewrite this code in a way that does not log sensitive data. If NSLog() calls are necessary for debugging, they can be commented out in release mode using #define directives.

Privacy Guidelines

  • Do Not Cache Sensitive Data
  • Do Not Log Location Data
  • Do Not Log Sensitive Data
