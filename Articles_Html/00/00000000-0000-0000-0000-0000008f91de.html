What to Check For

Verify that authentication is protected using TLS.

Why

Sensitive data, such as usernames, passwords, hashes, and keys has to be transmitted during authentication. The communication channel should be encrypted to prevent this data from being intercepted.

How to Check

To make sure that authentication is protected using TLS:

  1. Authenticate using a web browser. Open the application with a web browser. Log in to the application. The browser should provide indication whether TLS is being used, usually in the URL bar.
  2. Examine the Web.config file. Review the Web.config file and make sure that the requireSSL attribute is set to true for the authentication element, if your application uses Forms Authentication.

How to Fix

To protect authentication using TLS:

  1. Configure IIS to use TLS. Get a valid certificate. Hosting service providers often sell valid TLS certificates. Install the certificate. Create an HTTPS binding for the site. A link in the "Additional Resources" section describes how to do this in detail.
  2. Enable requireSSL for Forms Authentication. If your application uses Forms Authentication, edit the Web.config and set the requireSSL attribute to true for the authentication element, like so: <authentication mode="Forms" requiressl="true"></authentication>
  3. Enable TLS for all pages. It's recommended to enable TLS for the entire site. Use the IIS Manager to require TLS for the entire site. The detailed instructions on how to do this are linked to at the bottom of this page.

Additional Resources

  • For information about the FormsAuthentication.RequireSSL property, see http://msdn.microsoft.com/en-us/library/system.web.security.formsauthentication.requiressl.aspx
