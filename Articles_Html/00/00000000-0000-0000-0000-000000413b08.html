Applies To

  • ASP.NET

What to Check For

Verify that logged user data is validated.

Why

Logged user data should be validated to prevent log tampering vulnerabilities. If users can cause arbitrary data to be written to the log files, an attacker will be able to forge log entries or to corrupt the log file.

To verify that logged user is validated:

  1. Find code that writes user data to logs. Make a list of functions that are used to write to logs. Using these function names as search strings should make it simple to find code that writes to logs. Look through the application to find all code that writes to the log files. Make a list of source code files that have code that writes to log files.
  2. Determine if writing user data to log files is necessary. Examine each piece of code that writes data to logs to see if it writes user-controlled data to log files. Make a list of each instance where user data is being written to log files. For each instance where user-controlled data is being logged, consider whether this is necessary or whether it is possible to write comparable log entries without including user data.
  3. Examine validation routines. Look through the list of code that writes user data to logs and make a list of types of user data that is being logged. Check to make sure that validation functions that handle each type of user-controlled data that is logged by the application exist. Make sure that validation routines reject input that contains special characters that may be misinterpreted by the application or the operating system if written to the log files.
  4. Verify that logged user data is validated. Go through the list of code that writes user data to logs and add calls to the respective validation routines. Verify that validation routines are applied to the user data before user data is written to the log files.

How to Fix

To validate logged user data:

  1. Find code that writes user data to logs. Make a list of functions that are used to write to logs. Using these function names as search strings should make it simple to find code that writes to logs. Look through the application to find all code that writes to the log files. Make a list of source code files that have code that writes to log files.
  2. Avoid writing user data to logs. Examine each piece of code that writes data to logs to see if it writes user-controlled data to log files. Make a list of each instance where user data is being written to log files. For each instance where user-controlled data is being logged, consider whether this is necessary or whether it is possible to write comparable log entries without including user data.
  3. Check for data validation routines. Look through the list of code that writes user data to logs and make a list of types of user data that is being logged. Check that validation functions exist that handle each type of user-controlled data that is logged by the application. Make sure that validation routines reject or encode input that contains special characters that may be misinterpreted by the application or the operating system if written to the log files.
  4. Validate logged user data. Go through the list of code that writes user data to logs and validate that it calls to the respective validation routines. Make sure appropriate validation routines are used to validate all user data before writing it to log files.
