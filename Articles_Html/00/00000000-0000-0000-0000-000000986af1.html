Applies To

  • All iOS applications.

What to Do

Sign application binaries for distribution.

Why

Cryptographically signing applications assures that the application binary has not been tampered with and that it came from a trusted source. Applications have to be signed with a trusted certificate to run on iOS.

How

To sign iOS applications:

  1. Prepare the application for the release. Build the application in release mode. Remove or disable any development settings or code from the application. Disable support for self-signed TLS certificates if it has been enabled. Disable generation of log entries that were used for debugging. Disable file sharing by default if the application supports it. Make sure the default application settings are secure. Make sure the application does not use any private iOS APIs.
  2. Create a distribution provisioning profile. You have to be a member of the iOS Developer Program to publish iOS applications. Once you have an account, you can go to the iOS Dev Center and submit requests for provisioning profiles. Download the new distribution profile.
  3. Install the distribution provisioning profile. Double-click the downloaded distribution profile and it should be added to Xcode.
  4. Create an application archive. Build your application and create the .ipa file. Make sure that Xcode is configured to build the application in release mode and using the correct distribution profile.
  5. Examine the application record in iTunes Connect. Go to iTunes connect and fill out the forms to create the application record, if necessary.
  6. Submit the application for publication. Go to Product, Archive in XCode and follow on-screen instructions to submit the application for approval.
  7. Wait for approval notification. The e-mail account associated with the iOS Developer Program account should receive a notification regarding the status of the approval process. This might take a week or more, but after it is finished, the application should be signed and ready for distribution.
