Applies To

  • iOS applications that handle sensitive data.

What to Check For

Verify that sensitive data is not cached.

Why

Data from temporary files may be recovered by an attacker if the device is lost or compromised.

How to Check

To verify that sensitive data is not cached:

  1. Identify sensitive data. Make a list of all sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Identify code that creates temporary files used by the application. Review the application source code to find functions that are responsible for creating the temporary files.
  3. Verify that sensitive data is not cached. Examine the code that creates temporary files used by the application to make sure that sensitive data is not stored in these files.

How to Fix

To prevent caching of sensitive data:

  1. Identify sensitive data. Make a list of all sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Identify code that creates temporary files used by the application. Review the application source code to find functions that are responsible for creating the temporary files. Find all such functions that write sensitive data.
  3. Remove code that writes sensitive data to temporary files. Edit any functions that write sensitive data to temporary files so that they do not write sensitive data.

Privacy Guidelines

  • Do Not Cache Sensitive Data
  • Do Not Log Location Data
  • Do Not Log Sensitive Data

Privacy Checklists

  • Sensitive Data Is Not Cached
  • Location Data Is Not Logged
  • Sensitive Data Is Not Logged
