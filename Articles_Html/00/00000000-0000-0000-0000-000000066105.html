What to Check For

Verify that file formats are validated.

Why

Many vulnerabilities are exploited via malformed documents supplied to the application. Attackers will frequently embed malformed documents in web pages - some applications are integrated into the web browser and will open malicious documents automatically. Making sure the file format is valid before loading files into the application makes it harder to embed exploit code into documents.

How to Check

  1. Identify the functions that open data files.
  2. Review validation code. Verify that these functions include provisions for validating the input data file format before passing it on to other functions.

How to Fix

Perform the following actions to validate file formats before loading files into the application:

  1. Define the file format. Specify valid values for all possible fields in all document types used by your applications. This information should be a part of project documentation and the developers may already have it.
  2. Review code. Search for code that loads saved files into the application. Examine the extent of the validation already being performed by the application. If some validation is already being performed, some code may be recycled for a more thorough validation routine.
  3. Implement validation. Write validation functions for all fields in all document types supported by the application.
  4. Centralize validation. Move the validation functions for all document fields to the centralized input and data validation class or library. Write a master file format validation function that calls all individual data field validation functions.
  5. Use validation. Go back to code that loads data files into the application. Remove any old validation code. Call the master file format validation function on all files before loading them into the application.
