Applies To

  • iOS applications that handle sensitive data.

What to Do

Do not cache sensitive data in temporary files.

Why

Data from temporary files may be recovered by an attacker if the device is lost or compromised.

How

To prevent caching of sensitive data:

  1. Identify sensitive data. Make a list of all sensitive information handled by your application. Some examples are:
    • Passwords
    • Encryption keys
    • Personal information
  2. Identify code that creates temporary files used by the application. Review the application source code to find functions that are responsible for creating the temporary files. Check if any of these functions write sensitive data.
  3. Remove code that writes sensitive data to temporary files. Edit the functions that write sensitive data to temporary files to prevent caching of sensitive data.

Privacy Guidelines

  • Do Not Cache Sensitive Data
  • Do Not Log Location Data
  • Do Not Log Sensitive Data

Privacy Checklists

  • Sensitive Data Is Not Cached
  • Location Data Is Not Logged
  • Sensitive Data Is Not Logged
