Applies to

  • JAVA
  • Struts 1

Summary

Perform output encoding using Struts 1 within an application.

Objectives

Strong output encoding is a control that is typically used to prevent encoding (primarily XSS on the web) issues within an application. This article will describe how to use the built-in mechanisms provided by Struts 1 to perform appropriate output encoding.

Code Examples

In general, Struts 1 tags perform HTML entity encoding by default (see XSS prevention cheat sheet in references). The link tag provides a mechanism to perform url encoding on parameter values. A few examples are given below.

Using bean:write to output simple text (performs HTML entity encoding).

<bean:write name="customer" property="streetAddress"/>

Using html:hidden to output a hidden field (performs HTML entity encoding).

<html:hidden property="storeId" />

Using html:link to output a url (performs URL encoding on url parameter values).

<html:link action="/myBookAction" 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; paramId="id" 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; paramName="book" 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; paramProperty="bookId">
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <bean:write name="book" property="bookTitle"/>
</html:link>

Note: An important point to be aware of is that there are a few tags that allow you to disable the encoding performed by the tag. This is sometimes useful, the most common case being where you are allowing administrators for a site add HTML formatted messages to be displayed to users. There may also be cases where users can create formatted messages for other users. These approaches are often not handled properly to prevent XSS. However, through strong input validation, it is possible to allow users to enter limited HTML and still display that safely to users without allowing XSS.

You can determine if this is being done by simply looking for the filter attribute in struts tags. If it is set to false, then the HTML entity encoding is disabled (the default is true). An example of disabling encoding is shown below.

<bean:write name="adminData" property="headerMessage" filter="false" />

Struts provides various tags that can perform output encoding. However, you must be aware of the encoding properties of the tag(s) you are using and encode appropriately for your given output context.
