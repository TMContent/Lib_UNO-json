What to Do

Mask all password fields - use input controls that hide user input for password fields by covering user input with a "mask", typically a character, such as "*". When a user enters a password "password", the masked field may look like "********".

Why

Masking password fields helps protect user passwords from someone that may be looking at their screen.

How

To mask all password fields:

  1. Identify every password field in the application.&nbsp;Review all the pages in the application to find every password-type field. Password-type fields include secret answers, CVV, PIN, and other data which are equivalent to passwords.

  2. Mask user input in every password field.&nbsp;There are different approaches to masking user input:

    Masking is usually done by using a "password" type HTML input field, as follows:

    <form>
    <input type="password" name="pass" autocomplete="off" />
    </form>
    Set the autocomplete attribute to off to prevent the user's browser from caching the entered password.

  3. Use masking with custom controls. If you are using third party controls for password fields, choose ones that mask user input. If you are implementing custom controls for password fields, implement these controls in such a way that they mask user input.
