What to Check For

Verify that a strong password policy is used to ensure that your passwords are complex enough to prevent users guessing other user passwords and to prevent a successful Credentials Dictionary Attack against your user credential store.

Why

Insufficiently complex passwords may be guessed by attackers, allowing them to compromise user accounts. Strong passwords make it much more difficult for automated attacks against your password store to succeed. Most users will not choose sufficiently strong passwords without guidance from the application.

How to Check

The way to check that a strong password policy is in place differs whether your application uses Active Directory or Windows Authentication. Checking that strong passwords are enforced is an easy way to improve security.

Checking the password policy when using Windows Authentication:

  1. Run the group policy editor by opening %SystemRoot%\system32\gpedit.msc or from the start menu by running gpedit.msc.

  2. Browse to the user password policy that can be found under Computer Configuration->Windows Settings->Security Settings->Account Policies->Password Policies.

  3. Verify that the following values are set according to the application's password policy and that the values are sufficiently strong. For example:

      • Enforce password history remembers 6 passwords or more
      • Maximum password age is at most 42 days
      • Minimum password length is 8 characters or more
      • Password must meet complexity requirements is Enabled
      • Store passwords using reversible encryption is Disabled

Checking the password policy when using Active Directory:

  1. Setup a management console for group policy management:

    1. Log on to the domain controller as administrator.
    2. Click the Start->Run, type mmc, and click OK.
    3. In the Console1 window, click File -> Add/Remove Snap-in.
    4. In the Add/Remove Snap-in dialog box, click Add.
    5. In the Add Standalone Snap-in dialog box:
      1. Click Active directory users and computers, and then click Add.
      2. Click Group Policy Object Editor, and then click Add.
      3. In the Select Group Policy Object dialog box:
        1. Ensure Local computer is selected under Group Policy Object.
        2. Click Finish.
      4. Click Close.

  2. Check the password policy:

    1. In the MMC console, click File, and then click->Save.
    2. Expand Active Directory Users and Computers.
    3. Right-click the domain of choice and click Properties.
    4. Click the Group Policy tab.
    5. The Domain Password Policy should exist and be set as No Override.
    6. Double click on Domain Password Policy.
    7. In the Group Policy Object Editor:
      1. Expand Computer Configuration -> Windows Settings -> expand Security Settings -> expand Account Policies.
      2. Click Password Policies.
      3. Verify that the following values are set according to the application's password policy and that the values are sufficiently strong. For example:
        • Enforce password history remembers 6 passwords or more
        • Maximum password age is at most 42 days
        • Minimum password length is 8 characters or more
        • Password must meet complexity requirements is Enabled
        • Store passwords using reversible encryption is Disabled

Problem Example

An online banking web site is using the Windows Authentication for their authentication. They unfortunately left the default Windows password policy in place. As a result, an attacker is able to brute force user's passwords very quickly, gaining access to both online accounts and many ATM card numbers.
