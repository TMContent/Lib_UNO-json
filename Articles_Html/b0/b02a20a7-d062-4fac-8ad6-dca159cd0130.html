Applies To

  • Web Applications
  • Server-side

What to Check For

Verify that sessions are destroyed when users log out.

Why

Invalidating sessions when users log out makes it harder to hijack them.

How to Check

To verify that sessions are destroyed when users log out:

  1. Identify logout functionality. Find code that is responsible for the logout functionality.
  2. Verify that logout code destroys user sessions. Examine logout code to make sure that it clears all session data and make sure that it destroys the session cookie.

How to Fix

To invalidate sessions when users log out:

  1. Identify logout functionality. Find code that is responsible for the logout functionality.
  2. Add code to invalidate sessions. Add code to the logout function to invalidate the session.
