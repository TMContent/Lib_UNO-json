Applies to

  • Microsoft® Windows Communication Foundation (WCF) 3.5
  • Microsoft Visual Studio® 2008

What to Do

Ensure that you are not passing any sensitive information in SOAP headers when using Http Transport and Message Security.

Why

If you pass sensitive information in the SOAP headers, it&nbsp;is sent in plain text and can&nbsp;it&nbsp;be stolen if you use message security.

When

Consider using this guideline when you need to pass information in soap headers over http protocol.

How

Do not use message security if you need to pass sensitive information in soap headers over http protocol. Use transport security instead to protect sensitive data passed in soap headers, such as user identities passed for auditing purposes.

Information contained in the SOAP headers is sent in plain text and can be stolen if you use message security.&nbsp; Soap header information is signed by default using message security, so the information can be read but cannot be spoofed.
