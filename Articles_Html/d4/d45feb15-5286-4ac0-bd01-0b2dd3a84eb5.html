What to Check For

Project documentation should include the results of a data flow analysis.

Why

Dataflow analysis helps find vulnerabilities that are related to input validation.

How To Check

Examine project documentation to verify that it includes the results of a data flow analysis.

How To Fix

Perform a data flow analysis by following these steps:

  1. For each input location determine how much you trust the source of input.  When in doubt you should give it no trust.

  2. Trace the flow of data to each possible output noting along the way any attempts at data validation. Document the flow of data from source to sink.

  3. Move to the next input and continue. Document all the data handled by the application.
