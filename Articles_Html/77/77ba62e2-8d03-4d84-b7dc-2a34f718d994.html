Applies to

  • PCI DSS Requirement 11.2

What to Do

Regularly perform network vulnerability scans.

Why

Automated vulnerability scans require relatively little effort and may detect security holes and misconfiguration.

When

Vulnerability scans should be performed at least quarterly by a PCI DSS qualified ASV. Qualified staff can perform vulnerability scans after any significant changes to the network.

How

Use the following steps to conduct network vulnerability scans:

  1. Perform internal network vulnerability scans. Choose a vulnerability scanner to fit your needs and budget. Some vendors offer demo versions that can give a preview of the functionality and ease of use. A vulnerability scanner is usually a Windows-based application. Choose a workstation to use for carrying out vulnerability scans. Install the vulnerability scanner software on the workstation. Delegate responsibility for performing internal network vulnerability scans to a qualified staff member. Have the responsible staff practice with the vulnerability scanner. Vulnerability scans should be performed after any major changes to the network by qualified internal staff.

  2. Perform external network vulnerability scans. Schedule the quarterly external scans with a PCI DSS qualified ASV. See the list of Approved Scanning Vendors linked at the bottom of this page. If vulnerabilities are found during scanning, fix them and repeat the scan. Document each iteration and vulnerabilities that are fixed. The final iteration should not discover any serious vulnerabilities.
