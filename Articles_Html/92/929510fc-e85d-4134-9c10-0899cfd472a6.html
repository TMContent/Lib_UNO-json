Most software needs to be managed from a centralized and easy to use administrative panel. Whether this is done through a web interface or local piece of software, these tools are often the heart of an applicationâ€™s management and are invaluable to functionality. This type of interface should be secured to prevent a security breach, as these controls may give attackers full reign over every part of the system. View this room to learn how administrative controls are properly secured and which functions should be included to provide proper management of an application.

Protect Administrative Controls

Application administration requires higher privileges than the main application interface. Separating the two interfaces decreases the risk that a normal user will be able to escalate privileges to that of an administrator.

Allow Managing Cryptographic Keys

The customer should have the ability to manage cryptographic keys to be able to change the keys as necessary. Being able to change the keys allows replacing the keys if any are compromised.

  • Any

Allow Changing Passwords

Any application that uses authentication should provide functionality to change passwords when necessary. This is required to prevent account credentials from being compromised. If passwords are hard-coded or permanently stored in a database or file, they cannot be changed easily by the customer or another user.
