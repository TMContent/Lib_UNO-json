Description

Unchecked error conditions might allow attacks to go unnoticed. Checking error conditions helps the application to determine if something out of the ordinary is happening and to log that information. Log information helps administrators determine when an attack is taking place and provides information about the attack. Unchecked error condition flaws might occur in any type of application.

Impact

Unchecked error conditions might result in other, more serious vulnerabilities. An unchecked error condition might not actually give any advantage to the attacker. However, if the application contains a lot of code that can produce unchecked errors, that creates a fertile field for the attacker to look for vulnerabilities.

Countermeasures

To prevent unchecked error conditions, use structured exception handling, use a global exception handler for unhandled exceptions, and log error information.

  • Handle Exceptions

Use a global exception handler:

  • Use Global Exception Handlers

Log error information:

  • Log Error Information

Application Check

To check for adequate protection against this vulnerability, verify that structured exception handling is used, a global exception handler is used, and error information is logged.
