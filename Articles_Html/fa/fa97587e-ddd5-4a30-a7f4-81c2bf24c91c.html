Applies to

What to Check For

Verify that all functions that may cause errors either throw exceptions or return error codes.

Why

It is preferable to throw exceptions, but it is better to return error codes than not handle errors at all. Exceptions should be used for error handling when possible. An older, less effective approach to error handling is called error codes, when a function returns values that correspond to error conditions. If, for some reason, exceptions are not being used, error codes should be used for error handling instead.

How to Check

Identify all functions that may cause errors. Verify that all functions that may cause errors throw exceptions or return error codes.

How to Fix

Identify all functions that may cause errors. Add code that either throws exceptions or returns error codes when error conditions occur.
