Applies To

  • ASP.NET 4.0

What to Do

Deny all access by default.

Why

Denying all access by default assures that only explicitly authorized access is possible.

When

Deny all access by default in all web applications that implement access control.

How

To deny all access by default:

  1. Enumerate all sensitive pages. Make a list of all pages that handle sensitive or private information.

  2. Enable forms authentication. Add the following section to Web.config using the appropriate settings:

    <system.web>
      <authentication mode="Forms">
        ...
      </authentication>
    </system.web>

  3. Deny all access by default. Configure the following tags in Web.config:

    <system.web>
      <authorization>
        <deny users="*" />
      </authorization>
    </system.web>

    The line <deny users="*" /> denies access to all users, including authenticated users.

  4. Configure authorization. For each sensitive page, add a section to Web.config that looks like the following: 

    <location path="[Filename of the page]">
       <system.web>
          <authorization>
             <allow roles="[Authorized roles]">
             <deny users="*">
          </authorization>
       </system.web>
    </location>

    [Filename of the page] is the file name of the sensitive page. [Authorized roles] is a comma separated list of roles that are authorized to use the page.
