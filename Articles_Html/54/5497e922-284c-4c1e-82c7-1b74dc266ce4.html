Applies to

  • PCI DSS Requirement 8.5.8

What to Check For

Shared or group accounts should not be used. Each user should have an individual account.

Why

Not using shared accounts ensures that actions can be traced back to individual users; this is known as non-repudiation.

How to Check

Use the following steps to make sure that shared accounts are not used:

  1. Make a list of authorized users. Review the authorization forms in the network documentation and make a list of authorized users.

  2. Make&nbsp;a list of user accounts. Use the user management application to review user accounts. Make a list of accounts. In a Microsoft Active Directory environment, the user management application can be launched by clicking on "Start", "Programs", "Administrative Tools", "Active Directory Users and Computers".

  3. Examine the list of user accounts. The number of user accounts should not be less than the number of authorized users. Make sure that there are no shared accounts on the list. A shared account is an account that is used by more than one person.

How to Fix

Perform the following tasks to assure that shared accounts are not used:

  1. Delete shared accounts. Launch the the user management application by clicking on "Start", "Programs", "Administrative Tools", "Active Directory Users and Computers". Delete all shared accounts.

  2. Create new user accounts. Make a new unique account for each authorized person that has been using the shared account. To create a new user, navigate to the user list, right click on it, select "New", "User"

  3. Configure the new user accounts. Fill in the user account information in the "New Object - User" dialog. Configure group membership and data access control permissions for the new user account. Peform other actions associated with creating new user accounts, such as adding e-mail accounts, allocating personal work-space, etc.
