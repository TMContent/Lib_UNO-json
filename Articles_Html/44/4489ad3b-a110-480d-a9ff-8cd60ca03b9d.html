What to Check For

Ensure that absolute URLs are used for navigation where the site is partitioned with secure and non-secure folders.

Why

Navigating between the public and restricted areas of your site (that is, between HTTP and HTTPS pages) is an issue because a redirect always uses the protocol (HTTPS or HTTP) of the current page, not the target page.

How To Check

To make sure that absolute URLs are used for navigating secure pages:

  1. Verify that absolute URLs are used for navigating from secure pages. Examine all secure pages to make sure that all links, especially dynamically generated links use absolute URLs.

  2. Verify that absolute URLs are used when redirecting to secure pages. Find all pages that redirect to secure pages and make sure they use absolute URLs to address secure pages.

How To Fix

Use the following guidelines when implementing navigation on a site that uses TLS:

  1. Use absolute URLs when navigating from secure pages. After a user logs on and browses pages in a directory that is secured with TLS, relative links such as "..\publicpage.aspx" or redirects to HTTP pages result in the pages being served using the HTTPS protocol, which incurs an unnecessary performance overhead. To avoid this, use absolute links such as "http://servername/appname/publicpage.aspx" when redirecting from an HTTPS page to an HTTP page.

  2. Use absolute URLs when redirecting to secure pages. Similarly, when you redirect to a secure page (for example, the logon page) from a public area of your site, you must use an absolute HTTPS path, such as "https://servername/appname/secure/login.aspx" instead of a relative path, such as "restricted/login.aspx". For example, if your Web page provides a logon button, use the following code to redirect to the secure login page.

    private void btnLogon_Click( object sender, System.EventArgs e )
    {
      // Form an absolute path using the server name and virtual directory name  
      string serverName =
       HttpUtility.UrlEncode(Request.ServerVariables["SERVER_NAME"]);  
       string vdirName = Request.ApplicationPath;
       Response.Redirect("https://" +
        serverName + vdirName + "/Restricted/Login.aspx");
    }
