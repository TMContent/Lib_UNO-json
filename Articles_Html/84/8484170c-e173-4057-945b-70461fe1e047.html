Applies to

  • PCI DSS Requirement 9.10
  • PCI DSS Requirement 9.10.1.a
  • PCI DSS Requirement 9.10.1.b
  • PCI DSS Requirement 9.10.2

What to Do

Define a media destruction policy. Destroy all storage media, including paper, in a manner that prevents data recovery. Protect the containers used to store media until destruction.

Why

A media destruction policy should cover all media containing cardholder data to ensure that all such media is consistently destroyed. Paper documents should be thoroughly destroyed to prevent unauthorized information disclosure in the event that paper waste falls into the hands of an attacker. Containers used to store media until it can be destroyed should be secured to prevent an attacker from taking the media.

How

Add a point to the media policy that requires all media containing cardholder data to be thoroughly destroyed, including paper hard copies. Acquire a shredder or incinerator for destroying paper hard copies or hire a company that specializes in media destruction. Acquire lockable trash bins to store sensitive hard-copy media before it is destroyed. Only place discarded sensitive hard-copy media in the lockable bins. Use the shredder or incinerator for destroying hard copies - do not throw them in the trash. Follow the instructions in the guideline article called "Erase Files Securely" to destroy physical media securely. It is sufficient to securely erase the files on digital storage media before discarding such media; however, it is recommended to physically destroy the digital storage media itself, using industrial equipment.
