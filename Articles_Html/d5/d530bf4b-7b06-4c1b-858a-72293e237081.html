Applies to

  • PCI DSS Requirement 6.3

What to Do

Design and document the software development processes to comply with PCI DSS.

How

Design and document the software development processes in compliance with PCI DSS by performing the following actions:

  1. Develop code in compliance with PCI DSS. Be aware of the PCI DSS requirements through-out the development process to make sure that the resulting application is PCI DSS compliant. It may be helpful to review the PCI DSS requirements to see which ones apply to the application(s) you are developing. Links to the PCI DSS requirements to consider when organizing software development are listed at the end of the article under Related Items.

  2. Test all changes. Do not use production data for testing, especially live PANs. Test all changes, including updates and significant configuration changes. Set up a test environment to closely simulate the production environment.

  3. Separate&nbsp;the development and production environments. Designate a separate development environment. Remove all development elements from the production environment.&nbsp;Assign different staff to the development and the production environments.

  4. Remove test data. Do not use production data for testing, especially live PANs. Remove test data and test accounts before the production system becomes active or is released to customers.

  5. Review custom code. Review all custom application code. Have someone other than the author of the code do the review. Have someone who is knowledgeable about securing coding practices do the review. Security Innovation can assist you with the code review process. Correct all the discovered bugs and vulnerabilities. Have the management evaluate the review results and approve them before releasing the code.
