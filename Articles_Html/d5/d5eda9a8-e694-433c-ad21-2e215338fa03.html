Applies to

  • PCI DSS Requirement 4.1
  • PCI DSS Requirement 4.1.a

What to Do

Encrypt network traffic when sending cardholder data over open, public networks, such as the Internet.

Why

Encrypt the cardholder data when transmitting to prevent it from being read if intercepted by an attacker.

How

Identify if and how the cardholder data is transmitted over open, public networks, such as the Internet. Identify all the computers that are involved in the transmission. Choose a strong encryption solution to secure the transmission. Some acceptable encryption solutions are VPN and TLS:

  • VPN. VPN can be used to create an encrypted tunnel to send cardholder data. VPN can be used for multiple applications and network devices simultaneously. VPN is overall the most flexible solution.

  • TLS. TLS is best used for transmitting data between a web client and a web server.

Install and configure a strong encryption solution on each involved computer to secure the transmission.
