Applies to

  • PHP

What to Check For

Verify that sessions are destroyed when users log out.

Why

Invalidating sessions when users log out makes it harder to hijack them.

How to Check

To verify that sessions are destroyed when users log out:

  1. Identify logout functionality. Find code that is responsible for the logout functionality.

  2. Verify that logout code destroys user sessions. Examine logout code to make sure that it calls the session_destroy function to clear all session data and make sure that it calls setcookie to destroy the session ID cookie.&nbsp;For example:

    session_destroy();
    setcookie(session_name(), "", time() - 3600, "/");

How to Fix

To invalidate sessions when users log out:

  1. Identify logout functionality. Find code that is responsible for the logout functionality.

  2. Add code to invalidate sessions. Add the following code to the logout function to invalidate the session:

    session_destroy();
    setcookie(session_name(), "", time() - 3600, "/");
