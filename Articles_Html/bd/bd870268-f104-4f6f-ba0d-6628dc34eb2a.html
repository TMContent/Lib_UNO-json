Applies To

  • HTML5

What to Check For

Verify that WebSocket connections are protected using TLS.

Why

Just as TLS protects and encrypts the communication tunnel in HTTP, it can encrypt a WebSockets tunnel to protect against eavesdropping and man-in-the-middle attacks. (By default, the communication tunnel with the WebSocket protocol is not encrypted.)

How To Check

To verify that WebSocket connections are protected using TLS:

  1. Verify that the server is configured to use TLS. The exact details of configuring web servers to use TLS are different. Use documentation for your web server for exact instructions about how to determine whether the server is configured to use TLS. One simple and generic way is to connect to the server with a web browser using the https:// protocol and observe whether the connection is successful.
  2. Verify that the server certificate is valid. Connect to the server using the https:// protocol and use the functionality built into the browser to examine the certificate. Most modern web browsers will produce a warning if the certificate is not valid and will provide an easy way to make sure the certificate is valid.
  3. Review application code to make sure the wss:// protocol is used. Search application code for ws:// - there should be no such references, because only the wss:// protocol should be used. The wss:// protocol is secured with TLS.

How To Fix

The WebSockets protocol specification is built on HTTP. With that in mind:

  1. Enable security certificates and keys. Consult your web server documentation to ensure that your web site produces a valid certificate with HTTPS connections.
  2. Enable TLS on the WebSocket server. The configuration may vary depending upon the Websocket server implementation in use.
  3. Connect the client to the WebSocket server and ensure connections to the WebSocket are secure. When connecting to HTML5 web applications, ensure that client browsers connect with HTTPS. Note that the code accessing the WebSocket server should use the wss protocol, which is basically ws over TLS. As such, use wss://<url> instead of the typical ws://<url> in the HTML5 application code.

WebSocket Guidelines

  • Validate Data Coming from WebSockets
  • Check the Origin Header in the WebSocket Handshake
  • Use TLS to Encrypt WebSocket Connections
  • Check the Origin Attribute of WebSocket Messages
  • Use the Latest WebSocket RFC 6455 Standard
  • Do Not Tunnel TCP via WebSockets

WebSocket Checklist Items

  • Data Coming from WebSockets Is Validated
  • The Origin Header in WebSocket Handshake is Checked
  • TLS is Used to Encrypt WebSocket Connections
  • The Origin Attribute of WebSocket Messages Is Checked
  • Latest RFC 6455 Standard is Used
  • TCP Is Not Tunneled via WebSockets
