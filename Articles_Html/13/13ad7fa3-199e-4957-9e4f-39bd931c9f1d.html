Applies to

Scala and Play Framework 2.1.0+

What to Do

Protect session IDs set up via the use of cookies.

Why

Protecting session IDs helps protect user sessions from being hijacked.

How

To protect session IDs:

  1. Store session IDs in cookies.&nbsp;This is the default behavior in most Scala&nbsp;applications. Cookies have to be turned on for this to work.

  2. Set the "secure" flag on session cookies.&nbsp;This is the default behavior in most Scala&nbsp;&nbsp;applications. TLS has to be turned on for this to work.&nbsp;Enable the Secure flag on the cookie using “application.session.secure=true” in the “application.conf” file.

  3. Set the "HTTPOnly" flag on session cookies. Enable the HTTPOnly flag using “application.session.httpOnly=true” in the “application.conf” file.

  4. Do not disclose session IDs.&nbsp;Make sure that session IDs are not disclosed in URLs, logs, or error messages. Do not allow URL rewriting of session cookies.
