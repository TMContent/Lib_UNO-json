Applies to

  • PCI DSS Requirement 11.3

What to Do

Document penetration tests.

Why

Penetration tests should be documented to prove that they took place and to show what vulnerabilities may have already been found and fixed.

How

Perform the following actions to assure that penetration tests are performed and documented in compliance with PCI DSS:

  1. Choose a penetration testing team. Qualified professionals should perform penetration testing. It is acceptable to have the company staff perform the test if internal talent is readily available, but it is recommended to leave it to companies who specialize in security services. Security Innovation offers penetration-testing services.

  2. Perform penetration tests. Schedule yearly penetration tests. Perform network layer attacks, which include operating system and network protocol attacks. Perform application layer attacks. For web applications, have them tested for all the vulnerability types in the OWASP Top 10.

  3. Document fixed vulnerabilities. If vulnerabilities are found, fix them and perform another penetration test. Document all fixed vulnerabilities.

  4. Document the repeated tests. Continue testing until no more vulnerabilities are found. Document the results of each test.
