Applies To

  • Web Applications

What to Do

Require TLS for all private pages.

Why

TLS should be used to protect all pages that handle sensitive data, including all pages that use authentication or authorization features, to prevent unauthorized information disclosure.

How

The simplest approach for enabling TLS for all sensitive pages is to use TLS for the entire site. Enabling TLS for the entire site may require additional computing resources, so for performance reasons, it may be acceptable to enable TLS only for sensitive pages. To enable TLS only for sensitive pages:

  1. Identify sensitive pages. Make a list of all pages that handle sensitive information, including all pages that use authentication or authorization features.
  2. Enable TLS for each directory containing sensitive pages. The exact instructions for doing so are different for different web servers. Use the web server administration interface or edit the server configuration files to enable TLS for all folders containing sensitive pages.
  3. Redirect non-TLS requests to TLS-protected pages. The exact procedure for redirecting HTTP connections to HTTPS is different for different servers.
