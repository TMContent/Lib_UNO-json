Applies to

  • PCI DSS Requirement 9.7
  • PCI DSS Requirement 9.7.1
  • PCI DSS Requirement 9.7.2

What to Check For

Check that a media distribution policy is defined. Verify that sensitive media is classified as "confidential." Check that media distribution is logged. Verify that media is sent via secured courier or other delivery method that can be tracked.

Why

A media distribution policy should be defined to facilitate the making of decisions in a timely and secure manner when media has to be distributed. The policy should cover all types of media storage to ensure that the policy provides adequate guidance for all likely situations. Sensitive media should be classified to make it clear to the recipient that it should not be disclosed. Media distribution should be logged to help audit media distribution events if needed. Media should be sent via secured courier or other delivery method that can be tracked to assure that media reaches its recipient, to assist with locating it if it is somehow lost, and to have greater control of the delivery process by being able to monitor it and make calls.

How to Check

To verify that policy covers secure media distribution, perform the following steps:

  1. Check provisions for media distribution. Examine the storage media policy to make sure it includes provisions for media distribution.

  2. Check sensitive media classification requirements. Review the media policy to verify that it requires sensitive media to be classified as "confidential."

  3. Verify that media distribution is logged. Examine the media distribution log to verify that it exists.

How to Fix

Use the following steps to update policy with secure media distribution requirements:

  1. Identify media distribution requirements. Identify the possible scenarios which would require media containing sensitive information to be distributed, for example, an external audit.

  2. Document media distribution procedures. Write down the correct procedures for handling each scenario and add them to the media policy.

  3. Update the policy. Update the information security policy with the media distribution requirements. Add a point to the media policy that requires sensitive media to be classified as "confidential." Add a point to the media policy that requires keeping a log of distributed media. Add a point to the media policy that requires sending sensitive media via secured courier or some other shipping method that can be tracked. Attach the media policy to the information security policy.

  4. Keep a log. Start a log of distributed media that includes the following information: the name of the person authorizing distribution, the name of the intended recipient, the contents of the media, the date, and any other relevant information. Attach the media distribution log to the network documenttion.

  5. Comply with the policy. Follow the media distribution policy when distributing sensitive media.
