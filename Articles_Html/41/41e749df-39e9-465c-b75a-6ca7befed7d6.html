What to Do

Deploy a centralized log server.

Why

It may be possible to satisfy PCI DSS requirements without maintaining a centralized log server, but one is highly recommended because it can significantly reduce costs and complexity of satisfying the PCI DSS requirements and managing logs. For example, all logs are supposed to be promptly backed up, reviewed daily, and retained for at least one year to comply with PCI DSS - daunting tasks, if dozens of separate devices have to be checked. A single log server, however, which aggregates log entries from the network and presents them in a searchable index via a web interface, makes all of this much simpler. Some devices may not be compatible with the log server, in which case they may require special attention.

How

Add a point to the information security policy that requires all logs to be kept for at least a year and at least the last three months of logs to be available for immediate review. Choose a centralized log server solution. When choosing a centralized log server solution, it may be helpful to verify that it is explicitly designed to satisfy PCI DSS Requirement 10.2.2 - logging all activity by the administrator and other privileged users - because this functionality is difficult to implement otherwise. Install and configure a centralized log server by following the vendor's instructions. Configure all network devices that are compatible with the log server to send logs to the log server. Make a list of the incompatible devices and use this list to manually check and backup the logs on them daily. Consider replacing the incompatible devices with ones that are compatible with the log server.
