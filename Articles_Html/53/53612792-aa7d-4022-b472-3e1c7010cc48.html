What to Do

Set the httpOnlyAttribute to true on all authentication cookies to prevent client side scripts from accessing the cookie.

Why

Authentication cookies are only used on the server side to authenticate requests coming from a user; they're never used on the client.&nbsp; Preventing the client from accessing them via scripts means that if a script injection attack occurs on your site, the attacker will not be able to steal a user's cookies and emulate the user directly.&nbsp; Injection attacks still have the potential to completely compromise an account, but making authentication cookies unavailable does reduce their impact.

When

All systems which use authentication cookies should set their httpOnlyCookies attribute.

How

Set the httpOnlyCookies attribute on the authentication cookie.&nbsp; Internet Explorer 6 Service Pack 1 and later supports this attribute, which prevents client-side script from accessing the cookie from the document.cookie property.&nbsp; The System.Net.Cookie class in .NET Framework supports an HttpOnly property.&nbsp; You can set this property programmatically or by using the <httpCookies> element in your application's Web.config file. By default, the HttpOnly cookie property is not set. To set the HttpOnly property of the cookie, add the following entry in your application's Web.config file. 

<system.web>...
  <httpCookies httpOnlyCookies="true" requireSSL="false" topic="" />...
</system.web>

Note: &nbsp; &nbsp; Not all browsers support HttpOnly, and it is possible to circumvent it's protection in a number of cases.&nbsp; Thus, this should not be relied upon as an absolute security measure.

Problem Example

A web application has an existing cross-site scripting flaw, and does not set HttpOnly on its cookies.&nbsp; When a user is attacked, the attacker is able to easily retrieve the user's authentication and authorization cookies, allowing her to impersonate the user.

Solution Example

A web application has an existing cross-site scripting flaw, but sets HttpOnly on its cookies.&nbsp; When a user is attacked, the attacker is unable to easily retrieve the user's authentication and authorization cookies, preventing her&nbsp;from impersonating the user.
