Applies to

  • PCI DSS Requirement 12.5

What to Check For

Policy should define daily security procedures.

Why

Daily procedures should be clearly defined so that people can carry them out consistently and in a timely fashion.

How to Check

Examine the information security documentation to make sure it includes clear definitions of daily operating procedures that cover all applicable PCI DSS requirements. The following procedures should be defined:

  • Updating the anti-virus
  • Updating software
  • Identifying new vulnerabilities
  • Managing physical access
  • Backing up logs
  • Reviewing Logs
  • Managing the incident response plan
  • Managing encryption keys
  • Assigning unique user IDs
  • Managing user accounts
  • Handling visitors
  • Maintaining the visitor log
  • Secure media distribution
  • Authorization for moving data
  • Destroying media containing cardholder data
  • Performing background checks
  • Educating staff on security policies
  • Managing service providers

How to Fix

Document the procedures for each of the daily tasks that must be performed within your company and include them in your information security policy. The exact definitions should be specific to your company. Train the staff that corresponds to each procedure to ensure a consistent and timely execution of each task.
