What to Do

Do not put sensitive data into cookies, form fields or query strings.

Why

The preceding storage mechanisms are all well known and truly unsafe. None of the aforementioned storage types offer any protection against eavesdropping or tampering. They are also the traditional places that most attackers look for interesting data such as account credentials and credit card information.

When

This is appropriate whenever sensitive data needs to be sent between the browser and the web server.

How

If you need to access sensitive data for a user, keep it stored on the server and perform all of the state management there. The previously mentioned storage mechanisms are only appropriate for things like session IDs or non-sensitive client settings, such as how many result items to display in a table.
