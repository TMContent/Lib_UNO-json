Applies to

  • PCI DSS Requirement 7.1.2

What to Check For

The access privileges should be restricted to the least necessary to perform job responsibilities.

Why

The least privilege principle helps prevent unauthorized access.

How to Check

Verify that the data access policy follows the least privilege principles by performing the following steps:

  1. Review authorization documents. Check the access authorization documents in the network documentation to make sure they explicitly define access privileges.

  2. Check ACLs. Check the access control systems for a sample of users to make sure only authorized access is allowed. The authorization documents should follow the least privilege principle by only granting access necessary for the job function. Therefore access controls based on such authorization documents also follow the least privilege principle.

How to Fix

Use the following steps to implement the data access policy using the lease privilege principle:

  1. Identify job functions. Identify the job functions of each employee.

  2. Identify access requirements. Identify the data access required for each employee. Define the data access privileges for each employee based on job responsibilities.

  3. Get management approval. Have the management review and authorize the data-access privilege specifications.

  4. Configure ACLs. Once approved, implement the data access control systems.

  5. Update documentation. Attach the access authorization documents to the network documentation.
