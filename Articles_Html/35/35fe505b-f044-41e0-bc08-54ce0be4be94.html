What to Check For

Verify that TLS is used to protect Web Service connections.

Why

Protect Web Service connections using TLS to prevent man-in-the-middle attacks.

How to Check

To verify that TLS is used to protect Web Service connections:

  1. Make sure that valid certificates are present. Examine the certificates that are present on the server that is hosting the Web Services. Make sure that the certificates are valid for that server.
  2. Verify that TLS is configured on the server. Examine the configuration of the server that is hosting the Web Services and make sure that Web Services are setup to only be acceptable over TLS encrypted connections.
  3. Verify that client code connects to Web Services using TLS. Examine the client-side code of the application that uses the Web Services. Make sure that the application only uses TLS protected connections to connect to the Web Services.

How to Fix

To use TLS to protect Web Service connections:

  1. Get valid certificates. Provision valid certificates for the server hosting the Web Services.
  2. Set up TLS on the server. Configure the server to use TLS for the Web Services. Don't allow unencrypted connections to the Web Services â€“ don't bind Web Services to ports that don't use TLS.
  3. Update client code to use TLS. Change client code to connect to the Web Services using the port that uses TLS.
