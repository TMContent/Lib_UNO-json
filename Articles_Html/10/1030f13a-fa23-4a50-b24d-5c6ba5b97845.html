Applies To

  • ASP.NET 4.0

What to Check For

Verify that a new or blank session ID is assigned on logout.

Why

Reset the session ID of the user upon logout to prevent people using the same computer from being able to retrieve the session ID after the user has logged out.

How To Check

To verify that a new or blank session ID is assigned on logout:

  1. Identify logout functionality. Find the function or functions that are called to log the user out.

  2. Verify that the session state is cleared on logout. Review the logout function to make sure it clears the session state on the server by using code such as the following:

    Session.Abandon();

  3. Verify that the session ID cookie is reset. Review the logout function to make sure it resets the session ID cookie by using code such as the following:

    Response.Cookies.Add(new HttpCookie("ASP.NET_SessionId", ""));

How To Fix

To assign a blank session ID on logout:

  1. Identify logout functionality. Find the function or functions that are called to log the user out.

  2. Clear the session state on logout. Add the following code to the logout function to clear the session state on the server:

    Session.Abandon();

  3. Reset the session ID cookie. Add the following code to the logout function to set the session ID to null:

    Response.Cookies.Add(new HttpCookie("ASP.NET_SessionId", ""));
