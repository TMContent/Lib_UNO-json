Applies to

  • PCI DSS Requirement 8

What to Do

Formulate the PCI DSS compliant user authentication and password policies and attach them to the information security policy.

How

Perform the following actions to formulate a PCI DSS compliant user authentication policy:

  1. Require unique user IDs. Add a point that requires each authorized user to be assigned a unique user ID.

  2. Require user authentication. Add a point that requires all users to be authenticated before being granted access.

  3. Require two-factor authentication for remote access. Add a point that requires remote access to use two-factor authentication.

  4. Require passwords to be secured. Add a point that requires passwords to be encrypted or hashed when stored or sent.

  5. Require data access authorization. Add a point that requires access to be granted based on the access authorization forms. The access privileges should be explicitly stated by the authorization forms and implemented accordingly.

  6. Define password reset procedures. Add a point that requires user identity to be verified before performing a requested password reset. One way to verify the user's identity is by having a meeting in person between the user and the administrator performing the reset.

  7. Define handling of first-time passwords. Add a point that requires first-time passwords for new accounts to be unique and to be changed after first use.

  8. Define handling of terminated employee accounts. Add a point that requires the user account to be removed when an employee is terminated.

  9. Define handling of inactive accounts. Add a point that requires accounts that are not active for 90 days to be disabled or removed.

  10. Define handling of vendor remote access accounts. Add a point that requires accounts used by vendors for remote access to be disabled when not in use.

  11. Require user awareness of information security policies. Add a point that requires users to be familiar with user authentication and password policies and procedures. Users should know:

    • How to have their password reset
    • How to change their password
    • How often they have to change their passwords
    • The password strength requirements

  12. Prohibit shared user accounts. Add a point that requires there to be no shared or generic IDs. Administrators should not create shared IDs, even if asked to.

  13. Require database authentication. Add a point that requires all database access to be authenticated. Other processes or individual users should not be able to use the authentication credentials used by any database application to access the database(s).

  14. Require passwords to expire after 90 days. Add a point that requires passwords to be changed every 90 days.

  15. Define a password length requirement. Add a point that requires passwords to be at least 7 characters long.

  16. Define a password complexity requirement. Add a point that requires passwords to have both letters and numbers in them.

  17. Forbid reusing previous passwords. Add a point that requires new passwords to not be the same as any of the four previous passwords for the same user.

  18. Require account lockouts. Add a point that requires user accounts to be locked after a maximum of six failed login attempts. 

  19. Define account lockout duration. Add a point that requires user accounts to be locked for at least 30 minutes or until an administrator unlocks it. It is acceptable for an administrator to unlock the account before the lockout time is over.

  20. Require idle session time-outs. Add a point that requires interactive sessions that are idle for 15 minutes to be locked automatically and require the user's password to unlock.

  21. Update documentation. Attach the user authentication policy and the password policy to the information security policy. Instruct the responsible personnel to follow the stated policies. Implement the password policy by using Group Policy Objects in a Microsoft Active Directory environment. Please refer to the guidance article linked at the bottom of this page for more information about using GPOs to implement a PCI DSS compliant password policy.
