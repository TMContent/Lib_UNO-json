Applies to

Scala and Play Framework 2.1.0+

What to Check For

Verify that session IDs are protected.

Why

Protecting session IDs helps protect user sessions from being hijacked.

How to Check

To verify that session IDs are protected:

  1. Log in to the web site. Authenticate as a user to the web application.

  2. Examine the session cookie. Examine the cookies stored on the browser. Find the session cookie. Verify that it has the "secure" and "HTTPOnly" flags set to true.

  3. Verify that session IDs are not disclosed. Examine logs, error messages, and hyperlinks to make sure that session IDs are not disclosed in them.

How to Fix

To protect session IDs:

  1. Store session IDs in cookies. This is the default behavior in most Scala applications. Cookies have to be turned on for this to work.

  2. Set the "secure" flag on session cookies. This is the default behavior in most Scala applications. SSL has to be turned on for this to work. Enable the Secure flag on the cookie using “application.session.secure=true” in the “application.conf” file.

  3. Set the "HTTPOnly" flag on session cookies. Enable the HTTPOnly flag using “application.session.httpOnly=true” in the “application.conf” file.

  4. Do not disclose session IDs. Make sure that session IDs are not disclosed in URLs, logs, or error messages. Do not allow URL rewriting of session cookies.
