What to Check For

Make sure that users are not allowed to execute custom database queries.

Why

Prevent users from executing custom queries to prevent this feature from being abused to bypass access controls or use the database system in an unintended and malicious manner.

How to Check

To verify that users are not allowed to execute custom database queries:

  1. Find code that executes database queries. Make a spreadsheet of all the code that executes database queries.
  2. Identify code that allows users to execute custom queries. Custom queries are those where the syntax of the query can be manipulated by the user, instead of just letting the user to change query parameters. Review all code that executes database queries to find features that allow users to execute custom queries.

How to Fix

To prevent application's users from executing custom queries:

  1. Find code that executes database queries. Make a spreadsheet of all the code that executes database queries.
  2. Identify code that allows users to execute custom queries. Review all code that executes database queries to find features that allow users to execute custom queries. Custom queries are those where the syntax of the query can be manipulated by the user, instead of just letting the user to change query parameters.
  3. Remove code that allows users to execute custom queries. If any code or features that allow users to manipulate database query syntax are present, remove them.
