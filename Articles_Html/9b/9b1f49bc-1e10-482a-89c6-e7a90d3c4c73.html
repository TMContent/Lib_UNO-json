Applies to

  • PCI DSS Requirement 6.3

What to Do

Test all changes, including patches, before deploying them. Test each change for validation of input, error handling, secure cryptographic storage, secure communications, and proper role-based access control.

Why

Testing helps ensure that changes do not introduce new vulnerabilities or break functionality.

How

Create a testing environment that simulates the environment where changes are to be applied. Apply changes in the testing environment. Test each change for validation of all input, error handling, secure cryptographic storage, secure communications, and proper role-based access control. Document the results of the tests. If the tests are successful, apply the changes to the production environment.
