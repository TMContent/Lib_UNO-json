What to Check For

Ensure that the confidentiality of session cookies is protected.

Why

Session cookies are used by an application to identify the session associated with a particular user. Protecting session cookies can help prevent an attacker from hijacking the session and using the application as a legitimate user.

How to Check

Use the following steps to verify that your application's session cookies are handled securely:

  1. Verify that session identifiers cannot be guessed. Ensure that your application's session identifiers cannot be guessed by an attacker. Verify that your application's session IDs meet the following requirements:

    • Length: Verify that your application uses long session identifiers (e.g. string of length 32).

    • Triviality: Verify that your application uses randomly generated session identifiers.

    • Lifetime: Verify that your application uses timeout attribute to expire sessions. 

      <sessionState timeout="20"... >   
      ...</sessionState>

    • Recycling: Verify that your application does not reuse previously used sessions.

      <sessionState regenerateExpiredSessionId="false"... >   
      ...</sessionState>

  2. Verify that users can invalidate their own sessions. Ensure that your application's uses Session.Abandon to allow its users to invalidate their own sessions.

  3. Verify that session cookies are sent over encrypted channels. Ensure your application transmits session cookies over encrypted channels. Although this is not required for non-sensitive transactions, it is mandatory for any transactions that are sensitive in nature.

    <authentication mode="Forms">  
    <forms requireSSL="true" ... />
    </authentication> 
    ... 
    <roleManager cookieRequireSSL="true" ... >  
    ...
    </roleManager>
    If session cookies cannot be sent over an encrypted channel, ensure that cookieSlidingExpiration is set to false.
    <authentication mode="Forms">  
    <forms slidingExpiration="false" ... />
    </authentication> 
    ... 
    <roleManager cookieSlidingExpiration="false" ... >  
    ...
    </roleManager>

  4. Verify that sensitive cookies are marked httpOnly. Ensure that cookies are marked as httpOnly. 

    <system.web>  
    ...   
    <httpCookies httpOnlyCookies="true" ... /> 
    ...
    </system.web>

Problem Example

A web application uses SSL to provide a secure connection for user data, generates strong, new session IDs for each user, and times out and terminates sessions appropriately. However, it is susceptible to cross-site scripting flaws, and despite having a user base which mostly uses IE, does not set httpOnlyCookies on its cookies. The web.config file for this application looks like this:

<system.web>
...  
<httpCookies httpOnlyCookies="false" requireSSL="true" topic="" />
...
</system.web>
