What to Check For

Penetration testing results should be documented and the identified vulnerabilities should be fixed.

Why

Penetration testing helps discover vulnerabilities before someone else does.

How to Check

Check project documentation for information about penetration testing. Penetration testing results should be documented and the identified vulnerabilities should be fixed. There are many different methodologies for carrying out penetration tests. Procedures for carrying out penetration tests depend on specific aspects of the application or system being tested. One thing is common to all penetration tests - they should be documented. Verify that this documentation is included in project documentation to make sure that penetration testing has been done.

How to Fix

One option is to contract an information security consulting company to perform the penetration test. One methodology for carrying out penetration tests involves examining the application environment for common vulnerabilities, using automated tools to search for low hanging fruit, then manually checking the application for the most dangerous and common vulnerability types and attack vectors.
