Description

Resource-based authorization sets permissions on the resource itself. For instance you would set an ACL on a Windows resource and then use the identity of the original caller to determine access rights to the resource. If you use resource-based authorization in WCF you&#8217;ll need to impersonate the original caller through the application layer (e.g. ASP.NET application), through the WCF service layer and to the business logic code that is accessing the file resource.

To use resource-based authorization you need to sets permissions on the resource itself by setting an ACL and then impersonating the original caller.

The following code impersonates a specific (fixed) identity.

using System.Security.Principal;
&#8230;
WindowsIdentity wi = new WindowsIdentity(&#8220;userName&#64;fullyqualifieddomainName&#8221;);
WindowsImpersonationContext ctx = null;
try
&#123;
  ctx = wi.Impersonate();
  // Thread is now impersonating you can access resource needed&#8230;
&#125;
catch
&#123;
  // Prevent exceptions propagating.
&#125;
finally
&#123;
  // Ensure impersonation is reverted
  ctx.Undo();
&#125;
