Applies to

  • PCI DSS Requirement 6.3

What to Check For

All patches should be tested before being deployed. Each change should be tested for validation of all input, error handling, secure cryptographic storage,  secure communications, and proper role-based access control.

Why

Testing helps ensure that patches do not introduce new vulnerabilities or break functionality.

How to Check

Review the change control documentation to make sure it contains descriptions of the testing results for each change, including patches. Review the change control documentation to make sure that each change is tested for validation of all input, error handling, secure cryptographic storage, secure communications, and proper role-based access control.

How to Fix

Create a testing environment that simulates the environment where patches are to be applied. Install patches in the testing environment. Test each patch for validation of all input, error handling, secure cryptographic storage, secure communications, and proper role-based access control. Document the results of the tests. If the tests are successful, apply the patches in the production environment.
