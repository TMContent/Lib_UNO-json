Applies to

  • PCI DSS Requirement 6.3

What to Check For

Verify that code&nbsp;is reviewed by someone other than its author,&nbsp;developed according to secure coding guidelines, corrections are implemented before the&nbsp;release, and code review results&nbsp;are approved by management before&nbsp;the release.

Why

Changes to application code should be reviewed to ensure that they do not introduce vulnerabilities.

How to Check

Review the information security policy and the software develoment process&nbsp;to make sure that code&nbsp;is reviewed by someone other than its author,&nbsp;developed according to secure coding guidelines, corrections&nbsp;are implemented before the&nbsp;release, and code review results should be approved by management before the release. Interview the development staff to make sure that they comply with the requirements in the information security policy when reviewing code changes.

How to Fix

Add a requirement to the information security policy that code changes should&nbsp;reviewed by someone other than their author(s). Add steps to the software development process for someone other than the code's author review the code. It is a good idea to have a dedicated testing team in the development environment. Add a requirement to the information security policy for code to be developed according to secure coding principles. Integrate security engineering techniques with your software development process. Information about security engineering techniques may be found in TeamMentor in the "Security Engineering" folder. Add a requirement to the information security policy for corrections to be implemented before release. Add steps to the software development process to fix any errors found in the code during review to be fixed before the release. Add a requirement to the information security policy for&nbsp;code review results to be approved by management before release. Add this requirement to the software development process.
