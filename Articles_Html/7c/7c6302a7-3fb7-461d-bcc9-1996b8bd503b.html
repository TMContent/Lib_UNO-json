What to Check For

All system clocks should be synchronized.

Why

System clocks should be synchronized by internal and external sources so that log time stamps are accurate. This allows logs from multiple sources to be correlated. External time sources may become temporarily unavailable, so at least one internal time source should also be maintained.

How to Check

Use the following steps to check if system clocks are synchronized in compliance with PCI DSS:

  1. Examine documentation. Examine the network documentation to make sure it includes procedures for synchronizing system clocks on the network and configuring time related settings.

  2. Verify that NTP is used to synchronize system clocks on the network. A functional NTP (Network Time Protocol) implementation includes an NTP server and NTP clients deployed on host systems. Windows has an NTP client built-in. Windows Server has an NTP server and an NTP client built-in. Group Policy Objects may be used to tune the accuracy of the Windows NTP client.

  3. Verify that there are both an internal and external time sources. It is recommended to have one dedicated timeserver, which gets time from an external source, while Windows Server domain controllers are configured to get time from the internal timeserver. Workstations in the domain and other devices should get time information from the Windows Server domain controllers.

  4. Make sure the external time source(s) are explicitly defined. The recommended time sources are Stratum 1 servers. For a list of Stratum 1 servers, please see the link at the bottom of this page.

How to Fix

Perform the following actions to synchronize system clocks in comliance with PCI DSS:

  1. Install a dedicated NTP server. Configure domain controllers to get time information from the internal NTP server. Configure the workstations and network devices to get time from the internal NTP server.

  2. Configure the time servers. Choose a Stratum 1 timeserver. Configure the dedicated NTP server to use the external server as a source of time information. Inform the responsible staff that this is the external timeserver to use.

  3. Update documentation. Document your configuration and include this documentation in the network documentation as the "Time Synchronization Procedures."
