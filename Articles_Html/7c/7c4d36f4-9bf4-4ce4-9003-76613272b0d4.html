Applies to

  • PCI DSS Requirement 11.3

What to Check For

Penetration tests should be performed at least once a year and after any significant changes to the network.

Why

Penetration tests may expose security vulnerabilities and misconfigurations. Penetration tests exercise the response capabilities of the company to verify that they are sufficient for handling an actual attack.

How to Check

Use the following steps to check if penetration tests are performed regularly:

  1. Review policy. Examine the information security policy to verify that it requires:

    • Penetration tests to be performed at least once a year and after any significant changes to the network
    • All found vulnerabilities to be fixed
    • Tests to be repeated until no more vulnerabilities are found

  2. Review Documentation. Examine the results of the previous penetration tests to verify that they have been performed regularly and the discovered vulnerabilities were corrected.

How to Fix

Perform the following tasks to assure that penetration tests are performed regularly:

  1. Update policy. Add requirements to the information security policy to

    • Conduct penetration tests at least once a year and after any significant changes to the network
    • Fix all found vulnerabilities
    • Repeat testing until no more vulnerabilities are found

  2. Schedule penetration tests. Hire a qualified company to perform penetration tests. Perform penetration tests regularly, document the results, and attach them to the security documentation.
