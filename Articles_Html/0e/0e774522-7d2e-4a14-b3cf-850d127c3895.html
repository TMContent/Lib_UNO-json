Applies to

Scala and Play Framework 2.1.0+

What to Do

Use only FIPS 140-2 compliant encryption algorithms with TLS.

In practice, this means using TLS. SSL 3.0 and earlier versions have known vulnerabilities and are not FIPS 140-2 compliant.

Why

Using strong encryption algorithms with TLS helps reduce the risk of successful eavesdropping attacks and assures regulatory compliance.

When

Whenever possible, use only FIPS 140-2 compliant encryption algorithms with TLS.

How

To use strong TLS algorithms, configure the web application server to use TLS 1.2 and not SSL 3.0 or any earlier version of SSL. The exact instructions for doing so are different for different platforms. Please, refer to your web application server's documentation or technical support for detailed instructions on how to configure the server to use TLS.
