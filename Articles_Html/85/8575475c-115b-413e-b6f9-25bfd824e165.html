Applies to

  • PCI DSS Requirement 2.2.1.a

What to Do

Implement only one primary function per server and disable all other network services.

Why

The fewer network services that run on a server, the fewer potential vulnerabilities there are for an attacker to exploit.

When

The primary function should be implemented and all other network services should be disabled when a server is being configured.

How

Perform the following tasks to implement only one function per server:

  1. Implement the main function of the server. Identify the primary function of the server. Install and configure features and applications necessary to implement the primary function of the server.

  2. Remove all other services and functionality. Identify all the network services running on the server. If possible, remove or disable all the network services that are not necessary for implementing the primary stated function. Some services may be a part of the Operating System or firmware and difficult or impractical to remove. If an unnecessary network service cannot be removed, disable it.If an unnecessary network service cannot be removed or disabled, document this network service and include an explanation of why it is running, i.e., “It cannot be removed in an expedient manner.” Consider implementing a mitigation measure to compensate for an unneeded, embedded network service, such as a firewall or a white list. The exact mitigation measure depends on the exact configuration and requires an expert opinion to be implemented correctly. Document the mitigation measures, if any.

  3. Update documentation. Document the primary function of the server and the features and applications used to implement it.
