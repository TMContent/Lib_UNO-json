What to Check For

Verify that file and folder names are not used for security decisions.

Why

When a filename/path is used in a condition, an attacker may be able to trick the application to execute an unintended code branch. To prevent this, avoid using filenames/paths in conditional statements, except for validation or error handling.

How to Check

Identify all filename and path inputs. Search for conditional statements that use these filenames or paths. Verify that filenames and paths are only used in conditional statements for validation or error handling.

How to Fix

Rewrite code using ACLs to enforce filesystem security policies. Depending on the application, this might be a lot of work, but it needs to be done to assure security.
