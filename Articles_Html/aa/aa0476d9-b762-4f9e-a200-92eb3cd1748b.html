What to Check For

Verify that the latest security updates are installed.

Why

Software patches should be installed to protect from known and actively exploited vulnerabilities.

How to Check

To verify that the latest security updates are installed:

  1. Locate patching documentation. Find documentation for the patching process and records of previously installed patches.

  2. Identify software packages that may need updating. Make a list of all software packages that may need updating, including operating systems, servers, user applications, and code libraries.

  3. Identify update sources. For each software package that may need updating, identify the source of vendor-supplied updates. Make a list of all sources for software updates with notes on the latest applicable patches.

  4. Verify that the latest patches have been installed. Examine a representative sample of systems to verify that the latest vendor patches have been applied. It is possible to use automated scanning tools for this task.

How to Fix

To keep up with security updates:

  1. Identify software packages that may need updating. Make a list of all software packages that may need updating, including operating systems, servers, user applications, and code libraries.

  2. Identify update sources. For each software package that may need updating, identify the source of vendor-supplied updates. Identify the schedule used by the vendor to release updates; typically vendors release patches on a set schedule or whenever new vulnerabilities are discovered. Make a list of all sources for software updates with notes on when new updates may be available.

  3. Establish patching documentation. Start a document to keep track of installed patches and any issues that may crop up. Every now and again, a vendor patch may break something or fail to install correctly. Document these situations and the solutions. It is important to record the time when patches are installed.

  4. Establish a process for installing updates. Schedule periodic checks for new security updates. If possible, time these checks to match the vendor release schedules. Go through the list of vendor sources, check for new patches, download and install them, and document the process.

  5. Delegate responsibility. Once the process for installing updates and the documentation are established, delegate responsibility for these tasks to a system administrator.
