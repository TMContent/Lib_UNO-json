Applies to

  • PCI DSS Requirement 6.2

What to Do

Install the latest security updates supplied by the vendor.

Why

Installing vendor-supplied security patches fixes known security vulnerabilities.

When

The latest vendor-supplied security patches should be installed within a month of being released.

How

Perform the following actions to install the latest security updates:

  1. Install updates for Microsoft software. Windows Server Update Services may be deployed to update most common Microsoft software in compliance with PCI DSS. Installing and configuring WSUS is an involved process that is thoroughly documented by Microsoft on its TechNet web site linked at the bottom of this article.  

  2. Install updates from other vendors. Usually applying vendor-supplied patches means going to the vendor's web site, downloading the latest patches, and installing them, which is often a matter of running a packaged executable on the target host. The procedures for applying security updates vary by vendor, though, as some vendors package their updates differently, such as compressed archives or source code patches. Please refer to your vendor's documentation or technical support for detailed instructions on installing their security updates.

  3. Schedule regular updates. Add a point to the information security policy that requires critical new patches to be installed at least once a month. Install new security patches at least once a month.
