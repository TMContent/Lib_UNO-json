Description

Exception Information Disclosure is a type of vulnerability where an application error message discloses information that give an advantage to the attacker. Attackers can use application error information to improve their ability to attack the application.

Almost all method calls in computer programs provide some feedback to the calling method about the status of the operation. This feedback could be a function return value, an exception which was thrown by the called method, or an error string. Based on this feedback, an applications decides if it should tell the user about a significant result (usually success or failure). When an error occurs, some applications choose to echo the internal error messages to the user. Though this feature is helpful in debugging a problem, it could lead to an information disclosure issue. An attacker could use technical information disclosed in error messages to gain a better understanding about the back-end processing of the product and craft more malicious attacks against it.

Countermeasures

To prevent this vulnerability, do not disclose valuable information in error messages, avoid using custom error pages to denote error states, use structured exception handling, and use a global exception handler for unhandled exceptions.

Application Check

To check for adequate protection against this vulnerability, ensure that valuable information is not disclosed in error messages, custom error pages are not used to denote error states, structured exception handling is used, and global error handlers are used to catch unhandled exceptions.

Valuable information is not disclosed in error messages:

  • Valuable Information Is Not Disclosed in Error Messages

Custom error pages are not used to denote error states:

  • Custom Error Pages Are Not Used to Denote Error States
