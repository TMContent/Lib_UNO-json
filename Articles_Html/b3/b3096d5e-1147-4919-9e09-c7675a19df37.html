Applies to

  • PCI DSS Requirement 2.1
  • PCI DSS Requirement 2.1.1.b
  • PCI DSS Requirement 2.1.1.c

What to Do

Change the vendor default passwords on network devices.

Why

Vendor default passwords are public knowledge and, therefore, insecure. To secure a network configuration, the vendor default settings of network components must be changed.

When

Change the vendor defaults before deploying new system components.

How

Change the vendor defaults for security settings&nbsp;before deploying new system components by performing the following tasks:

  1. Secure the administrator account. Log into the new network device as an administrator using the vendor default password. Find the page or command that changes the administrator password, using vendor manuals, the Internet, or intuition. Change the administrator password to a password that is consistent with the company password policy. If possible, rename the administrator account.

  2. Secure or remove other default accounts. Identify all other default usernames and passwords by using the vendor’s manuals, commands on the network device, or the Internet; there may be none. If there are other default user accounts, change the passwords for them, using the same procedure used for changing the administrator password. If there are default user accounts that are not needed, remove them.The exact procedure to remove a user account varies by network device. Please refer to the vendor’s manual or technical support for detailed instructions on how to remove a user account on your network device.

  3. Secure SNMP settings. If the network device supports SNMP, verify that SNMP is necessary for network operations. If SNMP is not necessary, disable it. If SNMP is necessary, change the community string from the vendor default. The exact procedure for changing the SNMP settings varies by network device. Please refer to the vendor’s manual or technical support for detailed instructions on how to configure the&nbsp;SNMP settings on your network device.
