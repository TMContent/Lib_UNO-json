Applies to

  • Microsoft® Windows Communication Foundation (WCF) 3.5
  • Microsoft Visual Studio® 2008

What to Do

Use IIS to host your WCF service because it provides a large number of features for efficient service management and scalability.

Why

By using IIS as your WCF service host you can take full advantage of IIS features, such as process recycling, idle shutdown, process health monitoring, and message-based activation. .

How

IIS 6.0 only supports bindings over http so if you need to use TCP, MSMQ or named pipes you should host in a Windows service instead.  IIS 7.0 supports all the commonly used transport protocols such as HTTP, TCP, MSMQ, and named pipes.
