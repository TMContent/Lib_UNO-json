What to Do

Define time synchronization procedures. Implement NTP technology. Configure the time sources for the NTP implementation.

Why

System clocks should be synchronized so that log time stamps are accurate and the logs from multiple sources can be correlated.

How

Perform the following actions to synchronize system clocks in compliance with PCI DSS:

  1. Implement NTP technology. Install a dedicated NTP server. Configure it to get time information from a Stratum 1 public server. Configure domain controllers to get time information from the internal NTP server. Configure the workstations and network devices to get time from the internal NTP server. For details about configuring the NTP implementation in the Windows Operating Systems, please refer to the TechNet article linked to at the bottom of this page.

  2. Configure internal time sources. The dedicated NTP server may be used as an internal time source to satisfy this requirement.

  3. Configure external time sources. Choose a Stratum 1 timeserver. Document your choice in the network documentation. Inform the staff responsible that this is the external timeserver to use. Configure the dedicated NTP server to use the external server as a source of time information.

  4. Define time synchronization procedures. Once NTP is implemented, document your configuration and include this documentation in the network documentation as the "Time Synchronization Procedures."
