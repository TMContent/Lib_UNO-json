Applies to

  • PCI DSS Requirement 7.2.3

What to Check For

Access control systems should have a "deny-all" default setting.

Why

Access control systems should have a "deny-all" default to ensure that all access is explicitly authorized.

How to Check

Use the following steps to verify that all the access control system implementations have a "deny-all" default:

  1. Verify the ACL defaults on UNIX-type systems. Issue the "umask" command for each authorized user or check the login scripts for each user. The last two digits of the number returned by the "umask" command or supplied as an argument to the umask command in the login scripts should be 77.

  2. Verify the ACL defaults on Windows systems. On Windows, access is denied by default unless explicitly granted. Windows does grant access privileges generously because objects inherit permissions from their containing folder or drive. It is prudent to make sure that the inherited permissions do not exceed the defined privileges. To verify the access control settings on Windows: right click on each drive, select "Properties", go to the "Security" tab, and verify that only the needed access privileges are granted.

  3. Verify the ACL defaults on other system-types. Most operating systems implement either the UNIX or the Windows type ACL system. Some network devices may have to be configured using a web interface. Use intuition or follow vendor's instructions to check the ACL defaults on such devices.

How to Fix

Perform the following tasks to enable the "deny-all" default setting on access control systems:

  1. Enable the "deny-all" ACL default setting on UNIX-type systems. add the "umask 077" command to the login scripts for all users.

  2. Enable the "deny-all" ACL default setting on Windows systems. Delete any excessive permissions by using the controls in the “Security” tab: right click on each drive, select "Properties", go to the "Security" tab, and delete all access permissions that are not explitictly authorized.

  3. Enable the "deny-all" ACL default settings on other system-types. Most operating systems implement either the UNIX or the Windows type ACL system. Some network devices may have to be configured using a web interface. Follow vendor's instructions to configure ACL defaults on such devices.
