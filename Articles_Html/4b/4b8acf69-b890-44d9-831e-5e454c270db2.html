Applies To

  • Java

What to Check For

Verify that sensitive cookies are not made persistent.

Why

Allow sensitive cookies to expire when the user closes the browser to prevent attackers from stealing these cookies from the user's system. An attacker would have to gain unauthorized access to the user's system to be able to recover persistent cookies, but this does happen in practice, for example as a result of malware infection. Botnet operators often steal cookies stored on users' computers to hijack those users' accounts on sites that store session information in persistent cookies. If sensitive cookies are not persistent, then they will not be stored on the system and it will be more difficult for an attacker to recover them.

How to Check

To verify that sensitive cookies are not made persistent:

  1. Identify sensitive cookies. Cookies are represented by objects of Cookie class in Java, so you can search for the keyword "Cookie" to help identify all cookies used by the application. Once all the cookies have been identified, it should be simple to determine which ones contain sensitive data and which ones do not. Pay particular attention to cookies that contain session identifiers. Create a spreadsheet to list all the cookies that contain sensitive data in your application.
  2. Verify that sensitive cookies are not made persistent. Examine each piece of code that handles each sensitive cookie. Make sure that the expiration time for sensitive cookies is never set.
