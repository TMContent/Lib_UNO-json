Applies to

  • PCI DSS Requirement 3.4.1.c

What to Check For

Cardholder data should be encrypted when stored on removable media. Some disk encryption applications may not work on removable media, so this data may need to be encrypted separately.

Why

Encrypt the cardholder data on removable media to make sure that if the media falls into the possession of an attacker, the cardholder data will still be secure.

How to Check

Identify all removable storage media that has cardholder data on it. Attempt to access the data on such media without an encryption key. Check the information security policy to make sure it includes a requirement for encrypting the cardholder data stored on removable media.

How to Fix

Identify the types of removable storage media that are used to store cardholder data. Identify encryption applications that are compatible with each type of removable storage media. Install and configure the corresponding encryption application for securing the cardholder data on all removable media. Add a point to the information security policy that requires encrypting all cardholder data stored on removable media.
