Applies To

  • ASP.NET 4.0

What to Check For

Verify that strong encryption keys are used.

Why

Using long keys makes it more difficult to break encryption.

How To Check

To verify that strong encryption keys are used:

  1. Identify encryption functionality. Review code to find all functions that encrypt and decrypt data. Search for the string "System.Security.Cryptography" to find code that uses .NET cryptographic APIs.

  2. Verify that AES keys are strong. Search for the string "AesCryptoServiceProvider" to find code that uses AES encryption. Review code that generates encryption keys to make sure that keys are 256-bit and generated securely. An AesCryptoServiceProvider object should be used to generate strong AES keys.

  3. Verify that RSA keys are strong. Search for the string "RSACryptoServiceProvider" to find code that uses RSA. Review code that generates encryption keys to make sure that keys are 2048-bit and generated securely. The RSACryptoServiceProvider constructor generates RSA keys; make sure that when the constructor is used to generate keys, the dwKeySize argument is 2048 or more.

How To Fix

To use strong encryption keys:

  1. Generate strong AES keys. Set the KeySize property of a System.Security.Cryptography.AesCryptoServiceProvider object to 256. Use the GenerateKey method of the System.Security.Cryptography.AesCryptoServiceProvider object to generate an AES key.

  2. Export and import AES keys. Use the Key property of an AesCryptoServiceProvider object to retrieve or to set the AES key.

  3. Generate strong RSA keys. Initialize a RSACryptoServiceProvider object with the dwKeySize argument to the constructor being 2048 or greater.

  4. Export and import RSA keys. Use the ExportParameters method of the RSACryptoServiceProvider object to export the RSA key as XML. Use the FromXmlString method to load the RSA key from XML.
