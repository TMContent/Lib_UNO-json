Applies To

  • PHP

What to Check For

Make sure that dangerous functions are disabled by default.

Why

Dangerous functions should be disabled by default so that attackers do not abuse them.

How to Check

To make sure that dangerous functions are disabled by default:

  1. Identify code that implements dangerous functions. Determine which application features implemented by your application might be dangerous. Some examples are: file uploads, code execution, direct database access, and data deletion. Make a spreadsheet about all code that implements dangerous features.
  2. Verify that dangerous functions are configurable. Examine the code that implements the dangerous functions. Make sure that this code checks configuration settings before running, and that dangerous functions are only executed if they are explicitly enabled.
  3. Verify that dangerous functions are disabled by default. Examine the code that implements the dangerous functions. Make sure the default setting is for the dangerous functions not to run. If there are no explicit settings to enable dangerous functions, they should not run. The default settings that are present immediately after the application is installed should have dangerous functions disabled.
  4. Verify that dangerous functions are documented. Examine documentation to make sure that all dangerous functions are documented, their security impact is explained, and the procedures to enable them are described.

How to Fix

To disable dangerous functions by default:

  1. Identify code that implements dangerous functions. Determine which application features implemented by your application might be dangerous. Some examples are: file uploads, code execution, direct database access, and data deletion. Make a spreadsheet about all code that implements dangerous features.
  2. Make dangerous functions configurable. Modify each piece of code that implements a dangerous function to check application settings to see whether the user has explicitly enabled the dangerous feature. Only allow the dangerous features to be executed if the user has explicitly enabled them.
  3. Make dangerous functions disabled by default. For each dangerous feature, write code so that the feature can only be executed if it has been explicitly enabled by the user.
  4. Document the dangerous functions. Add information about all the dangerous features to the documentation. Explain why each feature is dangerous. Describe how to configure each feature.
