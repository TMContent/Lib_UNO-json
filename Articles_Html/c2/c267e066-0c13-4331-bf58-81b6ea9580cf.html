What to Do

Formulate a data access control policy.

Why

A data access control policy defines data access privileges and is useful for implementing access control systems correctly.

When

Formulate a data access control policy before assigning access privileges.

How

Perform the following tasks to formulate the data access control policy in compliance with PCI DSS:

  1. Define the data control policy. Express each requirement as a point of the data access control policy. For example, the least privilege requirement may be written as "Each user shall have the least privileges necessary to carry out his job functions." The data control policy should include the following requirements:

    • Principle of least privilege to be used to define access privileges for each user
    • Privileges to be defined based on each user's job functions
    • An access authorization form to be created for each user
    • Automated access control systems to be implemented

  2. Data Access Authorization Documents. Identify job functions of each employee. Identify data access required for each employee. Define the data access privileges for each employee based on job responsibilities. Prepare access authorization documents for each user. These documents may be digital or on paper. Have management review and authorize the data access privilege specifications. Once approved, implement the data access control systems.

  3. Update documentation. Attach the access control policy to the information security policy. Attach the access authorization documents to the network documentation.
