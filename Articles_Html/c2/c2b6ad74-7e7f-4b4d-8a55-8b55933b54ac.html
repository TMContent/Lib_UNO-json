Applies to

  • PHP

What to Check For

Verify that the "HTTPOnly" flag is set on session cookies.

Why

Set the "HTTPOnly" flag on session cookies to protect the contents of the session cookies from Cross-Site Scripting attacks. Setting the "HTTPOnly" flag on session cookies ensures their contents cannot be accessed by user-side scripts.

How to Check

To verify that the "HTTPOnly" flag is set on session cookies:

  1. Log in to the application. Open the application and authenticate. Navigate to pages that uses sessions to make sure that a session is created.

  2. Verify that session cookies have the "HTTPOnly" flag set. Use the web browser to view stored cookies. Make sure that the application session ID cookie has the "HTTPOnly" flag set.

  3. Check the php.ini file. If testing with a browser is not appropriate, check the php.ini file to make sure the "session.cookie_httponly" directive is set to "On".

How to Fix

To set the "HTTPOnly" flag on session cookies, edit the php.ini file and configure the "session.cookie_httponly" setting, like this:

session.cookie_httponly = On
