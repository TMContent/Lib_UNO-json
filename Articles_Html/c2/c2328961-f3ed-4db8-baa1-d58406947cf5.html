Applies to

  • PCI DSS Requirement 11.3

What to Do

Hire a qualified information security company to perform penetration tests for you.

Why

Penetration tests may expose security vulnerabilities and misconfigurations. Penetration tests also exercise the response capabilities of the company to verify that they are sufficient for handling an actual attack.

When

Penetration tests should be performed at least annually and after any significant changes to the environment.

How

Use the following steps to perform penetration tests in compliance with PCI DSS:

  1. Schedule annual penetration tests. Hire a qualified information security company to perform penetration tests for you. Security Innovation offers penetration-testing services. Perform penetration tests at least annually and after any major changes to the network. Express this as a requirement in the information security policy.

  2. Verify penetration tester qualifications. Verify the credentials of the penetration testers to make sure they are qualified. Things to look for are:

    • Independent projects
    • Previous penetration testing experience
    • Experience with penetration testing in the context of PCI DSS
    • Higher education should be a given.
    • Certifications are ok, but they may not accurately reflect the skill of the individual - individuals with vastly different skill levels may have similar certifications.

  3. Perform network layer penetration tests. Make sure the penetration testers perform and document network layer attacks. Negotiate this in advance.

  4. Perform application layer penetration tests. Make sure the penetration testers perform and document application layer attacks. Have them test all web applications for the OWASP Top 10 types of web application vulnerabilities. Negotiate this in advance.
