Applies to

  • PCI DSS Requirement 3.2.1

What to Check For

The full card track contents should not be stored under any circumstances.

Why

The full card track contents should not be stored to limit the potential damage in the event of a successful attack. The term "track contents" refers to the information stored on the magnetic strip of the physical card issued to a cardholder. New cards also include a chip, the contents of which should be also considered "track contents." If an attacker has this information, he may be able to use it to make fraudulent transactions; therefore, the full track contents should never be stored.

How to Check

Check the following to make sure that the full track contents are not stored:

  • Incoming transactions
  • All logs (transaction, history, debugging, error)
  • History files
  • Trace files
  • Database schemas
  • Database contents
  • Any other place where cardholder data is stored

How to Fix

Identify the storage mechanism that stores the full track contents, for example a script or a database query. Change the storage mechanism so that it does not record the full track contents.
