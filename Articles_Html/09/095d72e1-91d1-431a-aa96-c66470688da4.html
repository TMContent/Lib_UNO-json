What to Check For

Check that all access is denied by default in web server settings and in authorization code.

Why

Denying all access by default assures that only explicitly authorized access is possible.

How to Check

To make sure all access is denied by default:

  1. Enumerate all sensitive pages. Make a list of all pages that handle sensitive or private information.

  2. Verify that all access is denied by default on the web server. Examine the web server configuration to make sure that all access is denied by default unless explicitly authorized.

  3. Verify that all access is denied by default in authorization code. Review your application's authorization code to make sure that all access is denied by default.

How to Fix

To deny all access by default:

  1. Enumerate all sensitive pages. Make a list of all pages that handle sensitive or private information.

  2. Configure web-server authorization. Configure the web-server to deny all access by default for all locations that contain sensitive pages or data. The exact procedures are different for different web servers - please, refer to your web server's documentation for detailed instructions on configuring its access control settings.

  3. Deny all access by default in your authorization code. Implement authorization in your application to deny all access by default.
