What to Do

Formulate a storage media policy to comply with PCI DSS and provide guidelines for handling storage media that contains sensitive information.

Why

A storage media policy defines procedures for secure handling of physical storage media that contains or may contain cardholder information.

How

Perform the following actions to formulate a storage media policy in compliance with PCI DSS:

  1. Require a review of the back-up storage location. Add a point to the storage media policy to require the back-up storage location to be reviewed for security at least annually.

  2. Define a media distribution policy. Identify possible scenarios that would require media containing sensitive information to be distributed, such as an external audit. Write down the correct procedures for handling each scenario and add them to the media policy. Add a point to the media policy that requires sensitive media to be classified as "confidential." Add a point to the media policy that requires keeping a log of distributed media. Add a point to the media policy that requires sensitive media sent outside the facility to be shipped via secured courier or some other shipping method that can be tracked. Add a point to the media policy that requires sensitive media sent outside the facility to be authorized by management. Attach the media policy to the information security policy.

  3. Keep a log of distributed media. Start a log of distributed media that includes the following information: the name of the person authorizing distribution, the name of the intended recepient, the contents of the media, the date, and any other information you consider relevant. Attach the media distribution log to the network documentation.

  4. Define a media inventory policy. Add a point to the media policy that requires media inventory to be performed at least once a year. Identify all media storage devices in the cardholder data environment. Number each storage media device. For each storage device in the cardholder data environment, record its number, its location, and its description in the media inventory. Attach the media inventory to the network documentation.

  5. Define a media destruction policy. Add a point to the media policy that requires all media containing cardholder data to be thoroughly destroyed, including paper hard-copies. Acquire a shredder or an incinerator for destroying paper hard-copies or hire a company that specializes in media destruction. Acquire lockable trash bins to store sensitive media before it is destroyed. Only place discarded sensitive hard-copy media in the lockable bins. Follow the instructions in the guideline article called "Erase Files Securely" to destroy physical media securely. It is sufficient to erase the files on digital storage media securely before discarding such media; however, it is recommended to physically destroy the digital storage media itself with a piece of industrial equipment. Number the paragraphs and combine them into a single document. Call the document "The Storage Media Policy," and add it to the information security policy.
