Applies To

Applications written using Scala and Play Framework 2.1.0

What to Check For

Check if helper classes are used to ensure that user input is validated.

Why

Helper classes may help make sure that user input has been validated.

How to Check

To check if helper classes are used to ensure input validation:

  1. Search code for input validation helper classes. Search for a definition of a class that is used as a container for validated input.

  2. Examine the helper class. Verify that the helper class includes a member variable for storing user input, a validation function, and a modifier that calls the validation function.

  3. Verify that the helper class is used. Identify where in code user input is handled. Make sure that the helper class is used to handle user input instead of direct operations.

How to Fix

To assure that user input is validated, helper classes can be used. Below are the steps to make sure that helper classes ensure proper input validation: 

  1. Define a helper class. Define a container class for user input. Since user input is most likely to be in the form of a string, declare a private string member. Declare an accessor to retrieve the value of the string member.

  2. Define a validation function. Define a private validation function as a member of the helper class. This validation function may call other validation functions that are implemented in the input validation class or library.

  3. Define a modifier. Define a modifier for the helper class that takes user input as an argument. Call the private validation function on user input. Return an error if user input is not valid. If user input is valid, assign user input as the value for the private string member. Now, whenever an instance of this helper class has data assigned to it, there is assurance that the data it contains has been validated.

  4. Use the helper class. Identify all locations where user input is handled. Replace direct operations on user input with operations using the helper class.
