What to Do

Isolate multiple applications on a shared server from one another and from shared system resources by using the Medium trust policy setting in the machine-level config file.

Why

If a hosted, malicious application is able to read another application's data or interfere with another application in any way, the integrity of all accessible applications on the server has effectively been compromised. Applications in a hosted environment should not trust each other, and lacking enforcement of application isolation prevents applications from effectively protecting themselves.

When

Whenever multiple applications operate in a shared hosting environment, use the medim trust  privilege level to isolate the applications from each other. If your application needs operations that are not available in medium trust such as event log access, or file I/O, create a custom trust policy rather then elevating privilege to a higher policy level. Medium trust is appropriate when the following permission are not required or needs restricted permission for application's operating capablities:

  • OleDbPermission. This means the application does not use the ADO.NET managed OLE DB data provider to access databases. However, the application can use the managed SQL Server provider to access SQL Server databases.
  • EventLogPermission. This means the application does not use the Windows event log.
  • ReflectionPermission. This means the application does not use reflection.
  • RegistryPermission. This means the application does not access the registry.
  • WebPermission is restricted. This means the application communicate only with an address or range of addresses that you define in the <trust> element.
  • FileIOPermission is restricted. This means the application access files only in your application's virtual directory hierarchy. Your application performs Read, Write, Append, and PathDiscovery operations in its virtual directory hierarchy.
  • The application does not call unmanaged code
  • The application does not use enterprise services

How

To enforce Medium trust for all Web applications on a server, use the following configuration in the machine-level Web.config file :

<location allowOverride="false">
  <system.web>
   <trust level="Medium" originUrl="" />
  </system.web>
 </location>
