Applies to

  • PCI DSS Requirement 8.1

What to Check For

Each user should have a unique user account.

Why

Giving each user a unique account allows managing access control effectively.

How to Check

Use the following steps to check if all authorized users have unique user IDs:

  1. Make a list of authorized users. Review the authorization forms in the network documentation and make a list of authorized users.

  2. Make&nbsp;a list of user accounts. Use the user management application to review user accounts. Make a list of accounts. In a Microsoft Active Directory environment, the user management application can be launched by clicking on "Start", "Programs", "Administrative Tools", "Active Directory Users and Computers".

  3. Compare the list of authorized users to the list of user accounts. Identify the user corresponding to each account to make sure the user exists and uses the account. Make sure that all the authorized users have accounts.

How to Fix

Perform the following actions to make a new user account for each authorized user that does not have one in a Microsoft Active Directory environment:

  1. Launch the user management console. Launch the the user management application by clicking on "Start", "Programs", "Administrative Tools", "Active Directory Users and Computers".

  2. Create a new user account. To create a new user, navigate to the user list, right click on it, select "New", "User"

  3. Configure the new user account. Fill in the user account information in the "New Object - User" dialog. Configure group membership and data access control permissions for the new user account. Peform other actions associated with creating new user accounts, such as adding e-mail accounts, allocating personal work-space, etc.
