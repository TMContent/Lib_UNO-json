Applies to

  • PCI DSS Requirement 1.1.1

What to Do

The network documentation and the information security policy should include a formal testing and approval process for changes made to the firewall settings. The exact nature of any test depends on the change that has been made, but most tests should involve the same common network diagnostic tools. One way to test the changes to the firewall configuration is by performing a detailed port scan of the host(s) protected by the firewall. The nmap port scanner is the industry standard and is available for many common platforms.

Why

Testing the firewall configuration after an update makes sure that the firewall behaves as intended.

When

The firewall configuration should be tested after changes to the configuration have been made.

How

Test changes to the firewall configuration and update documentation in compliance with PCI DSS by performing the following actions:

  1. Define a test of the firewall configuration. Write down the steps required to execute it in the network documentation. A test description may look like this: "Use a computer across the network link secured by the firewall from the target network to execute a port scan using nmap. Use the command "nmap -v -PN -n -p 1-65535 TARGETIP," where TARGETIP is the target IP to execute the port scan. Save the port scan report and compare it to the expected firewall behavior based on the changes made to the firewall configuration. If the port scan produces expected results, then the firewall is believed to be configured correctly."

  2. Conduct a test of the firewall configuration. A typical test may consist of scanning the target host with a port scanner over a network link secured by the firewall. The results of the port scan will indicate whether the firewall configuration is working correctly.

  3. Update documentation. The network documentation should also include a formal process for approving changes to the firewall configuration. Designate a person responsible for the changes to the firewall configuration and decide on the process for approving the changes to the firewall configuration. An approval process can be something like this, "When changes to the firewall configuration are required, contact the person responsible for the firewall configuration. After changes are made, perform a port scan to make sure that the changes are implemented correctly. If the test shows that the changes are correct, the staff member responsible for the firewall configuration may approve them by signing off on the test results.
