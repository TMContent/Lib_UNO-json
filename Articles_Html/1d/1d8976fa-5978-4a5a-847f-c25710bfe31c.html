What to Do

Protect the cookies that contain the session IDs.

Why

Do not disclose session IDs to make it harder for an attacker to compromise a user's session.

When

Protect session IDs in all web applications that use authentication.

How

To prevent disclosure of session IDs:

  1. The session IDs should be sent in non-persistent cookies. This is the setting used by ASP.NET by default. Review the application design to make sure that this default has not been changed.

  2. Find all pages that use session IDs. Make a list of all pages that use ASP.NET session management functions.

  3. Enable a HTTPS binding for the site. Start the "Internet Information Services (IIS) Manager". Right-click on the site and select "Edit bindings...". Add a binding of the "https" type.

  4. Enable TLS on all pages that use session IDs. Start the "Internet Information Services (IIS) Manager". Navigate the folders with pages that use ASP.NET session management. Double-click on the SSL icon in the right pane and enable TLS for the folder (the terms SSL and TLS used to be practically interchangeable).

  5. Mark session cookies as secure. Marking session cookies as secure assures that they are only sent over HTTPS connections.

    if (Response.Cookies.Count > 0)
    {
        foreach (string cookie in Response.Cookies.AllKeys)
        {
            if (cookie == FormsAuthentication.FormsCookieName)
            {
                 Response.Cookies[cookie].Secure = true;
            }
            else if (cookie == “ASP.NET_SessionId”){
                 Response.Cookies[cookie].Secure = true;
            }
        }
    }
