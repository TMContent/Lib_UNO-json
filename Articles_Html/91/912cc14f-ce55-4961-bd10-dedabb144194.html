Applies to

  • PCI DSS Requirement 6.3

What to Check For

All changes, including patches, should be tested before being deployed. Each change should be tested for validation of all input, error handling, secure cryptographic storage, secure communications, and proper role-based access control.

Why

Testing helps ensure that changes do not introduce new vulnerabilities or break functionality.

How to Check

Review the change control documentation to make sure that each change, including patches, is tested for validation of all input, proper error handling, secure cryptographic storage, secure communications and proper role-based access control. While reviewing the documentation, verify that changes are applied only if the tests are successful.

How to Fix

Create a testing environment that simulates the environment where changes are to be applied. Apply changes in the testing environment. Test each change for validation of all input, error handling, secure cryptographic storage, secure communications, and proper role-based access control. Document the results of the tests. If the tests are successful, apply the changes to the production environment.
