Applies to

Scala and Play Framework 2.1.0+

What to Do

Make authentication and authorization policies configurable, rather than hard-coded.

Why

Authentication and authorization policies should be configurable to make it possible for customers to adapt the application's security to their environment.

How

To make authentication and authorization policies configurable:

  1. Identify application requirements.&nbsp;Define options for configuring access control and authentication policies. At the very least, there should be an administration panel for managing user accounts, and a page for users to manage their own accounts.

  2. Make authentication and authorization code configurable.&nbsp;Find all code responsible for authentication and authorization features. Modify this code to read settings defined in the application requirements.

  3. Design the administrative pages.&nbsp;Design the pages to be used for configuring access control and authentication policies based on the requirements for your application.

  4. Implement the administration interface.&nbsp;Write code for the administrative pages to interface with authentication and authorization code. The administration interface should be able to read and write settings to a secure location, where the authentication and authorization code may access them. The saved settings should be protected with access controls to prevent unauthorized disclosure and tampering.
