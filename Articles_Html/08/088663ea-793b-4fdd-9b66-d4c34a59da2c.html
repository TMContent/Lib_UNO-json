Applies To

  • ASP.NET 4.0

What to Check For

Verify that important event details are logged.

Why

Log events entries should include important details so that the contents of the logs are useful for forensics and system administration.

How To Check

Either examine the logs or the code that produces the logs to make sure that the following information is included in log entries:

  • User identification
  • Type of event
  • Date and time stamp
  • Success or failure indication
  • Source of event
  • Names or other identifiers of affected data or resources

How To Fix

Find code that generates log entries. Add code that records the following information in log entries:

  • User identification
  • Type of event
  • Date and time stamp
  • Success or failure indication
  • Source of event
  • Names or other identifiers of affected data or resources
