What to Check For

The information security policy should define acceptable use of technology.

How to Check

Use the following steps to check that usage policies are defined in compliance with PCI DSS:

  1. Verify that management approval is required: Examine the acceptable use policies in the information security policy to verify that they require management authorization to use the technology.

  2. Check that authentication is required: Examine the acceptable use policies to verify that they require authentication to use the technology.

  3. Check that an authorized device list is required: Examine the security documentation to verify that it includes a list of devices and users authorized to use them.

  4. Check that labeling is required: Examine the acceptable use policies to verify that they require devices to be labeled with information about the owner, contact information, and purpose.

  5. Check that acceptable use is required: Examine the acceptable use policies to verify that they require acceptable use.

  6. Verify that acceptable network locations are defined: Examine the acceptable use policies to verify that they require acceptable network locations.

  7. Check that a list of approved products is required: Examine the acceptable use policies to verify that they require a list of approved products.

  8. Verify that remote access time-outs are required: Examine the acceptable use policies to verify that they require idle remote access sessions to be disconnected automatically after some period of time.

  9. Verify that handling of vendor remote access is defined: Examine the acceptable use policies to verify that they require remote access accounts for vendors to be only enabled when in use.

  10. Verify that remote access downloads are forbidden: Examine the acceptable use policies to verify that they prohibit downloading cardholder data over remote access connections to local storage media.

How to Fix

Perform the following actions to define usage policies in compliance with PCI DSS:

  1. Require management approval: Add a point that requires explicit management authorization to use the technologies.

  2. Require authentication: Add a point that requires users to be authenticated with a password or another security mechanism to use the technologies.

  3. Require an authorized device list:  Add a point that requires a list of all devices and personnel authorized to use them.

  4. Require labeling:  Add a point that requires devices to be labeled with owner, contact information, and purpose.

  5. Require acceptable use:  Add a point that requires acceptable uses for the technology. This requirement seems redundant, but it is actually explicit in demanding that technology be used only in its intended purpose.

  6. Define acceptable network locations:  Add a point that requires acceptable network locations for the technology. Acceptable network locations are the networks secured in compliance with PCI DSS. For example, a laptop from the cardholder data environment may not be connected to the Internet directly.

  7. Require a list of approved products:  Add a point that requires a list of company-approved products. Make a list of company-approved products and add it to the information security documentation.

  8. Require remote access time-outs:  Add a point that requires idle remote sessions to be disconnected automatically after a certain amount of time.

  9. Define handling of vendor remote access accounts:  Add a point that requires remote access accounts for vendors to be enabled only when needed and disabled immediately thereafter.

  10. Prohibit remote access downloads:  Add a point that forbids downloading cardholder data to local storage media over a remote access connection.
