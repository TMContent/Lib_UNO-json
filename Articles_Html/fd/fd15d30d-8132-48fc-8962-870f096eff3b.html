What to Check For

Verify that PINs, verification codes, and full track contents are not stored.

Why

One of the most prized assets that attackers are after is cardholder data that can be used for fraud. Do not store PINs, verification codes, or full track contents to prevent this data from becoming available to attackers even if the system is completely compromised.

How to Check

To verify that PINs, verification codes, and full track contents are not stored:

  1. Identify code that handles PINs, verification codes, and full track contents. Make a spreadsheet of all the code that handles PINs, verification codes and full track contents.
  2. Examine code that handles particularly sensitive cardholder data. Examine each piece of code that handles PINs, verification codes and full track contents to make sure that such information is never stored.

How to Fix

To avoid storing PINs, verification codes, or full track contents:

  1. Identify code that handles PINs, verification codes, and full track contents. Make a spreadsheet of all the code that handles PINs, verification codes and full track contents.
  2. Examine code that handles particularly sensitive cardholder data. Examine each piece of code that handles PINs, verification codes and full track contents to make sure that such information is never stored.
  3. Remove code that stores PINs, verification codes, or full track contents. If there is code that stores such sensitive information, remove it.
