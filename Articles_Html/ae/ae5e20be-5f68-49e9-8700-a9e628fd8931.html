What to Do

Write applications that include important details in log entries.

Why

Log event entries should include important details so that the contents of the logs are useful for forensics and system administration.

How

Perform the following actions to add code that logs important event details:

  1. Identify existing logging functionality. Look for libraries or classes that are designed specifically for logging. Make a list of functions used to generate log entries and document them.

  2. Centralize logging. If a dedicated logging library or class doesn't exist, design and write one. Consider implementing functionality for sending logs to an external log server. One common protocol for sending logs to a log server is syslog (RFC 5424).

  3. Implement logging. Add code that records the following details in log entries:

    • User identification
    • Type of event
    • Date and time stamp
    • Success or failure indication
    • Source of event
    • Names or other identifiers of affected data or resources
